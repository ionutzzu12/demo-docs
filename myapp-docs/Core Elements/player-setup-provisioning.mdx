Setting up a player for use involves applying a **Setup** to the player. The Setup is a file which includes various player settings. Some of the available settings and configurations available are:

- Player name and description
- Publishing Mode
  - Sets the method by which content will be delivered to the player. This critical part of a Setup is covered in more detail on the next page.
- Network configuration
- Diagnostic Web Server (DWS)
  - Provides access to detailed info about the player.
  - Contains tools to help monitor and troubleshoot player health and status.
- Logging capabilities
- BrightSignOS updates
  - Apply specific versions of the OS to the player.
- Screenshots
  - See screenshots of what the player is displaying.
- Debugging tools

Applying a Setup to a player using BSN.Cloud (via **Admin > Provision** using BrightAuthor\:connected, bsn.cloud, or control.bsn.cloud) is known as *provisioning*.

<hint type="info">
  Since most situations involve using BSN.Cloud to apply Setups to players, the term *provisioning* is often used to refer to the process of setting up a player.
</hint>
