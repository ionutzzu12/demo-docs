One of the most important settings in a player Setup is the method by which content will be delivered to the player. This is referred to as the player’s Publishing Mode.

Descriptions for each Publishing Mode and their requirements are summarized below. Color-coded labels highlight the key aspects of each mode.

**Labels**:
Network access required for player: <font color="#007dff">NO NETWORK</font> <font color="#007dff">LOCAL NETWORK</font> <font color="#007dff">INTERNET</font>
CMS: <font color="#ff0000">FREE CMS</font> <font color="#ff0000">BSN.CONTENT</font> <font color="#ff0000">PARTNER CMS</font>
User app / interface: <font color="#018826">BA:CONNECTED</font> <font color="#018826">WEB BROWSER</font>
 = Internet

# Standalone

<font color="#007dff">NO NETWORK</font><font color="#ff0000">FREE CMS</font><font color="#018826">BA:CONNECTED</font>

- Content is manually published / copied to a storage device (often a microSD card).
- The storage device is manually inserted into the player.

# Local Network

<font color="#007dff">LOCAL NETWORK</font><font color="#ff0000">FREE CMS</font><font color="#018826">BA:CONNECTED</font>

- Content is published to the player over the local network.

<Image src="https://archbee-image-uploads.s3.amazonaws.com/511c8QBH-VHiwWnyzIwUb/_1-9_MoA7jlo5qk6pM-X3_image-20240723-202645.png" size="60" width="896" height="633" position="center" alt="image-20240723-202645.png" showCaption="false"></Image>

# Web Folder

<font color="#007dff">INTERNET</font><font color="#ff0000">FREE CMS</font><font color="#018826">BA:CONNECTED</font>

- Content is published to a player-accessible web folder.
- The player downloads the content from the web folder.
- Players periodically check the web folder for content updates and when new content is available, it is automatically downloaded to the player.

<Image src="https://archbee-image-uploads.s3.amazonaws.com/511c8QBH-VHiwWnyzIwUb/vtxNI9Z0uGz6BHoQ1l2sd_image-20240723-202734.png" size="60" width="901" height="683" position="center" alt="image-20240723-202734.png" showCaption="false"></Image>

# bsn.Content

<font color="#007dff">INTERNET</font><font color="#ff0000">BSN.CONTENT</font><font color="#018826">BA:CONNECTED</font> or <font color="#018826">WEB BROWSER</font>

- Content is uploaded, stored, published, and managed using BrightSign’s bsn.Content service.

<Image src="https://archbee-image-uploads.s3.amazonaws.com/511c8QBH-VHiwWnyzIwUb/8gvXFsaEe2elUyPyDmYMr_image-20240723-202814.png" size="60" width="927" height="656" position="center" alt="image-20240723-202814.png" showCaption="false"></Image>

# Partner App

<font color="#007dff">INTERNET</font><font color="#ff0000">PARTNER CMS</font><font color="#018826">WEB BROWSER</font>

- BrightSign has many partners who provide an alternative CMS for managing content. This Publishing Mode sets up players to utilize the specified partner’s CMS for content publishing and management.

<Image src="https://archbee-image-uploads.s3.amazonaws.com/511c8QBH-VHiwWnyzIwUb/36P3Osh1fc7qLkz4YkzO9_image-20240723-202902.png" size="60" width="921" height="712" position="center" alt="image-20240723-202902.png" showCaption="false"></Image>

# App URL

<font color="#007dff">INTERNET</font><font color="#ff0000">PARTNER CMS</font><font color="#018826">WEB BROWSER</font>

- Content is packaged and placed at a player-accessible URL.
- The player downloads the package from the URL.

This method is similar to Web Folder but does not have the automatic checks for content updates that the Web Folder method offers.
