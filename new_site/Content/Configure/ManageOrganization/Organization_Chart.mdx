## Viewing the Organization Chart

1. From the Configuration Console, click **Configure > Organizational Profile > Org Chart**. The application displays an organization tree.
2. Expand units within the tree to view the organization hierarchy.

## Viewing Organizational Units

1. Do one of the following:
   - From the Configuration Console, click **Configure > Organizational Profile > Organizational Units**. The application displays an organization tree.
   - From the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Console}} console, open the **Org Unit** workspace.
2. Double-click an item from the list to view the details.

## Creating an Organizational Unit

1. From the Configuration Console, click **Configure > Organizational Profile > Org Chart**. The application displays an organization tree.
2. Click **Edit Org Units**. The **Organizational Unit** page appears, listing the existing organizational units.
3. Click **New Organizational Unit**. The **Organization Unit** page appears. The following default tabs appear (results may vary depending on your application setup):
   - **Details**: Continue to the next step.
   - **Child OUs**: Refer to :Link[Adding Child Organizational Units]{label="Adding Child Organizational Units" overridedLabel="Adding Child Organizational Units" spaceId docId="4aJDXsFykigxrk0n4DVMX" version="v2" docAnchorId="#AddingChildOrgUnits" loadingMethod="dynamic" newTab="false" href="Organization_Chart.mdx"}.
   - **Service Agreements**: One subscription can be linked to one organizational unit. Refer to [Working with Service Level Management](../../ServiceDesk/ServiceLevelMgmt/About_Service_Level_Management.mdx).
   - **Charging Account**:
   - **Fulfillment Item Package**: Refer to [Working with Fulfillment Item Packages](<../../ServiceDesk/ServiceRequest/MultiItemRequests/Fulfillment Item Packages.mdx>).
   - **Audit History**: Refer to [Accessing the Audit History](../AuditHistory/Audit_History.mdx).
4. Enter information into the fields.

| Field                  | Description                                                                                                                                                                                                                         |
| ---------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Name                   | A unique name for this unit.                                                                                                                                                                                                        |
| Parent OU              | The parent organizational unit from the tree.                                                                                                                                                                                       |
| Owner                  | The owner of the organizational unit. By default, the application populates this field with the administrator. To allow another role owner to modify this unit (to restructure parent-child relationships), use the drop-down list. |
| IT Overhead Allocation | A number representing the percentage of IT overhead.                                                                                                                                                                                |
| Deleted                | Removes this organizational unit from the hierarchy. It remains stored in the database.                                                                                                                                             |

5. Click **Save**.

## Adding Child Organizational Units

1. From the Configuration Console, click **Configure > Organizational Profile > Org Chart**. The application displays an organization tree.
2. Click **Edit Org Units**. The **Organizational Unit** page appears, listing the existing organizational units.
3. Click **New Organizational Unit** > **Child OU**.
4. Click the **link** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/3aNmOvuWqAurGzGYP6YMW_link.png" alt caption size="10" width="18" height="11"}. The application displays the list of organizational units.
5. Select an organizational unit from the list.
6. Click **Add**. The application adds the organizational unit to the list of linked organizational units.
7. To create a new organizational unit, click **New Organization Unit**.
8. Click **Save**.

## Deleting an Organizational Unit

1. From the Configuration Console, click **Configure > Organizational Profile > Organizational Units**.
2. Select the organizational unit to remove.
3. Click the **Delete** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/USPl9Egu4Y2Blx7HKC3mA_deleteredx.png" alt caption size="10" width="14" height="13"}.
