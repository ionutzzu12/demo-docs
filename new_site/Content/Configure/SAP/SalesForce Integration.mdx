### About the Integration

The Salesforce.com integration is an optional integration. This integration allows you to synchronize contacts and accounts between Salesforce.com and \{\{FRS\_VARS.ProductName}}. There are three supported mechanisms in this integration:

- You can do a bulk import of accounts and contacts from Salesforce.com to \{\{FRS\_VARS.ProductName}} to initially synchronize the two systems.
- You can configure Salesforce.com to send updates to \{\{FRS\_VARS.ProductName}} when new accounts and contacts are created or existing ones are updated.
- \{\{FRS\_VARS.ProductName}} can update Salesforce.com contacts when modified through a quick action. You can call the quick action manually or configure a workflow to automate the quick action, depending on whether you want all updates to \{\{FRS\_VARS.ProductName}} contacts to be reflected back to Salesforce.com.

You must configure both \{\{FRS\_VARS.ProductName}} and Salesforce.com to enable this integration. :Link[Additional Ivanti Information]{label="Additional Ivanti Information" overridedLabel="Additional Ivanti Information" spaceId docId="OCcKfzf85OCyeXvEhYuIe" version="v2" docAnchorId="#Configuring_the_HEATSettings" loadingMethod="dynamic" newTab="false" href="SalesForce Integration.mdx"} describes changes to \{\{FRS\_VARS.ProductName}} to support importing records, to receive the notifications of changes from Salesforce.com, and to be able to send updates to Salesforce.com. :Link[Setting Up the Salesforce.com System]{label="Setting Up the Salesforce.com System" overridedLabel="Setting Up the Salesforce.com System" spaceId docId="OCcKfzf85OCyeXvEhYuIe" version="v2" docAnchorId="#Setting_Up_the_SalesForce.comSystem" loadingMethod="dynamic" newTab="false" href="SalesForce Integration.mdx"} describes how to configure Salesforce.com to notify \{\{FRS\_VARS.ProductName}} when an account or contact is created or edited.

### Prerequisites

To use the Salesforce.com integration, you must have originally installed \{\{FRS\_VARS.ProductName}} Release 2013.2 or later on your application. If you originally installed an earlier version of \{\{FRS\_VARS.ProductName}} and then upgraded to \{\{FRS\_VARS.ProductName}} Release 2013.2 or later, the Salesforce.com integration will not work.

You must have the following components installed and running on your \{\{FRS\_VARS.ProductName}} application for the Salesforce.com integration to work:

- Message queue
- Integration service
- Workflow service

### Creating or Resetting the Security Token

1. Log into Salesforce.com. The application displays the Salesforce.com home page.
2. At the top of the page, click the down arrow next to your name and select either **Setup** or **My Settings**.
3. From the left panel, do one of the following:
   - If you clicked **Setup**, select **My Personal Information > Reset My Security Token**.
   - If you clicked **My Settings**, select **Personal > Reset My Security Token**.
4. Click **Reset Security Token**. Salesforce.com emails you a new security token. Keep this email.

### Creating a Remote Site

1. Log into Salesforce.com. The application displays the Salesforce.com home page.
2. Click **Remote Site Settings**. Salesforce.com displays the **All Remote Sites** page.
3. Create a new remote site by clicking **New Remote Site**. Salesforce.com displays the **Remote Site Edit** page.
4. Enter information into the fields.

| Field                     | Description                                                                           |
| ------------------------- | ------------------------------------------------------------------------------------- |
| Remote Site Name          | The name of the new remote site. For example, enter **HEATTarget**.                   |
| Remote Site URL           | The URL of the new remote site. For example, enter **https\://mycompany.saasit.com**. |
| Disable Protocol Security | Disables security for the new remote site.  Leave unchecked.                          |
| Description               | An optional description.                                                              |
| Active                    | Specifies if the new remote site is active. Check this option.                        |

5. Click **Save**.

### About Configuring the Settings in Ivanti

You can import data from Salesforce.com into \{\{FRS\_VARS.ProductName}} by importing a .CSV file using the \{\{FRS\_VARS.ProductName}} data import tool (as described in [Configuring Data Import Connections](../ImportExportData/Data-Import-Connections.mdx)). There are connectors created already in \{\{FRS\_VARS.ProductName}} for importing account and contact data. These connections are called **Account Import** and **Contact Import**.

The subsequent data flow between \{\{FRS\_VARS.ProductName}} and Salesforce.com is bi-directional. From the Salesforce.com side, any new accounts or contact business objects and succeeding updates on either business object triggers a web service call to insert or update into \{\{FRS\_VARS.ProductName}}. From the \{\{FRS\_VARS.ProductName}} side, any update on an existing contact triggers a \{\{FRS\_VARS.ProductName}} workflow where a quick action calls the Salesforce.com web service to update the corresponding contact in Salesforce.com.

### Specifying Trusted IP Addresses

Specify three trusted host ranges. Check [https://help.salesforce.com/apex/HTViewSolution?id=000003652](https://help.salesforce.com/apex/HTViewSolution?id=000003652) for a list of all of the published Salesforce IP addresses. The following IP ranges are for the American registry:

- Name: SF1, starting IP: 96.43.144.0, ending IP: 96.43.159.254
- Name: SF2, starting IP: 136.146.0.0, ending IP: 136.147.255.254
- Name: SF3, starting IP: 204.14.232.0, ending IP: 204.14.239.254

Follow these steps to specify a trusted host range:

1. Log into the Service Desk Console.
2. Open the **Integration Trusted Host** workspace.
3. Click **New Integration Trusted Host**.
4. Enter information into the fields.

| Field       | Description                                                |
| ----------- | ---------------------------------------------------------- |
| Name        | A unique name for this trusted host.                       |
| Description | (Optional). A description of this trusted host.            |
| Start IP    | The first IP address in a series of IP addresses to allow. |
| End IP      | The last IP address in a series of IP addresses to allow.  |

5. Click **Save**.

### Updating the CreateSFContact Quick Action

The **External Contact** business object has a workflow for the outbound flow from \{\{FRS\_VARS.ProductName}} to Salesforce.com. You implement a quick action from within the workflow. The quick action calls the Salesforce.com web service to update an existing contact record.

Before you begin, go to the Service Desk Console and open the Salesforce.com configuration that you added in :Link[Adding the Salesforce.com Configuration to Ivanti]{label="Adding the Salesforce.com Configuration to Ivanti" overridedLabel="Adding the Salesforce.com Configuration to Ivanti" spaceId docId="OCcKfzf85OCyeXvEhYuIe" version="v2" docAnchorId="#Adding_the_Salesforce.comConfiguration_toHEAT" loadingMethod="dynamic" newTab="false" href="SalesForce Integration.mdx"}. Make a note of the RecID.

You need to add the RecID to the quick action. Follow these steps:

1. From the Configuration Console, click **Build > Automation Tools > Quick Actions**.
2. For the **ExternalContact** quick action, click **Add/Edit** next to it.
3. In the quick action editor, click the **Your Actions** tab and click **CreateSFContact**.
4. Make a note of the connection name. By default, it is called **CreateSFContact**.
5. From the Configuration Console, click **Extend > Integration Tools > Web Service Connections (BETA)** to open the **Web Service Connections (BETA)** workspace.
6. Under the **Calls from Ivanti to external web services (outgoing)** section, view the script called **CreateSFContact\_ExternalContact**.
7. On the **Integration Script** field, under the **Scripts** heading, in the line that starts with var sfConfig = Get("Frs\_def\_integration\_sfdc\_config#", enter the RecID at the end. The line should look like this: var sfConfig = Get("Frs\_def\_integration\_sfdc\_config#",*RecID*).
8. Click **Save**.

### Updating the Salesforce.com Connector

Before you begin, go to the Service Desk Console and open the Salesforce.com configuration that you added in :Link[Adding the Salesforce.com Configuration to Ivanti]{label="Adding the Salesforce.com Configuration to Ivanti" overridedLabel="Adding the Salesforce.com Configuration to Ivanti" spaceId docId="OCcKfzf85OCyeXvEhYuIe" version="v2" docAnchorId="#Adding_the_Salesforce.comConfiguration_toHEAT" loadingMethod="dynamic" newTab="false" href="SalesForce Integration.mdx"}. 2. Click the **edit** :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/WTLKbMa5PI7qeazzbj0FD_edit-old.png" alt caption size="10" width="16" height="16"} icon next to the **SF Contact Reference** service reference. ### Generating the Enterprise.WSDL API File

The Salesforce.com package includes the **Enterprise.WSDL** file for Salesforce.com for the \{\{FRS\_VARS.ProductName}} integration.

If you modify the schema for the **Account** or **Contact** business objects from Salesforce.com, you must regenerate an updated WSDL from Salesforce.com. Follow the steps below to update the file after you regenerate it:

1. Log into Salesforce.com. The application displays the Salesforce.com home page.
2. At the top of the page, click the down arrow next to your name and select **Setup**.
3. From the left panel, click **Build > Develop > API**.
4. Click **Generate Enterprise WSDL**.
5. Save the **Enterprise.WSDL** file to a convenient location.
6. Open the **Enterprise.WSDL** file with a text editor.
7. Every time that you regenerate the **Enterprise.WSDL** file, you must add the xsd\:attribute as shown below:

\<complexType name="ListViewRecord">

\<sequence>

\<element name="columns"type="tns\:ListViewRecordColumn" maxOccurs="unbounded"/>

\</sequence>

\<xsd\:attribute name="tmp" type="xsd\:string" />

\</complexType>

6. Click **Save**.
