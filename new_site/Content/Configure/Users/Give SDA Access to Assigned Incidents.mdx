At minimum, a \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Analyst-Service-Desk-Analyst}} should be able to view and update his assigned incidents (as specified in the **Owner** field in the **Incident** workspace).

1. From the Configuration Console, click **Configure > Users and Permissions > Roles and Permissions** to open the **Roles and Permissions** workspace.
2. Click **Service Desk Analyst**.
3. Click **Object Permissions**.
4. Navigate to Incident.
5. Click **Edit** in the **Access** column.

*Service Desk Analyst Incident Business Object Permissions*

  The **Service Desk accessing Incident** page appears.
</Paragraph>

6. Click the gray plus sign to add a case.

*Add a Case*

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/gq4RnStoQWO5cxRPFd16r_sda-add-a-case.png" alt="Adding a case for a Service Desk Analyst." position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/sda add a case.png" githubPath="Content/Resources/Images/help-images/sda add a case.png" showCaption="false"></Image>

7. Enter information:

| Value                                |
| ------------------------------------ |
| Update and view                      |
| where                                |
| Employee (via IncidentOwnerEmployee) |
| is                                   |
| the logged in user                   |

8. Click **Save**.

<Paragraph indent="1">
  Upon logging in as a \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Analyst-Service-Desk-Analyst}}, the incident list only shows their assigned incidents.
</Paragraph>
