Modern Survey is the enhanced and improved survey feature for gathering and evaluating customer feedback. Modern Survey offers modern aesthetics, responsive design, and real-time preview, ensuring optimal usability across all devices.

## Enabling Modern Survey

1. Log in to \{\{FRS\_VARS.ProductName}} as an \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}.
2. Open the Configuration console > **Configure** > **Communication Templates** > **Surveys**.
3. In the **Surveys** page, select the **Switch to Modern Survey** toggle button on the top-right-corner.

## Creating a Modern Survey

1. Open the Configuration console > **Configure** > **Communication Templates** > **Surveys**.
2. In the **Surveys** page, select a template or click the **Add Survey** button.

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/GjdTte71lZ0P535QOmTEv_modern-survey-templates.png" alt="Modern Survey Templates - Customer Satisfaction and ITSM User Experience Survey" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/modern-surveys/Modern-Survey-templates.png" githubPath="Content/Resources/Images/help-images/modern-surveys/Modern-Survey-templates.png" showCaption="false"></Image>

3. Edit the properties and questions to customize it to your requirements.
   - To edit the Properties - click the right arrow beside each of the **Properties**.

<hint type="info" indent="2">
  **Survey Title** and **Expiry Date** are the mandatory fields. Currently, only one **Survey Type** option - **on-demand** is available.
</hint>

<Paragraph listStyleType="disc" indent="3">
  **Common Properties** - enter the survey title, survey type, header logo, and pick a color for the survey header, in this section. The recommended dimensions for logo is - **90px** width and **30px** height, with transparent background in **.png** format.
</Paragraph>

<Paragraph listStyleType="disc" indent="3">
  **Welcome & Completion** - enter the welcome message, description of the survey, and a thank you message in this section.
</Paragraph>

<Paragraph listStyleType="disc" indent="3">
  **Suspension/Expiration** - enter the survey expiration message and date in this section.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  Drag or double click a **Standard** o&#x72;**&#x20;Ratings** variety of questions from the **Questions** pane. You can also click **Add question** to add a question to the survey.
</Paragraph>

<Paragraph listStyleType="disc" indent="3">
  **Question** - customize the survey question.
</Paragraph>

<Paragraph listStyleType="disc" indent="3">
  **Question Type** - select the question type. The following options are available:
</Paragraph>

<Paragraph listStyleType="disc" indent="4">
  Select **Single-line text** when you want user to enter information only in single line.
</Paragraph>

<Paragraph listStyleType="disc" indent="4">
  Select **Multi-line text** when you want user to enter information in multiple lines.
</Paragraph>

<Paragraph listStyleType="disc" indent="4">
  Select **Radio Buttons** when you want the user to select a single option from multiple options. You can add or delete the options using plus and cross :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/JJlRsTINYrWYWN8lPpxod_modern-survey-rb-add-del-options.png" alt caption size="10" width="36" height="16"} icons. You can also set the layout as **Horizontal** or **Vertical** to vie&#x77;**&#x20;Radio Buttons** horizontally and vertically from the **Options** section.
</Paragraph>

<Paragraph listStyleType="disc" indent="4">
  Select **Checkboxes** when you want the user to select multiple options. You can add or delete the options using plus and cross :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/JJlRsTINYrWYWN8lPpxod_modern-survey-rb-add-del-options.png" alt caption size="10" width="36" height="16"} icons. You can also set **Horizontal** or **Vertical** as **Layout** to view **Checkboxes** horizontally and vertically from the **Options** section.
</Paragraph>

<Paragraph listStyleType="disc" indent="4">
  Select **Rating - Stars** when you want the user to provide a star rating. You can set the range from one to five stars and can define your own level in the **Options** section.
</Paragraph>

<Paragraph listStyleType="disc" indent="4">
  Select **Rating - Smileys** when you want the user to provide a smiley rating. You can set your own weightage and provide description for each smiley, in the **Options** section. You can toggle to show the **Descriptive text** for each smiley. You can also delete the number of smileys, although minimum of two smileys are mandatory.
</Paragraph>

<hint type="info" indent="3">
  For all the questions there is an **Options** section to the left in which you can define :Link[Manage categories]{label="Manage categories" overridedLabel="Manage categories" spaceId docId="Bb_KYSxR-6zOUUczhKVRM" version="v2" docAnchorId="#categories" loadingMethod="dynamic" newTab="false" href="Modern-Survey.mdx"} and **Response Mandatory**. Also, depending on the **Question Type** the options changes in the **Options** section.
</hint>

<Paragraph listStyleType="disc" indent="2">
  You can also drag the questions to rearrange the sequence.

4. Click **Save** to save all the changes to the survey.
5. Click **Preview** to preview the survey.

<Paragraph indent="1">

### Conditions to display questions

You can set the conditions to render the next question only when the user has selected one of the particular response type.
For example, once the user select a radio button or select a rating (smileys or stars) option in the current question and the next question renders automatically because you have set such type of condition in the survey.

To set conditions on the questions:

1. Click the edit icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/_OSFJ8qCfyY9dTKT2Uqdr_modern-survey-edit.png" alt="Edit icon to edit the survey questions" caption="Edit icon to edit the survey questions" size="10" width="16" height="17"} beside the question and select the **Conditions** tab.
2. Select **Question display logic** to **Display only if a condition is met**. By default, options is selected to **Always display**.
3. In **Condition** section:

<Paragraph indent="1">
  a. Select the question on which you want to apply a condition.
</Paragraph>

<Paragraph indent="1">
  b. Select the condition type - **Is equal to**, **Is not equal to**, **Answered**, or **Not answered**.
</Paragraph>

<Paragraph indent="1">
  c. Select one of the radio button options or one of the rating (smileys or stars) options to fulfill the condition.
</Paragraph>

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/RuLgcd7aYscgdHalwDhTs_conditions.png" position="flex-start" size="40" indent="2" initialPath="../../Resources/Images/help-images/modern-surveys/conditions.png" githubPath="Content/Resources/Images/help-images/modern-surveys/conditions.png" showCaption="false"></Image>

<hint type="info" indent="1">
  Currently **Conditions** are applicable with **Rating - Smileys**, **Rating - Stars**, and **Radio buttons** question types.
</hint>

## Manage categories

Create and use tags to categorize survey questions. You can manage the categories from the Modern Surveys home page.

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/uGVAB_z8T7SvI_0x3PTjh_modern-survey-manage-categories.png" position="flex-start" size="40" initialPath="../../Resources/Images/help-images/modern-surveys/Modern-Survey-manage-categories.png" githubPath="Content/Resources/Images/help-images/modern-surveys/Modern-Survey-manage-categories.png" showCaption="false"></Image>

Click the **Manage categories** button on the **Modern Survey** page.

In the **Manage categories** you can view **In use** and **Not in use** categories. You can delete the **Not in use** categories using the delete icon.

In **New Category** enter the category name and select **Add** button to add a new category
