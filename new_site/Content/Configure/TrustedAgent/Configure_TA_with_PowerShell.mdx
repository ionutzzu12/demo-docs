You can configure an agent to automate specific tasks or workflows by executing external PowerShell scripts. Using PowerShell script an agent can receive and process requests to execute.

## Prerequisites

A self-signed certificate is required during the configuration procedure. The certificate is required to sign the PowerShell script so it can be executed securely. Ensure that you have completed the following prerequisites before you configure Trusted Agent for PowerShell.

<hint type="info">
  To validate the self-signed certificate, you will need a PowerShell script (.ps1) file, and locate this file on the system you are working on, taking note of the file path.
</hint>

<ExpandableHeading>
  ## Create a Self-Signed Certificate

  Self-signed certificate is required for signing the PowerShell script.

  1. Open Windows PowerShell ISE, and then enter the following code in the **Script Pane** (you need to provide a name for the certificate).

  <CodeblockTabs indent="1">
    ```bash
    #certificate  name should be provided
    $CertificateName = "<name of certificate>"
    #Create a Self-Signed SSL certificate
    $Certificate = New-SelfSignedCertificate -CertStoreLocation Cert:\CurrentUser\My -Subject "CN=$CertificateName" -KeySpec Signature -Type CodeSigningCert
    Write-host "Certificate Thumbprint:" $Certificate.Thumbprint
    ```
  </CodeblockTabs>

  2. Run the code.

  <Paragraph indent="1">
    This creates a code signing certificate in your Personal\Certificates folder in Windows Certificate Manager.
  </Paragraph>

  3. Open the Windows Certificate Manager console (search "Manage user certificates" to locate this app), and locate the certificate file in the Personal\Certificares folder.
  4. Copy the certificate file to the Trusted Root Certification Authorities\Certificates folder.
  5. Copy the certificate thumbprint:

  <Paragraph indent="1">
    a. Select the certificate file in Trusted Root Certification Authorities\Certificates.
  </Paragraph>

  <Paragraph indent="1">
    b. Right-click, select **Open**, and then select the **Details** tab.
  </Paragraph>

  <Paragraph indent="1">
    c. Scroll down to find the **Thumbprint** entry, select it, and then copy it from the text field.
  </Paragraph>

  6. In Windows PowerShell ISE, run the following script, entering the thumbprint you copied in the previous step, and the file path to the PowerShell script (.ps1) file you created as mentioned in the note under "Prerequisites":

  <Paragraph indent="1">
    This verifies and signs the code signing certificate in the Trusted Root Certification Authorities\Certificates folder.
  </Paragraph>

  <CodeblockTabs indent="1">
    ```bash
    $thumbprint = "<enter the thumbprint you copied>"
    $ScriptPath = "<enter the file path to the powershell script>"
    #Get the Certificate from Cert Store$CodeSignCert = Get-ChildItem -Path Cert:\CurrentUser\My | Where-Object {$_.Thumbprint -eq $thumbprint}
    #Sign the PS1 fileSet-AuthenticodeSignature -FilePath $ScriptPath -Certificate $CodeSignCert
    ```
  </CodeblockTabs>
</ExpandableHeading>

<ExpandableHeading>
  ## Export the Self-signed Certificate

  1. In Windows Certificate Manager, select the certificate you copied to the Trusted Root Certification Authorities\Certificates folder.
  2. Right-click and select **All Tasks > Export**.

     The Export Wizard opens.
  3. Click **Next**, select **No, do not export the private key**, and then click **Next** again.
  4. Select **DER encoded binary X.509 (.CER)**, and then click **Next**.
  5. Enter a file name, click **Browse** and navigate to the folder you want to save in, and then click **Save**.
  6. Click **Next**, check the settings are as expected, and then click **Finish**.
</ExpandableHeading>

## Upgrade to a Hybrid Policy

You can upgrade an existing LDAP policy to hybrid with both PowerShell and LDAP features.

<hint type="info">
  When you update an LDAP policy to hybrid, you cannot revert the policy back to LDAP only.
</hint>

<ExpandableHeading>
  ## Upgrade to Hybrid

  <hint type="info">
    Before starting this procedure, you need to complete the procedures as described in :Link[Prerequisites]{label="Prerequisites" overridedLabel="Prerequisites" spaceId docId="nKXYfrhJASQND9VFtbedg" version="v2" docAnchorId="#config_powershell_for_TA_Prerequisites" loadingMethod="dynamic" newTab="false" href="Configure_TA_with_PowerShell.mdx"} above to provide a self-signed certificate for the hybrid configuration.

  2. Select the LDAP policy to open the **Trusted Agent Connections** form.
  3. Click the **Policy Type** drop-down and select **Hybrid**.
  4. Click **Select a File**, and then navigate to the folder where you saved the self-signed certificate, select the file, and then click **Open**.
  5. Click **Update Policy Settings**.
</ExpandableHeading>

## Invoke a PowerShell operation with Trusted Agent

Invoke an agent that executes with PowerShell script to update a business object.

1. Log in to \{\{FRS\_VARS.ProductName}} as an \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}.
2. From the **Service Desk** console, open the **Incident** workspace.
3. Open an incident, click **More**, and select **Edit Actions**.
4. Create a quick action using \{\{FRS\_VARS.Graphical\_Action\_Designer}}.

<Paragraph indent="1">
  By default, **Start Block** is selected.
</Paragraph>

5. Enter the **Settings** details for the *mandatory* fields **Name**, **Description**, **Enter Category**,**&#x20;Publish To**, and **UI Options** in the \{\{FRS\_VARS.Graphical\_Action\_Designer}}.

<hint type="info" indent="1">
  Manually enter "Trusted Agent" or "TrustedAgent" (the white space is not required for function) if the **Trusted Agent** category is not present in the **Enter Category** option.
</hint>

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/XpKXS-uudOmD77U8iGsEq_ta-category.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/trustedAgents-images/PowerShell/TA-Category.png" githubPath="Content/Resources/Images/trustedAgents-images/PowerShell/TA-Category.png" showCaption="false"></Image>

6. Drag the **Run Program** from the **Common Actions** to the block.
   - Enter a unique name in the **Block Label** field.
   - Select the PowerShell script in the **Connection** drop-down.
   - In the **Arguments** field specify the argument that needs to be parsed through the Powershell script.

<hint type="info" indent="2">
  Declare arguments, for example: declare a 10, b 5 variables. You can also declare a variable and expression, for example, a 10, UserEmail $(PrimaryEmail), where UserEmail is a parameter in the script, and $(PrimaryEMail) is a field in the business object.
</hint>

<Paragraph listStyleType="disc" indent="2">
  Select the **Output Field** to verify if the PowerShell request was successfully submitted.
</Paragraph>

7. Link the **Start block** to the **Run Program** block and click **Save**.

## Verify Execution

To verify the execution of the agent:

1. From the Configuration console click **Extend > Integration Tools > Trusted Agent Monitoring**.
2. In the **Operational Logs**, select a PowerShell executed operation and verify the **Log Entry** for the associated operation.

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/I2KFIaGXExS406DUiimLd_ta-logs.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/trustedAgents-images/PowerShell/TA-logs.png" githubPath="Content/Resources/Images/trustedAgents-images/PowerShell/TA-logs.png" showCaption="false"></Image>

The **Log Entry** shows whether the script executed successfully with the expected result.

## Creating a workflow for PowerShell Script Execution with Trusted Agent

1. Select a business object and create a [workflow](../Workflows/Using_Workflows.mdx).

<hint type="info" indent="1">
  Ensure the Run Program quick action is created and selected in the Run Program Block Properties.
</hint>

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/DpS-HEbNqX_Gp2n2AmWT8_workflow-configuration.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/trustedAgents-images/Workflow-Configuration.png" githubPath="Content/Resources/Images/trustedAgents-images/Workflow-Configuration.png" showCaption="false"></Image>

2. Set the workflow configuration before saving it.
3. In the respective business object workspace, select a record, and trigger the workflow.

<Paragraph indent="1">
  Once the workflow is executed, refresh the business object to view the powershell execution request status.
</Paragraph>
