<hint type="info">
  This feature is only for ITSM on-premises users.
</hint>

Attaching extremely large files (such as logs) to the application can slow down the application and cause problems. In these cases, you can configure the application to save attachments as external files, rather than attaching them to a specific business object record.

1. Log in to the \{\{FRS\_VARS.ProductName}} Configuration Database (ConfigDB). See the Configuring Attachment Options section in the *Configuration Database Guide for&#x20;*\{\{Primary.Company}} \{\{FRS\_VARS.ProductName}} for more information about using the \{\{FRS\_VARS.ProductName}} Configuration Database.
2. Select the link to your tenant.
3. Set the following:
   - **Attachment path**: Default path where attachments are saved.
   - **Attachment Save Type**: Database, FILESTREAM, File system, Amazon S3, or Azure Blob.
4. Click **Save**.
