title: Opening a Workspace From a URL
docTags: 
createdAt: Tue Jun 30 2026 17:16:51 GMT+0300 (Eastern European Summer Time)
---

For certain workspaces, you can send a URL to users that allows them to open a specific workspace directly from the URL, if you configure them and include them in the workspace. However, this feature is not available for all workspaces.  

Follow these steps to add a workspace that users can open from a URL: 

1. From the Configuration Console, click **Configure&#x20;**>**&#x20;Users and Permissions&#x20;**>**&#x20;Roles and Permissions**.
2. Select a role from the **Role Name** column. The application displays the **Role Details** page.
3. Click **Top Level Tabs**. The application displays the **Top Level Tabs** page. (To create a workspace in \{\{FRS\_VARS.ProductName}} for a role, you create a new top-level tab in the Configuration console for that role.)
4. Click **Add New Tab**. The application displays the **Add New Tab** page.
5. Select a **URL Workspace** as the workspace type. The application displays the **New Tab Details** page.

<hint type="info">
  This feature is applicable only for **URL Workspace** type.
</hint>

6. Enter information into the fields:
   - In the **Name** field, enter a name for this tab. This is the name of the new workspace.
   - Ensure that **Use Url address** is checked.
   - In the **Url** field, click **not set** and enter a valid URL. You must enter either HTTP or HTTPS. Make sure that you enter a valid URL in the **Url** field. If you enter a URL that is not valid and click **Preview**, the application displays a blank page. If you configure a URL without SSL (simple HTTP), Chrome displays an unsafe script message. If you select that badge and allow unsafe scripts, you can display the page. Other browsers may display an error message.
7. Click **Add This Tab**.
8. Click **Save**.

If the application cannot display the external URL that you entered, it displays an empty page.
