You can now add a workspace to \{\{FRS\_VARS.ProductName}} that always displays the contents of a URL. For example, you may want to show information from a monitoring application directly on your dashboard.

You can also add a workspace to \{\{FRS\_VARS.ProductName}} that always displays certain HTML content. For example, you may want to create a nice landing page for end users to see when opening your application.

## Adding a Workspace that Displays a URL

1. From the Configuration Console, click **Configure > Users and Permissions > Roles and Permissions**.
2. Select a role from the **Role Name** column. The application displays the **Role Details** page.
3. Click **Top Level Tabs**. The application displays the **Top Level Tabs** page.

<hint type="info" indent="1">
  To create a workspace in the \{\{FRS\_VARS.ProductName}} Application for a role, you create a new top-level tab in the Configuration Console for that role.
</hint>

4. Click **Add New Tab**. The application displays the **Add New Tab** page.
5. Click **Url Workspace**. The application displays the **New Tab Details** page.
6. Enter information into the fields:
   - In the **Name** field, enter a name for this tab. This is the name of the new workspace.
   - Ensure that **Use Url address** is checked.
   - In the **Url** field, click **not set** and enter a valid URL. You must enter either HTTP or HTTPS.

<hint type="info" indent="1">
  Make sure that you enter a valid URL in the **Url** field. If you enter a URL that is not valid and click **Preview**, the application displays a blank page. If you configure a URL without SSL (simple HTTP), Chrome displays an unsafe script message. If you select that badge and allow unsafe scripts, you can display the page. Other browsers may display an error message.
</hint>

7. Click **Add This Tab**.
8. Click **Save**.
9. To verify that the application displays the new tab, do the following:

<Paragraph indent="1">
  a. Log into the Service Desk Console.
</Paragraph>

<Paragraph indent="1">
  b. Open the workspace that you just created.
</Paragraph>

<Paragraph indent="1">
  c. Verify that the content displays correctly.


</Paragraph>

## Adding a Workspace that Displays HTML Content

1. Log into the Service Desk Console.
2. Open the **URL Workspace Content** workspace.
3. Click **New Frs\_data\_workspace**. The application displays a blank form.
4. In the **Name** field, enter a name for this HTML content.
5. In the **Details** field, enter the HTML to display in the new workspace. For example, you can copy the HTML from a web page and paste it into the **Details** field.

<hint type="info">
  You can also paste images into the HTML; however, each image must be 3 MB or smaller.  Larger images may crash the application and affect performance.
</hint>

6. Click **Save**.
7. From the Configuration Console, click **Configure&#x20;**>**&#x20;Users and Permissions&#x20;**>**&#x20;Roles and Permissions**.
8. Select a role from the **Role Name** column. The application displays the **Role Details** page.
9. Click **Top Level Tabs**. The application displays the **Top Level Tabs** page.

<hint type="info">
  To create a workspace in \{\{FRS\_VARS.ProductName}} for a role, you create a new top-level tab in the Configuration Console for that role.
</hint>

10. Click **Add New Tab**. The application displays the **Add New Tab** page.
11. Click **Url Workspace**. The application displays the **New Tab Details** page.
12. Enter information into the fields:
    - In the **Name** field, enter a name for this tab. This is the name of the new workspace.
    - Uncheck **Use Url address**.
    - In the **Data Name** field, click **not set**. From the drop-down list, select the name of the HTML content that you created in step 4.
    - If you provided translated content for this HTML, click a language to see a preview of the translated content. Refer to [Editing Translation Strings](<../Localization/Editing the Translation Strings.mdx>) for information about translating the HTML content.
13. Click **Add This Tab**.
14. Click **Save**. The application displays all of the top-level tabs for this workspace.
15. To verify that the application displays the new tab, do the following:

<Paragraph indent="1">
  a. Log into the Service Desk Console.
</Paragraph>

<Paragraph indent="1">
  b. Open the workspace that you just created.
</Paragraph>

<Paragraph indent="1">
  c. Verify that the content displays correctly.


</Paragraph>
