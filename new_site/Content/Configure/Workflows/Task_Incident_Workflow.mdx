## About the Task-Incident Workflow

Before you can set an incident to closed, all of its tasks must be closed. Also, once all tasks within an incident are closed, the incident status should automatically change to resolved, or automatically create a new task. However, you can close an incident regardless of the status of its tasks. This automatically closes any open tasks within the incident.

You cannot update an incident when it is in the logged status.

## Enabling the Task-Incident Workflow

The Task-Incident workflow is not enabled by default; you must manually enable this feature to use it. To enable this feature, you must set the Task.Assignment and incident validation on save rules.

### Enabling the Incident Validation On Save Rule

1. From the Configuration Console, click **Build > Business Objects** to open the **Business Objects** workspace.
2. Click **Incident** to open the **Incident** workspace.
3. Choose the **Business Rules** tab. The list of rules appears.
4. At the bottom of this list is the **Validation On Save Rules** section. The only rule listed is shown crossed out (disabled).

*Incident Validation On Save Rule*

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/Yw4HVHQskCWaBD5oHAMln_validationonsave.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/validationOnSave.png" githubPath="Content/Resources/Images/help-images/validationOnSave.png" showCaption="false"></Image>

<Paragraph indent="1">
  The full expression is:
</Paragraph>

$(((Status != "Closed") && (Status != "Resolved")) || IsTrueForAllChildren("Incident#", RecId, "Task#Assignment.Rev3", "$(Status == 'Completed' || Status == 'Cancelled' || Status == 'Rejected')"))

5. Click the **enable** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/bWi-zca3qrtIgxFJBc9Iy_enable.png" alt caption size="10" width="18" height="16"} next to the rule. The rule is no longer crossed out.
6. Click **Save**.

### Enabling the Task.Assignment Validation On Save Rule

1. From the Configuration Console, click **Build > Business Objects** to open the **Business Objects** workspace.
2. Open the **Task.Assignment** business object.
3. Choose the **Business Rules** tab. The list of rules appears.
4. At the bottom of this list is the **Validation On Save Rules** section. The only rule listed is shown crossed out (disabled).

*Task.Assignment Validation On Save Rule*

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/gET0rHH6LxpqHNlATQtoS_taskassignmentvalidationrule.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/task_assignment_validation_rule.png" githubPath="Content/Resources/Images/help-images/task_assignment_validation_rule.png" showCaption="false"></Image>

<Paragraph indent="1">
  The full expression is:
</Paragraph>

$(IfAllSiblings("Incident#.Rev3", "$(Status != 'Closed' && Status != 'Resolved')", "$(Status == 'Completed' || Status == 'Cancelled' || Status == 'Rejected')", "$(PromptAndExecuteAction('$(Parent(Incident#.Rev3))', 'Task belongs to an Incident with no other unfinished Tasks. Select \[Yes] to set Incident resolved, select \[No] to create a new Task, or select \[Cancel] to cancel the Task save operation.', 'Resolve Incident', 'New TaskAssignment'))"))

5. Click the **enable** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/bWi-zca3qrtIgxFJBc9Iy_enable.png" alt caption size="10" width="18" height="16"} next to the rule. The rule is no longer crossed out.
6. Click **Save**.
