Use the **Quick Action** workflow block to define a quick action to execute when a business process has completed. You can select an action associated with this business object.

You can add a quick action to execute when a business process has completed.

<hint type="info">
  The **Export to Excel** quick action is not supported in this workflow block.
</hint>

## Exit Ports

| Exit Port | Description                                                                                          |
| --------- | ---------------------------------------------------------------------------------------------------- |
| OK        | Continue to the next block.                                                                          |
| Failed    | General exit port for a failed integration action. A logical next step is to trigger a notification. |

## Properties

| Property         | Description                                              |
| ---------------- | -------------------------------------------------------- |
| Title            | A unique name for the workflow block.                    |
| Run Quick Action | The quick action to run. Choose from the drop-down list. |

## Actions

| Button | Action                 |
| ------ | ---------------------- |
| Save   | Commits your settings. |
| Cancel | Exits without saving.  |
