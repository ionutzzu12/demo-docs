The **CMDB Import** workflow block performs the SCCM integration. It has a parameter that starts the import from SCCM to the \{\{FRS\_VARS.ProductName}} CMDB. This workflow block submits an integration job but does not keep track of the job status.

## Exit Ports

| Exit Port | Description                                                                                          |
| --------- | ---------------------------------------------------------------------------------------------------- |
| Started   | Continue to the next block.                                                                          |
| Failed    | General exit port for a failed integration action. A logical next step is to trigger a notification. |

## Properties

| Property  | Description                                                                                                                                                         |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Title     | A unique name for the workflow block.                                                                                                                               |
| CMDB Name | The defined integration. Choose from the drop-down list. See [About Importing Data from SCCM into Neurons for ITSM](../../ImportExportData/SCCM-Configuration.mdx). |

## Actions

| Button | Action                 |
| ------ | ---------------------- |
| Save   | Commits your settings. |
