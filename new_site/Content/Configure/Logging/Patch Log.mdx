The **Patch Log** workspace lists information about \{\{FRS\_VARS.ProductName}} development packages. Refer to [Managing Packages](<../Development Project/Managing-Packages.mdx>) for complete information about \{\{FRS\_VARS.ProductName}} development packages and how to use them.

1. From the \{\{FRS\_VARS.ProductName}} application, open the **Patch Log** workspace. You can also access the **Patch Logs** workspace from the Configuration Console by clicking **Monitor > Patch Logs**.

<Paragraph indent="1">
  The application displays a list of logs. Each entry in the **Patch** log contains the following information:
</Paragraph>

| Field        | Description                                                                                                                                                                                                 |
| ------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Patch Time   | The date and time when the patch was created.                                                                                                                                                               |
| Package Name | The name of the package.                                                                                                                                                                                    |
| Index        | Not used. Defaults to 1.                                                                                                                                                                                    |
| Patch Name   | The name of the package. By default, each package has only one patch associated with it, so the patch name is always the same as the package name.                                                          |
| Start Time   | The date and time when the package started to be applied.                                                                                                                                                   |
| Finish Time  | The date and time when the package finished being applied.                                                                                                                                                  |
| Errors       | The number of errors encountered when working with the package. To see the error message, double-click an entry.<br />Do not ignore errors that are reported in the [patch log](Patch-Log-Errors.mdx).      |
| Warnings     | The number of warnings encountered when working with the package. To see the warning message, double-click an entry.<br />You can safely ignore any warnings that occur as a part of the patch application. |
