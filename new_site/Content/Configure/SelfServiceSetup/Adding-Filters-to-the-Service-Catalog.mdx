**Role**: Administrator

***Minimum Version****:&#x20;*\{\{Primary.Company}} \{\{Primary.Product}} 2024.2.

You can enhance the searches available from the Service Catalog by adding filters. Filters allow users to refine their searches and to narrow the list of items that they see on the page. The Service Catalog in the Self Service Mobile portal lists service requests, such as requests for software, computers, or email, similar to what a user would see in \{\{FRS\_VARS.ProductName}}. A default filter category is available to filter searches when using the Self Service Mobile portal. Users can combine filters to create their own searches. This section describes how to use the different types of searches and how to create filters.

## Searching in the Service Catalog

**Search field**

Use the search field to locate a request offering:

- You can perform a free-text search by entering part of the request name, such as **Audio**, **Conferencing**, or **Services**.
- If enabled, you can also perform an exact match search by enclosing the full request offering name in quotes, for example: **"Audio Conferencing Services"**.

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/LA3lfzwF6JvS9YPF_DjTt_text-search-in-service-cataloge.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/SelfServiceMgmt-images/Text search in service cataloge.png" githubPath="Content/Resources/Images/SelfServiceMgmt-images/Text search in service cataloge.png" showCaption="false"></Image>

<hint type="info" indent="1">
  Set the *EnableQuotedFullTextSearch* :Link[global constant]{label="global constant" overridedLabel="global constant" spaceId docId="6CPp-wWweHGMGP37lRlsY" version="v2" docAnchorId="#QuoteSearchGC" loadingMethod="dynamic" newTab="false" githubPath="../Constants/Default_Global_Constants.htm#QuoteSearchGC" href="../Constants/Default_Global_Constants.mdx"} to **True** to perform the exact match search with quotes. Setting the global constant to **False** will perform an exact match search without quotes. This will also disable the ability to search for multiple terms in one search query.
</hint>

**Saved searches:**

Select a predefined filter from the Saved searches dropdown. Each saved search applies filters to display matching request offerings.

**Category filter:**

Use the **Browse by Category** panel to filter request offerings by one or more categories.

## Creating a New Filter Search

The Service Catalog provides a selection of default filter searches, but you can also create your own filters.

For a filter search to appear in the Service Catalog, it must be based on the **ServiceReq** template. There are two parts to creating a new filter search for the Self Service Mobile portal Service Catalog:

- :Link[Creating Filters and Setting Filter Criteria ]{label="Creating Filters and Setting Filter Criteria " overridedLabel="Creating Filters and Setting Filter Criteria " spaceId docId="nm4p2j8idpqcz2gOse0lC" version="v2" docAnchorId="#Part_1:_HEAT_Configuration_Console" loadingMethod="dynamic" newTab="false" href="Adding-Filters-to-the-Service-Catalog.mdx"}describes how to create the filters and set the filter criteria.
- :Link[Enabling the Filter Search in the Self Service Mobile portal]{label="Enabling the Filter Search in the Self Service Mobile portal" overridedLabel="Enabling the Filter Search in the Self Service Mobile portal" spaceId docId="nm4p2j8idpqcz2gOse0lC" version="v2" docAnchorId="#Part_2:_Self_Service_Mobile_Interface" loadingMethod="dynamic" newTab="false" href="Adding-Filters-to-the-Service-Catalog.mdx"} describes how to enable the filter search criteria in the Self Service Mobile portal.

### Creating Filters and Setting Filter Criteria

Administrators must set up the filters from the Configuration Console in \{\{FRS\_VARS.ProductName}}. You should have a clear idea of the types of filters you want to create. It is helpful to name both the **Condition** (or category) and the **Filters** so that they are easily identifiable by your users. Setting up a **Cost** filter is used as an example in the following steps:

1. Log in to \{\{Primary.Product}} as an \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}.
2. Open the Configuration console > **Build&#x20;**>**&#xA0;Search Tools&#x20;**>**&#xA0;Filter Searches** to display the **Filter Searches** workspace.
3. Do one of the following:
   - If the **ServiceReqTemplate** business object is already available, click it to open the **Edit Business Object Filter: ServiceReqTemplate#** workspace.
   - Add the **ServiceReqTemplate** business object by doing the following:

  a. Click **Add**.

<Paragraph indent="2">

<Paragraph indent="2">

5. Click **ServiceReqTemplate** to add or edit its filters.

### Adding a Filter Criteria to the Filter Condition

Do the following from the **Filter Condition** list:

1. Click **Add filter criteria**. The application displays the **Filter Criteria** dialog box.
2. Enter a label. The label is a name that describes what the criteria does. Be as specific as possible so that users understand the intent of this search filter.
3. From the **Control** drop-down menu, choose **Checkbox**.

<Paragraph indent="1">
  This control selection specifies how the filter is displayed in the Self Service Mobile portal. Using a checkbox control allows users to check or clear the filter, especially when enabling the Self Service Mobile portal for mobile devices. The type of control that you choose also affects the available expression choices. Other choices available are **None**, **Calendar**, **Pick List**, and **Textbox** but they are not appropriate for the Self Service Mobile portal.
</Paragraph>

4. Complete the expression by choosing items from drop-down lists. Also refer to :Link[About Expressions]{label="About Expressions" overridedLabel="About Expressions" spaceId docId="4_t7-8mnatsAE0wnALxRp" version="v2" docAnchorId="#AboutExp" loadingMethod="dynamic" newTab="false" githubPath="../../Reference/Expressions/Expression Grammar.htm#AboutExp" href="../../Reference/Expressions/Expression Grammar.mdx"}. ### Creating a Search with Default and Customized Filters

There are two ways to create a search:

- **Combining Criteria from Different Conditions**: When you create a search by combining criteria from different conditions, the application assigns an AND operator. For a service request to appear in the workspace, it must meet the criteria from *both* conditions.
- **Combining Criteria from the Same Condition**: When you create a search by combining multiple criteria from the same condition, the application assigns an OR operator. For a service request to appear in the workspace, it must meet just one of the criteria.

Follow these steps to create a new search:

1. Log in to \{\{Primary.Product}} Self Service Mobile portal.
2. Change your role to \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}.
3. Click the **customize application** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/YAAB5DDDMlOQUFmZec-q2_configureicon2.png" alt caption size="10" width="16" height="15"}.
4. Click **Service Catalog**.
5. Click the **add** icon:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/4ttpBFlTepMuzrkXoCTSV_addblack.png" alt caption size="10" width="19" height="16"} to add a new default search to the **Searches** field.
6. In the **Search Name** field, enter a name for the new search. The name should relate to what the search looks for.
7. Check any boxes under **Category** or any other condition parts that you created.
8. Click **Save**. The application adds the new saved search to the **Search** list. Services that do not meet the search criteria disappear from the workspace.
