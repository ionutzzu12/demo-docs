### Viewing the Integration Queue

The integration queue only shows active or running jobs. Once the import is completed or aborted, the job disappears from the list.

1. From the Configuration Console, click **Extend > Integration Tools > Integration Queue** to view the **Integration Queue** workspace.

<Paragraph indent="1">
  The application displays running jobs. If there are no running jobs, the workspace is empty. The application can process multiple integration jobs at one time.
</Paragraph>

### Viewing the Data Integration Log - File Integration

1. Log in to \{\{FRS\_VARS.ProductName}}.
2. Open the **Integration Log** workspace. The list of data imports appears.

*Sample Integration Log*

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/xULKTUy4Rsn5goMqecdzO_integrationlog.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/integration_log.png" githubPath="Content/Resources/Images/help-images/integration_log.png" showCaption="false"></Image>

<Paragraph indent="1">
  If an import was successful, one line appears under **LogType**: Stats.
</Paragraph>

<Paragraph indent="1">
  Errors encountered are shown with two lines: Stats and ErrorStats.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **Stats**: Double-click to view a summary of the job.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **ErrorStats**: Double-click to view a summary of the errors. Open the attachment to view the rows containing errors. The last column contains specific details about the error. You can modify the data (correct the errors) in the attachment file and save it to the source location for re-import. The error message will be ignored and the data will be imported into the database.
</Paragraph>

### Viewing the Integration Log - SCCM

View the details of the processed job from the **Integration Log** workspace.

1. Log in to \{\{FRS\_VARS.ProductName}}.
2. Open the **Integration Log** workspace. The list of import jobs appear with the following information:

| Field           | Description                                                                                                                                                                                                                                                                                                                                                    |
| --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Log Type        | Shows three log types:<br />**Queued**: Indicates that the source ID from SCCM has been queued in the inventory management asset data processor.<br />**Verified**: Indicates that the source ID from SCCM has been successfully created in the \{\{FRS\_VARS.ProductName}} database.<br />**Stats**: Shows the statistics about the number of rows processed. |
| Source ID       | SCCM root table key (Computer\_System\_DATA) shown on the **Object Mapping** page of the wizard.                                                                                                                                                                                                                                                               |
| Target ID       | Asset ID that is derived from the asset identifier mapping function on the **Field Mapping** page of the wizard.                                                                                                                                                                                                                                               |
| Start Time      | Time when the job should run.                                                                                                                                                                                                                                                                                                                                  |
| End Time        | If applicable, the time by which the job should end.                                                                                                                                                                                                                                                                                                           |
| Total Processed | Total number of records processed. If there are no new mappings added, there will probably be no records processed.                                                                                                                                                                                                                                            |
| Message         | Informational message, which varies according to the log type.                                                                                                                                                                                                                                                                                                 |

<Paragraph indent="1">
  While the job is being processed, you can view it from the **Integration Queue** workspace.
</Paragraph>

### Canceling a Running or Queued Job

You can only cancel a running or queued job.

1. Log in to \{\{FRS\_VARS.ProductName}}.
2. Open the **Integration History** workspace. A list of queued, current (running), and past jobs appear.
3. From the toolbar, click **Abort Integration Event**. The application adds the cancellation to the queue and the job status changes to abort requested. Usually this takes only a few seconds. You might need to refresh the screen to update the status.

*Cancel Integration Job*

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/TJSNvP9WMkVxq5J4AtAQJ_cancelintegration.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/cancel_integration.png" githubPath="Content/Resources/Images/help-images/cancel_integration.png" showCaption="false"></Image>

<Paragraph indent="1">
  When the cancellation process begins, the status changes to aborting. It usually takes less than a minute for a job to cancel, depending on the amount of data being processed.
</Paragraph>

<Paragraph indent="1">
  When the cancellation is complete, the status changes to aborted. This status remains in the **Integration History** workspace.
</Paragraph>
