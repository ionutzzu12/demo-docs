## About Receiving Emails in XML Format

This feature allows external users to send emails formatted in XML to \{\{FRS\_VARS.ProductName}}. The application uses an XSLT script to convert the XML-formatted email into the supported XML format that \{\{FRS\_VARS.ProductName}} uses to create a new incident or other business object.

To support Unicode characters within the email body, check **Compress Message** in the **Inbox Configuration** workspace. Refer to [Working with Email Inboxes](Inbox_Configuration.mdx) for more information.

Refer to [Examples of Processing Emails Received in XML Format](<Export Email Examples.mdx>) for examples of the files described below. Refer to [Hierarchical XML File Examples](<../ImportExportData/Hierarchical XML Examples.mdx>) for information about another \{\{FRS\_VARS.ProductName}} feature that uses XSLT to convert XML.
