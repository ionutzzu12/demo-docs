## About Working with Demo Data

The demo data refresh script brings outdated demo data to current in an out-of-the-box application. This can be useful for sales and services organizations when conducting product demonstration on new features and reporting functionality.

## Prerequisites for Working with Demo Data

The application where the script is applied on must be an out-of-the-box application before any customization takes place.

## Refreshing Demo Data for an Ivanti Cloud Tenant

To refresh demo data on a \{\{FRS\_VARS.ProductName}} Cloud tenant, do the following:

- Open a request with the \{\{FRS\_VARS.Company-short}} Operations team by using the Self-Service Portal.
- In the Service Catalog, choose the **Request to Run SQL Scripts** service offering and specify the URL you are running the script against. Refer to [Working with the Service Catalog](<../../ServiceDesk/ServiceCatalog/Service Catalog.mdx>) for more information.

## Refreshing Demo Data for an On-Premise Tenant

To refresh demo data on an on-premise tenant, you must do the following:

- Run the **Demo Data Update (Final).sql** script on a Microsoft SQL server in your environment.
- Specify the database where you are running the script against.

Refer to the Installation and Deployment Guide for \{\{Primary.Company}} \{\{FRS\_VARS.ProductName}} for more information on the \{\{FRS\_VARS.ProductName}} environment and \{\{FRS\_VARS.ProductName}} databases.
