**Role**: Administrator

Use this procedure to create a request offering in the Self Service Portal V3. A request offering is a predefined form that allows users to submit service requests through the Service Catalog. You can define the service, configure form fields, specify fulfillment actions, and control visibility and access based on organizational units.

1. Log in to the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Console}} console.
2. Open the **Request Offering** workspace.
3. Click **New > Request Offering**.

<hint type="info" indent="1">
  If the **Service** name appears as GUID, the service has been deleted or is marked non-published.
</hint>

You can use these tags in the following fields: **Service Name**, **Offering Name**, **Description**, and **Category** to add customized font styles to display in \{\{FRS\_VARS.Self\_Service\_Portal-Self-Service}} portal.
</hint>

## **Design request form**

<hint type="info">
  From Release 2026.2, Self Service UI V3 supports two and three column layouts for request offerings. On normal screen sizes, the form displays the configured number of columns. On smaller screens, the layout adapts to a single column view to maintain readability and usability.
</hint>

1. Click **Change Image** to edit the default image. See [Default Icons for Request Offerings](../../ServiceDesk/RequestOfferings/Default-Icons-for-Request-Offerings.mdx).
   - The image size cannot be larger than 50 KB.
   - We recommend using square images.
   - The image size can be as large as you like, but is scaled down to 48 by 48 pixels.
2. Click **Layout Configuration** to define into how many columns the fields are to be arranged on the form.

## **Plan Request Fulfillment**

<hint type="info">
  - You cannot edit the workflow if the production metadata is in read-only mode. When this happens, the system displays a message.
  - For more information about setting the production metadata to read-only, see the *Configuration Database Guide for&#x20;*\{\{FRS\_VARS.ProductName}}.
</hint>

1. Select a fulfillment option from the **Fullfill as** dropdown:

<Paragraph indent="1">
  a. No fulfillment - No fulfillment action occurs for this request offering.
</Paragraph>

<Paragraph indent="1">
  b. Email - An email form appears. Enter the information as required. See [Setting Up Email](../Email/Email.mdx).
</Paragraph>

<Paragraph indent="1">
  c. Workflow - Creates a workflow in the Workflow Designer. See [Using Workflows](../Workflows/Using_Workflows.mdx).
</Paragraph>

<Paragraph indent="2">
  You can define a multi-step fulfillment workflow with the Workflow Designer. The designer supports the inclusion of all required approvals and fulfillment tasks. If financial approval is required, include an approval block dedicated for financial approval.
</Paragraph>

<hint type="info" indent="1">
  All Service Request submissions with associated workflows are queued as a high priority. Once queued, the workflow uses the priority defined in the Request Offering.
</hint>

<Paragraph indent="2">
  When you overriding an approval in a Service Request workflow, it may cause the workflow to complete instead of creating additional approval steps. This occurs because the workflow does not receive the value required to determine whether further approvals are required. To ensure the approval process continues, configure the field used in the approval condition with a default or appropriate value in the Service Request template.
</Paragraph>

2. Click **Next**.

## **Publish Action Access**

1. Select an option from the **Status** drop-down list.

| Field     | Description                                                                                                                                                                                                                                                                                                                                  |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| In Design | Saves the form without publishing it to users. You can modify this form later.                                                                                                                                                                                                                                                               |
| Published | Saves the form and makes the request offering available under request offerings, but not under the Service Catalog. When setting this status, by default the organizational unit of the creator/owner is added with all permissions checked. This offering is available in the Service Catalog for this organizational unit unless modified. |

2. Click **Add**.

| Field     | Description                                                                                                                                                                                                                                                                                                                    |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Org. Unit | Selects an organizational unit from the list. You can only publish request offerings to organizational units to which you have access. For example, if you do not have access to the entire organization, your published offering will only be available to users in the organizational unit that you have selected and below. |
| Location  | Only publishes to specific locations. You can designate multiple locations.                                                                                                                                                                                                                                                    |
| Submit    | To submit the request offering from the Service Catalog by the designated organizational unit.                                                                                                                                                                                                                                 |
| Edit      | To modify the request offering (not the resulting service request).                                                                                                                                                                                                                                                            |
| Cancel    | To cancel the request offering.                                                                                                                                                                                                                                                                                                |
| Delete    | To delete the request offering (not the resulting service request).                                                                                                                                                                                                                                                            |
| Copy      | To duplicate the request offering (not the resulting service request).                                                                                                                                                                                                                                                         |

3. Click the **remove** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/USPl9Egu4Y2Blx7HKC3mA_deleteredx.png" alt caption size="10" width="14" height="13"} to remove the organizational unit from the list.

<hint type="info" indent="1">
  - The request offering owner/creator has edit permissions by default.
  - You can only publish to an organizational unit to which you belong, and its parent and child units.
</hint>

4. Click **Save**, **Save & Exit**, or **Cancel**.

<ExpandableHeading>
  ## Enabling a Cancellation Option for Request Offerings

  To enable the cancellation option for request offerings, follow these steps.

  1. Log in to \{\{Primary.Product}} as \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}.
  2. Open the **Request Offerings&#x20;**&#x77;orkspace.
  3. Search for the specific request offering you want to enable the cancellation option for.
  4. Double-click the request offering record.
  5. Under the **Define Request Offering** tab, select the **Enable Request Cancellation** checkbox.

  *View of the Enable Request Cancellation Checkbox*

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/L_YMxP2OZZzlohKCewwTr_enable-cancellation-ro.png" alt="Enabling the ability for a Self Service user to cancel a request offering." position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/Common/Enable cancellation RO.png" githubPath="Content/Resources/Images/Common/Enable cancellation RO.png" showCaption="false"></Image>

  6. Click **Save**.

  Once this is done, when a **Self Service&#x20;**&#x75;ser logs into the **Service Catalog** workspace and clicks **View My Request**, the **Cancel** button will be displayed for this request offering. The user can select **Yes** to cancel the request offering, and a confirmation message will appear, indicating that the request has been canceled with the current date.

  <hint type="info">
    The **Hide Price information** and **Hide Delivery Time information** are supported along with **Enable Request Cancellation**,

    Currently the following options are not supported in Self Service Portal V3: **Enable Request Editing** **Use Price List** and **Anonymous Submit**.
  </hint>
</ExpandableHeading>
