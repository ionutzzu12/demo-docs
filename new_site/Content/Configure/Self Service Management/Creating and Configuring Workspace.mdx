**Role**: Administrator

Workspaces such as **Knowledge**, **Announcements**, **Alerts**, and **FAQ** can be configured to align with your organizational needs. This section provides step-by-step guidance for configuring each workspace

<ExpandableHeading>
  ## Configuring Knowledge Workspace

  You can configure the Knowledge workspace by selecting the **knowledge.SelfServiceNextgen** layout. You can configure this layout in both grid and form views.

  Additionally, you can also refine the knowledge content by adding object filters. For example, if you want to display only published knowledge articles, you can include that filter under object permissions.

  Configuring the Knowledge workspace

  1. Log in to \{\{Primary.Product}} as \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}.
  2. From the Configuration console, click **User and Permissions** > **Role and Permissions**. Select **Self Service** role. The **Role: SelfService** page appears.
  3. Clic&#x6B;**&#x20;Top Level Tabs**, and then click **Add New Tab**.
  4. Under **Add New Tab**, click **Object Workspace**.
  5. Under **Adding New Tab** pane, enter Knowledge in the **Name field**.
  6. In the **Object Workspace's Configuration** pane, select the following in the **Object** and **Layout** fields:
     - Object: **Knowledge**
     - Layout : **Knowledge (Knowledge.SelfService.NextGen)**
  7. Clic&#x6B;**&#x20;Add This Tab**.

  <Paragraph indent="1">
    The **Knowledge** tab is added to the workspace.
  </Paragraph>

  8. Click **Save**.

  <hint type="info">
    To delete this tab, clic&#x6B;**&#x20;Delete This Tab** or click :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/2BTAHsC-8tTlWHYBOIbDP_delete-icon-122x19.png" alt caption size="10" width="22" height="19"}in th&#x65;**&#x20;Top Level Tabs**.

  ## Configuring My Items Workspace

  By default, the **My Items** workspace is configured out of the box. However, you can customize it to meet your organization’s requirements. When you log in to the UIv3 Self-Service portal, the workspace displays your raised requests along with their status.

  1. Log in to \{\{Primary.Product}} as \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}.
  2. From the Configuration console, click **User and Permissions** > **Role and Permissions**. Select **Self Service** Role.

  <Paragraph indent="1">
    The **Role: Self Service** page appears.
  </Paragraph>

  3. In **Top Level Tabs**, click **My Items**.
  4. Select **Enable Issue Templates** check box to modify or delete the existing object names.
  5. In **Configure My Items card**, you can modify the view of **My items** details to appear in \{\{FRS\_VARS.Self\_Service\_Portal-Self-Service}} portal.
  6. Click **Save**.

  <hint type="info">
    To delete this tab, click **Delete This Tab**.
  </hint>
</ExpandableHeading>

## Managing status for My Items

<ExpandableHeading>
  ## Adding a Status to the My Items Table

  To add a custom status or modify an existing status for a request offering and display it in the UIv3 \{\{FRS\_VARS.Self\_Service\_Portal-Self-Service}} workspace, update the **My Item Status** field in home page. After updating this field, the new status appears alongside existing statuses in the workspace.

  1. Log in to \{\{Primary.Product}} as \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}.
  2. On the home page, select **More**.
  3. Search for **My Item Status**, and open the workspace.

  <Paragraph indent="1">
    The workspace displays the following fields:
  </Paragraph>

  <Paragraph listStyleType="disc" indent="2">
    **Status** - Name of the Item status. For an example, statuses are: **Active**, **Draft**, **Approved**, **Rejected**, **Assigned**.
  </Paragraph>

  <Paragraph listStyleType="disc" indent="2">
    **Created Date Time** - Displays the date and time when the status was created.
  </Paragraph>

  <Paragraph listStyleType="disc" indent="2">
    **Lastmod date time** - Displays the date and time when the status was last modified.
  </Paragraph>

  4. Click **New My Item Status**.

  <Paragraph indent="1">
    The application displays a blank form.
  </Paragraph>

  5. Enter the new status in to the text box.
  6. Click **Save**.
</ExpandableHeading>

<ExpandableHeading>
  ## Configure Status Mapping

  You can customize the status train view in **My Items** workspace in the \{\{FRS\_VARS.Self\_Service\_Portal-Self-Service}} portal. You can configure in \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}} home page. Use the **My Items Status Mapping** workspace to define status values, control their display order, and maintain localized labels for service requests.

  Define status values and assign a **sort number** to control the order in which the statuses appear. You can also modify existing status names to meet organizational requirements. For more information, see [Pick\_Lists.](../PickLists/Pick_Lists.mdx)

  The system renders the **Status Train** in the **My Items** workspace in the \{\{FRS\_VARS.Self\_Service\_Portal-Self-Service}} portal based on this configuration. If the configuration is missing or incorrect, the system displays the **Status Train** in a disabled state to indicate a configuration issue.

  ## Create or Edit a My Item Status Mapping

  1. Click **New My Item Status Mapping**.

  <Paragraph indent="1">
    The **New My Item Status Mapping** window appears.
  </Paragraph>

  2. In **Business Object Name** select the required object from the dropdown.
  3. In **My Item Status**, click the **search** icon to select an existing status from the list, or click the arrow and perform one of the following actions:
     - **Add New** - Enter a new status in text field and click **Save**.
     - **Edit** icon - Edit an existing status, update the value and click **Save**.
     - **Go to** - Navigate to the required status and click **Save**.
  4. In **Business Object Status**, select the corresponding status from the dropdown.
  5. In **Sort order**, enter unique numeric value to define the display sequence.

  The following are the limitations of status mapping:

  - You cannot assign a sort order value that already exists for another status within the same business object.
  - You cannot map a single business object status to multiple **My Item Status** values within the same business object. The combination of **Business Object Name**, **My Item Status**, and **Business Object Status** must be unique. In the figure *Viewing My Item Status Mapping* each combination is unique.
  - When the sort order is updated, all mappings with the same **My Item Status** within a **Business Object** are assigned the new sort order.
