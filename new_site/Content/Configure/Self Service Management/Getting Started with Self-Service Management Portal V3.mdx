**Role**: Administrator

**Minimum Version**: \{\{Primary.Product}} 2024.3

**Availability**:

- For new customers - The \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} V3 Portal is available by default.
- For existing customers - The \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} V3 Portal must be enabled manually.

Getting Started with Self-Service Management Portal V3 provides administrators with the necessary steps to enable and configure the latest version of the Self Service Portal. It outlines upgrade paths, UI configuration using the Style Editor, and role-based access setup for both new and existing customers.

## Upgrading to Self Service V3 portal

The following table shows the upgrade paths available to each role type when migrating from V1 (Classic):

| Roles            | Supported Versions                                                          |
| ---------------- | --------------------------------------------------------------------------- |
| Self Service     | V2 (Modern \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} UI) and V3 |
| Non Self Service | V3 only                                                                     |

<hint type="info">
  \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} roles can upgrade from V1 (Classic) to \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} V2 (formerly Modern \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} UI) or directly to \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} V3. You can also move between versions of \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} V1, V2, and V3 (either upgrading or reverting), whereas Non \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} roles are restricted to upgrading directly from V1 to \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} V3.
</hint>

### Enable Self Service V3 portal

This section is applicable for existing customers upgrading to the latest version.

1. Log in to \{\{Primary.Product}} as \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}.
2. In the Configuration console, click **Build** > **User Interface Styles** > **Style Editor**.
3. In the **Exceptions / Custom Roles** section, click **Add Exception**.

4. The **Add Exceptions / Custom Roles** dialog is displayed. You can define the exception based on the current version in use.

<hint type="info" indent="1">
  Modify and save only one option at a time. The system allows only one change per instance.
</hint>

For non \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} roles using V1 (Classic)

To enable a non self service user role from V1 (Classic) to \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} V3 portal, do the following:

<Paragraph indent="1">
  a. Select the appropriate role from the **Role** dropdown list.
</Paragraph>

<Paragraph indent="1">
  b. Select **UI v3** from **UI** dropdown list.
</Paragraph>

<Paragraph indent="1">
  c. Click **Save**.
</Paragraph>

For \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} roles using V1 (Classic) or V2 (Modern)

To enable a \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} user role from V1 (Classic) existing to \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} V3 portal, do the following:

<Paragraph indent="1">
  a. Select the required self service role type from **Role** dropdown list.
</Paragraph>

<Paragraph indent="1">
  b. Turn on the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} toggle to enable \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}} page option.
</Paragraph>

<Paragraph indent="1">
  c. Select **UI v3** from **UI** drop-down list.
</Paragraph>

<Paragraph indent="1">
  d. Select an option from the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Self\_Service\_User}}**&#x20;Page Options** dropdown list.
</Paragraph>

5. Click **Save**.
6. Click the **Log out** icon.
7. Log in as a **Self Service** user to view Self Service Management Portal V3.

<hint type="info" indent="1">
  To revert from the current SSM V3 version to a previous version, see [Rollback from V3](<Rollback from Selfservice V3.mdx>) section.
</hint>
