The \{\{FRS\_VARS.ProductName}} integration web service uses standard web service technologies and can be consumed using various programming languages or platforms such as Microsoft .NET, Java, etc.

This section describes how to create a test console application using Microsoft Visual Studio 2010, specifically for creating the web reference to access the SOAP-based \{\{FRS\_VARS.ProductName}} Integration API.

You can also create other .NET based clients, such as WPF, ASP.NET, and Microsoft Windows forms, by using the same concepts.

To create a new test console application in Visual Studio, follow these steps:

1. Launch Microsoft Visual Studio 2010 and create a new Visual C# console application.

*Creating a New Console Application*

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/JyITngojw2XsE0ThsZNsG_03000009.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/03000009.png" githubPath="Content/Resources/Images/help-images/03000009.png" showCaption="false"></Image>

2. In the Solution Explorer, right click the **References** folder and select **Add Service Reference...**. The **Add Service Reference** dialog box appears.

*Solution Explorer*

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/RSCWScoFPNEJrmWOYN_Ni_0300000a.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/0300000A.png" githubPath="Content/Resources/Images/help-images/0300000A.png" showCaption="false"></Image>

3. Click **Advanced...** in the lower left hand corner. The application displays the **Service Reference Settings** dialog box.
4. Click **Add Web Reference...**. The application displays the **Add Web Reference** dialog box.

*Add Web Reference Dialog Box*

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/5ANqNam0HwEuuzIqtabLS_0300000d.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/0300000D.png" githubPath="Content/Resources/Images/help-images/0300000D.png" showCaption="false"></Image>

5. In the **URL** field, enter the WSDL URL for connecting to the \{\{FRS\_VARS.ProductName}} integration web service. The WSDL URL is https\://*Tenant\_Name*/ServiceAPI/FRSHEATIntegration.asmx?wsdl.
6. In the **Web reference name** field, enter the name of the web reference (for example, frSaaS).  
7. Click **Add Reference**. The application adds a new entry called frSaaS to the **Web References** folder in the Solution Explorer.

*Solution Explorer*

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/VmUCmruLQm1brsU055DEQ_0300000f.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/0300000F.png" githubPath="Content/Resources/Images/help-images/0300000F.png" showCaption="false"></Image>

You can now write code to use the \{\{FRS\_VARS.ProductName}} integration web service.
