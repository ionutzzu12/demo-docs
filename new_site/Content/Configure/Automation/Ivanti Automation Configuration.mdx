## Configuring and Syncing Run Books

In Ivanti Automation, you can create and configure specific tasks by creating separate modules. These modules are included in Run Books, which can carry out the set of tasks that are needed to be executed for routine IT tasks. \{\{FRS\_VARS.ProductName}} integration requires the ISM\_Transaction and p\_status to trigger completed/failed in the \{\{FRS\_VARS.ProductName}} workflow.

Run Books that have been created must contain defined parameters:

1. **ISM\_TransactionID**: RecordID of the Ivanti Automation Transaction Record which will be sent by \{\{FRS\_VARS.ProductName}}.
2. **P\_Status**: Job status that need to be sent back to \{\{FRS\_VARS.ProductName}} using the **Update Record** task. When configuring the run book, ensure that the parameter setting on the **Input** tab new input value “When scheduling job” is unchecked.

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/eLSXafb9YrEqBIHs1UV-7_edit-run-book-parameter.jpg" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/Automation-Images/edit-run-book-parameter.jpg" githubPath="Content/Resources/Images/Automation-Images/edit-run-book-parameter.jpg" showCaption="false"></Image>

<hint type="info" indent="1">
  When configuring the Run Book in Automation, valid values for P\_Status are ‘Completed’ and ‘Failed’. Make sure to enter in one of these two values only.
</hint>

Once Run Books have been created, they must be synced with \{\{FRS\_VARS.ProductName}} by completing the following steps:

1. Log in to \{\{FRS\_VARS.ProductName}} and select your role as an **Administrator**.
2. Click the configuration icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/YAAB5DDDMlOQUFmZec-q2_configureicon2.png" alt caption size="10" width="16" height="15"} in the upper right corner.
3. In the left-hand pane, go to **Integration Tools** and then select **Web Service Connections**.
4. Scroll down to **IVNT\_Automation\_Runbooks** and select the Edit icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/4SRj-mUJbRmnn6kE-ARyB_editicon.png" alt caption size="10" width="33" height="31"}.
5. Follow the steps to run the Integration process and select **Next** after each step. Once you are done, select **Publish**.

<hint type="info" indent="1">
  You can also schedule this to run every day/hours by defining the schedule entry.
</hint>

## Configuring Ivanti Automation Block Properties

1. Log in to \{\{FRS\_VARS.ProductName}} and select your role as an **Administrator**.
2. Navigate to **MORE...**, and under **My Workspace**, select the **Request Offering** workspace.
3. From the list available, select the desired Request Offering.
4. Select the **Plan Request Fulfillment** tab.
5. From the **Fulfill as:** list, the **Workflow** option is selected by default.

<Paragraph indent="1">
  The **Ivanti Automation Block Properties** dialog box opens.
</Paragraph>

7. Depending on the Run Books created in the Ivanti Automation Tool, add or edit the required parameters. For example, for account unlock, only the **p\_username** parameter is required.
   - In the example below, a service request variable was used as the p\_username parameter.

## Enabling Automation for Other Business Objects

To enable the ability for other business objects to call runbooks and trigger automation, follow these steps:

**Prerequisite:**

- Create a relationship between the target business object for example, Change and the IVNT\_Automation\_Transactions business object.

<Paragraph indent="1">
  Example: Relationship between ServiceReq and IVNT\_Automation\_Transactions
</Paragraph>

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/eBdzj14Gsvdp7N5gI36vp_relationships.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/Automation-Images/relationships.png" githubPath="Content/Resources/Images/Automation-Images/relationships.png" showCaption="false"></Image>

<hint type="info">
  It is mandatory to create a relationship between the target business object and the automation business object before you run the automation workflow, else it will not work.
</hint>

**Steps:**

1. Configure the Automation integration per the instructions above; ensuring runbooks sync without issue.
2. Run the workflow.

## Troubleshooting

If your Run Book does not execute as planned or the workflow does not execute properly, there are a number of areas you can go to troubleshoot.

### Viewing the Automation Transaction Business Object

Ivanti Automation Transaction - This business object logs all the transactions that have been carried out particular to this integration. This provides the status of the job and the logs related to a particular transaction.

### Viewing the Integration Log

To view the details of the processed job, you can view the Integration Log workspace.

1. Log in to \{\{FRS\_VARS.ProductName}} using the **Administrator** role.
2. Open the **Integration Log** workspace.
   - To open the workspace, select **More > Integration Log**.
3. From the **Integration Log** workspace, view the information in the columns for the list of logs or open a record to view details.

### Viewing the Integration Queue

While the job is being processed, you can view the record from the **Integration Queue** workspace.

After the job is processed, the record will not be available from the **Integration Queue** workspace. You can then view it from the Integration Log workspace.

1. Log in to \{\{FRS\_VARS.ProductName}} using the **Administrator** role.
2. Open the **Integration Queue** workspace.
   - To open the workspace, select **More** > **Integration Queue**. The workspace will be empty when there are no records to process.
3. When the record appears in **Queued Status** in the **Integration Log** workspace, then the record appears while it is being processed in the **Integration Queue** workspace.
