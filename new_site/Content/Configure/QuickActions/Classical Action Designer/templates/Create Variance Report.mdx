## About the Create Variance Report Action

The **Create Variance Report** action creates a variance report for the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Configuration\_Manager}} or \{\{FRS\_VARS.Neurons\_ITSM\_-\_Discovery\_Manager}} showing the changes that have been made since the last recorded baseline.

### Parameters

| Parameter                | Description                                                                             |
| ------------------------ | --------------------------------------------------------------------------------------- |
| Enter action name        | A unique name for the action.                                                           |
| Enter action description | Optional. A description of what the action does.                                        |
| Enter category           | Optional. The category. Select from the drop-down list.                                 |
| Run Async                | Generates the report asynchronously, that is, after the event that caused the variance. |

### Actions

| Button | Action                 |
| ------ | ---------------------- |
| Save   | Commits your settings. |
| Cancel | Exits without saving.  |

<hint type="info">
  This action does not respect the **Auto-save** checkbox option in the **Action Settings** tab. It does save changes directly to the database, but please ensure that the behaviour you expect still occurs.
</hint>
