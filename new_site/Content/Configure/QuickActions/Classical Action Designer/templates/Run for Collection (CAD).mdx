## About the Run for Collection Quick Action

<hint type="info">
  To use this \{\{FRS\_VARS.Quick\_Action}}, set the global constant **EnableActionService&#x20;**&#x74;o **True**. If **EnableActionService&#x20;**&#x69;s not available in the global constants list (this may be the case for upgraded instances), Administrators can manually add the constant.
</hint>

The Run for Collection \{\{FRS\_VARS.Quick\_Action}} allows you to run an action against multiple records in a collection or array. The collection of records can be supplied by an expression or from a webhook, in JSON or XML. This ability means you can create or update business objects based on the data received via a webhook or Web Service Request.

You can also choose whether an action should stop if an error occurs or carry on. The maximum size of a collection is fifty records.

## Suggestions for Use

Here are a few ways you could use the Run for Collection \{\{FRS\_VARS.Quick\_Action}}:

- **Use in conjunction with the Go to Record&#x20;**\{\{FRS\_VARS.Quick\_Action}}: Specify a JSON or XML collection of records to update, where each entry has a unique identifier for a different record. Use the Run for Collection \{\{FRS\_VARS.Quick\_Action}} and the Go to Record \{\{FRS\_VARS.Quick\_Action}} together to iterate over the collection, go to the record for each item in the collection, and perform some action on the record before proceeding to the next.
- **Export data from another system and import it into&#x20;**\{\{FRS\_VARS.ITSM}}**&#x20;business objects**: If an external system allows the export of data into JSON or XML format, you could map that data to an \{\{FRS\_VARS.ITSM}} business object by defining an action to create a record based from the JSON data. This data could be sent to \{\{FRS\_VARS.ITSM}} via a webhook. Use the Run for Collection \{\{FRS\_VARS.Quick\_Action}} to create multiple records at once .

<hint type="info">
  Re: data exports, the limit of 50 records per run of an action may not be suitable for bulk imports at this time.
</hint>

- **Enable real-time sharing of data between applications of a business:** An external application can send multiple data items in a JSON/XML array to \{\{FRS\_VARS.ITSM}} via webhooks at any time, which \{\{FRS\_VARS.ITSM}} can respond to with the desired action to create or update business objects, send emails, etc.
- **Use in combination with another action:&#x20;**&#x59;ou could configure an action, such as the [Run REST Web Service Quick Action](<Run REST Web Service.mdx>), that calls an external web service. That web service could return some data in JSON or XML format, which is stored in a variable, and then processed using the Run for Collection \{\{FRS\_VARS.Quick\_Action}}.

## Service Log Entries

To see the results of using the **Run for Collection** \{\{FRS\_VARS.Quick\_Action}}, open the Service Log. See: :Link[Viewing Logs]{label="Viewing Logs" overridedLabel="Viewing Logs" spaceId docId="oRhaO5_Yg_ZmNqqGeAhQE" version="v2" docAnchorId="#Viewing" loadingMethod="dynamic" newTab="false" githubPath="../../../Logging/Logging.htm#Viewing" href="../../../Logging/Logging.mdx"}.

You can use the following information to view the outcome of a run, and for diagnostic and debugging purposes:

- The total length of the iteration.
- The number of successes/total.
- The number of failures.
- Failure messages, if any.
