## About the Execute Workflow Action

Use the **Execute Workflow** action to run a workflow on demand. Executing this action is equivalent to clicking **Re-run Workflow** from the **View workflow definition and instance** dialog box in the Action Menu. See [About Workflows](../../../Workflows/Workflows.mdx) for more information about workflows.

<hint type="info">
  The Workflow action does not work when executed from the form view of a Business Object unless the **Automatically save changes when action is run in UI form context** check box is selected while creating the workflow.
</hint>

### Parameters

| Parameter                | Description                                             |
| ------------------------ | ------------------------------------------------------- |
| Enter action name        | A unique name for the action.                           |
| Enter action description | Optional. A description of what the action does.        |
| Enter category           | Optional. The category. Select from the drop-down list. |
| Workflows                | The name of the workflow to execute.                    |

### Actions

| Button | Action                 |
| ------ | ---------------------- |
| Save   | Commits your settings. |
| Cancel | Exits without saving.  |

<hint type="info">
  This action does not respect the **Auto-save** checkbox option in the **Action Settings** tab. It does save changes directly to the database, but please ensure that the behaviour you expect still occurs.
</hint>
