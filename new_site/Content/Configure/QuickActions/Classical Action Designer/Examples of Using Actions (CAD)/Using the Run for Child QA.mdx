## Prerequisites

This example assumes you have administrator-level access to \{\{FRS\_VARS.ITSM}} and are familiar with creating both business objects and \{\{FRS\_VARS.Quick\_Action}}s.

## How to create a Run for Child Quick Action

To create a **Run For Child** \{\{FRS\_VARS.Quick\_Action}}:

1. From the Configuration console, click **Build > Automation Tools > Quick Actions** to open the **Actions Manager** workspace. The list of business objects that have actions appears.
2. Select a business object from the list and click **Add/Edit**.
3. In the Actions Center, click the **Create New Actions** tab.
4. Under the **Create New Actions** tab, click **Run for Child**.
5. Enter your parameters in the fields provided. Examples of business objects and their parameters are shown in the tables below.
6. Click **Save**.

## Example using the Release (ReleaseProject) Business Object

### Scenario

A company has a number of change records linked to a Release business object. They want to use \{\{Primary.Company}}\{\{Primary.Product}} to update the status of these changes (RFCs) in a batch to the **Implemented&#x20;**&#x73;tatus. By leveraging the Run for Child \{\{FRS\_VARS.Quick\_Action}}, the company can easily set up a way to update these associated "child" records in \{\{Primary.Product}}.

| Field                    | Parameter                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Enter action name        | Update Linked Changes to Implemented Status                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| Enter action description | Updates the status of a linked change to Implemented if its status is approved.                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| Enter category           | Form actions<br />Choose a category to place your new action in the **Your Actions** tab on the left of the screen. OOTB category options include **No Category, Form Actions, New Milestones, Print, Search and Link, System** or **Windows 7 Migration Milestones**.<br />To create your own category, type a name instead of selecting from the **Category&#x20;**&#x64;rop-down list.                                                                                                                                          |
| Relationship             | ChangeAssociatedReleaseProject                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| Child Object Filter      | $(ApprovalCondition == "true")<br />This means that the action only runs on change records where the field **ApprovalCondition** is **true**.                                                                                                                                                                                                                                                                                                                                                                                      |

### Run the Scenario

1. Log in as a Change Manager.
2. Create a new Change with Typ&#x65;**&#x20;Emergency**.
3. Run the **Update Status to Implemented** \{\{FRS\_VARS.Quick\_Action}}.

<Paragraph indent="1">
  The **Status&#x20;**&#x66;ield is set to **Implemented**, and if the Approvals and Approval Vote Tracking tabs do not exist yet, they are created.
</Paragraph>

## Example using the Service Request Business Object

### Scenario

A company has a number of task records linked to the Service Request business object. They want to use \{\{Primary.Company}}\{\{Primary.Product}} to update the status of these records in a batch so that once the task reaches 30 days old, its Status gets sets to **Cancelled**.

| Field                    | Parameter                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Enter action name        | Cancel Tasks                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| Enter action description | Cancels the service request assignment once it reaches 30 days old.                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| Enter category           | System<br />The category chosen sets the location for your new action in the **Your Actions** tab on the left of the screen. Choose from **No Category, Button, Escalation Engine** or **Mobile** OOTB categories.<br />To create your own category, type a name instead of selecting from the **Category&#x20;**&#x64;rop-down list.                                                                                                                                                                    |
| Relationship             | ServiceReqAssocTaskAssignment                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Child Object Filter      | $(BeginningOfDay(30))                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| Action to run            | Cancel<br />This action is set up as follows:<br /><Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/b0G1VpxOGyKDW_0kFCJEx_examplecadrunforchild4a.png" position="flex-start" size="40" initialPath="../../quick-actions-images/example_CAD_RunforChild4a.png" githubPath="Content/Configure/QuickActions/quick-actions-images/example_CAD_RunforChild4a.png" showCaption="false"></Image><br />It sets the Task record's **Status&#x20;**&#x66;ield to **Cancelled**. |

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/GIYWTcKNgXgfVWiFEH3YB_examplecadrunforchild4.png" position="flex-start" size="40" initialPath="../../quick-actions-images/example_CAD_RunforChild4.png" githubPath="Content/Configure/QuickActions/quick-actions-images/example_CAD_RunforChild4.png" showCaption="false"></Image>

### Run the Scenario

1. Log in as an Administrator.
2. Open the Service Request workspace and select a record with associated tasks that are older than 30 days. If you cannot find tasks of that age, modify the Child Object filter value in the \{\{FRS\_VARS.Quick\_Action}} to a lower number, such as 5 days, for testing the action.
3. Check that the **Status&#x20;**&#x66;ield is not set to **Cancelled&#x20;**&#x61;lready.

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/PQnpZe7ymeAp4WHAN5kUj_examplecadrunforchild4c.png" position="flex-start" size="40" indent="1" initialPath="../../quick-actions-images/example_CAD_RunforChild4c.png" githubPath="Content/Configure/QuickActions/quick-actions-images/example_CAD_RunforChild4c.png" showCaption="false"></Image>

4. Click on **More > System > Cancel Tasks**.
5. Enter a **Reason for Cancellation** when prompted.
6. When the Cancel Tasks \{\{FRS\_VARS.Quick\_Action}} has been run, the Status field is updated to **Cancelled**.
