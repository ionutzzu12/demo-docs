## About the Neurons for Healing Quick Action

The \{\{FRS\_VARS.Neurons\_for\_Healing}} action can be used to automatically invoke and execute a Neurons workflow for a target device or user from a support ticket in \{\{FRS\_VARS.Company}} \{\{FRS\_VARS.ProductName}}. For more information on how to configure and create a Neurons workflow: [Ivanti Neurons Workflow Integration](../../../Integrations/Neurons-workflow-config.mdx)

## Parameters

To access the parameters, select the **Action Block** to display them in the **Settings** pane.

### Action Block Settings

- **Block Label**: A unique name for the block. This is useful to identify each action when you have more than one of the same action block added or cloned in the design area.
- **Workflow**: Select the workflow from the list that you created in the \{\{FRS\_VARS.Neurons}} tenant.
- **On Success**: Select the Quick Action to run when the workflow executes successfully. You can either use an existing Quick Action or create a new Quick Action for the **On Success** field.
- **On Failure**: Select the Quick Action to run when the workflow is a failure. You can either use an existing quick action or create a Quick Action for th&#x65;**&#x20;On Failure** field.
- **Enter Keywords**: Keywords for searching the **Neurons** workflow.

Click **Save** to commit your settings.
