## About the Run for Child Quick Action

**Run for Child** applies an action to a child business object of the current business object. For example, you can run an **Update** \{\{FRS\_VARS.Quick\_Action}}, and then an **Export Data to Microsoft Excel** \{\{FRS\_VARS.Quick\_Action}}. Refer to [Example: Using the Run for Child Action (CAD)](<../../Classical Action Designer/Examples of Using Actions (CAD)/Using the Run for Child QA.mdx>) for examples of using this action.  

## Parameters

To access the parameters, select the **Action Block** to display them in the **Settings** pane.

### Action Block Settings

- **Block Label**: A unique name for the block. This is useful to identify each action when you have more than one of the same action block added or cloned in the design area.
- **Relationship**: The relationship between the current business object and a child object.
- The action is applied to the child object in the relationship that you select from the drop-down list.
- **Child Object Filter**: Optional. Opens the **Expression Editor**. Enables you to constrain the child object being evaluated.
- **Action to Run**: The action to run for the child business object. Choose from the drop-down list.

### Additional Settings Functions

Three additional functions are available in the **Settings** pane for this action:

- **New**: Opens the \{\{FRS\_VARS.Graphical\_Action\_Designer}} (GAD) in a new tab, where you can add a new action to use for the **Run for Child** action. Select a relationship in the **Relationship** drop-down list to enable this function. The **Run for Child** action is left open in its own GAD tab. Once the new action is created, return to the **Run for Child** action, and click **Refresh** to display the new action in the **Action to run** drop-down list.
- **Edit**: Opens the action you select in the **Action to run** field, and displays it in a new GAD tab.
- **Refresh**: Click to refresh the **Action to run** drop-down list. For example, if you added a new action as described in **New** above, the refresh adds it to the **Action to run** drop-down list.

Click **Save** to commit your changes.
