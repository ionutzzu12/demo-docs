## About the Create a Child Record Quick Action

The **Create a Child Record** action inserts a child business object into a form or list, such as a new incident, field value, or a note. Use this action to add clarity, constructing additional expressions restricting the viewing and use of the action to certain field states.

## Parameters

To access the parameters, select the **Action Block** to display them in the **Settings** pane.

### Action Block Settings

- **Block Label**: A unique name for the block. This is useful to identify each action when you have more than one of the same action block added or cloned in the design area.
- **Child Object**: The child business object to insert. Select from the dropdown list. The fields that appear are based on this value.
- **Field Name**: The name of the field to insert.

<hint type="info" indent="1">
  For version prior to 2025.4 release, to enable the save functionality for any custom object, enter ‘Save’ as the name text field. This changes the button’s function to ‘Save’ for that custom object.
</hint>

- **Field Value**: The value of the field to insert.
- **Web UI parameters**: This field is only used to post the object to the Social Board, and it takes only one expression:

<Paragraph indent="2">
  \{socialshare\:true, elementId: $(elementId)}
</Paragraph>

<Paragraph indent="1">
  For example, in an Incident, this field is used in the Post social massage associated with Incident action.
</Paragraph>

- **Add Field**: Adds a field to the list.

Click **Save** to commit your changes.
