## About the **GoTo Action** Quick Action

<hint type="info">
  To use this \{\{FRS\_VARS.Quick\_Action}}, set the global constant **EnableActionService&#x20;**&#x74;o **True**. If **EnableActionService&#x20;**&#x69;s not available in the global constants list (this may be the case for upgraded instances), Administrators can manually add the constant.
</hint>

Use the **GoTo Action** to enable a user to jump to a specific action within a composite action.

For example, the **GoTo Action** is useful when implemented to jump to a **Decision Block** action from within a composite action. The user can then decide on a course of action offered by the **Decision Block** action.

## Parameters

To access the parameters, select the **Action Block** to display them in the **Settings** pane.

### Action Block Settings

You must complete all required fields to save your \{\{FRS\_VARS.Quick\_Action}}.

- **Block Label**: A unique name for the block. This is useful to identify each action when you have more than one of the same action block added or cloned in the design area.
- **Target Action**: Select the action within your composite action that you want to jump to.
- **Max Number of Iterations**: Select the maximum number of times, between 1 and 10, that the action can be run.
