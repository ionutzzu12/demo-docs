## About the Knowledge Uploader (Knowledge Import Tool)

You can import knowledge articles from \{\{FRS\_VARS.Company}} Classic, ITSM, or a local or networked file application into \{\{FRS\_VARS.ProductName}}. \{\{FRS\_VARS.ProductName}} currently has two knowledge import tools: the Knowledge Import Tool NXT (for NXT formats such as ITSM or \{\{FRS\_VARS.Company}} Classic) and the Knowledge Import Tool (to use with file systems). These tools are collectively called the Knowledge Uploader and are also called the Knowledge Import Tool.

## About Installing the Knowledge Uploader (Knowledge Import Tool)

When you install \{\{FRS\_VARS.ProductName}}, one of the components that you can choose to install is the Knowledge Uploader (also called the Knowledge Import Tool).

For complete information about installing \{\{FRS\_VARS.ProductName}}, including the Knowledge Uploader (also called the Knowledge Import Tool), see the Installation and Deployment Guide for \{\{Primary.Company}} \{\{FRS\_VARS.ProductName}}.

## Initially Configuring and Logging Into the NXT Knowledge Import Tool

<hint type="info">
  There are two knowledge tools. This topic is about the NXT Knowledge Import Tool and NOT the Knowledge Import Tool.
</hint>

After you have installed \{\{Primary.Company}}, including the Knowledge Uploader (also called the Knowledge Import Tool), do the following to initially configure and log into the Knowledge Import Tool NXT:

1. On the application where you installed \{\{FRS\_VARS.Company}} \{\{FRS\_VARS.ProductName}}, open the application called **Knowledge Import Tool NXT**. The application displays the **NXT Knowledge Server Configuration** dialog box.

*NXT Knowledge Server Configuration Dialog Box*

2. Enter information into the fields.

| Parameter      | Description                                                                                                                                                                                                                                                                                                                           |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Server Version | The version of the NXT server that you are taking the knowledge articles from. To determine the server version, open the program and find the server information.<br /><br />You can choose from any of the following:<br />Knowledge Service V5.0.4003.3<br />Knowledge Service Plus V9.6.1.13<br />Knowledge Service Plus V9.6.1.14 |
| Server Address | Enter the following:<br />The protocol type (uses TCP by default)<br />The server path of the NXT server that you are taking the knowledge articles from<br />The port (uses 2242 by default)                                                                                                                                         |
| User Name      | The user name of a person who can access the NXT server that you are taking the knowledge articles from.                                                                                                                                                                                                                              |
| Password       | The password associated with the user name.                                                                                                                                                                                                                                                                                           |
| Save password  | Saves the password.                                                                                                                                                                                                                                                                                                                   |

<hint type="info">
  To change these settings after the initial configuration, from the menu bar select **Configuration > Configure NXT server**.

3. Click **Continue**.
4. The application displays the **Login** dialog box.

5. Enter information into the fields.

| ------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| User Name     | The user name of a person who can access \{\{Primary.Company}} \{\{FRS\_VARS.ProductName}}.                                                                            |
| Password      | The password associated with the user.                                                                                                                                 |
| Tenant        | The tenant to log into. You can select either ConfigDB to log into the configuration database or HEATSM to log into \{\{Primary.Company}} \{\{FRS\_VARS.ProductName}}. |

  To change these settings after the initial configuration, from the menu bar select **Configuration > Configure application server**.

## Using the NXT Knowledge Import Tool

<hint type="info">
  There are two knowledge tools. This topic is about the NXT Knowledge Import Tool and NOT the Knowledge Import Tool.
</hint>

Follow these steps to import knowledge into \{\{Primary.Company}} \{\{FRS\_VARS.ProductName}}.

1. Click **Select Articles** to select the knowledge articles to import. The application displays the **NXT articles selection** dialog box.
2. Expand the list to see where knowledge articles are located, and then highlight one or more articles.

*Select the Knowledge Articles to Import*

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/eq6xMuP3OoYDcXW4Do2M1_choose-knowledge-article.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/Choose knowledge article.png" githubPath="Content/Resources/Images/help-images/Choose knowledge article.png" showCaption="false"></Image>

3. Click **OK**. The application lists the selected articles on the bottom part of the page.
4. Select the articles to upload and click **Upload** to launch the **Article Creation** wizard and start the upload process.

<Paragraph indent="1">
  The **Article Creation wizard** page contains two main areas: a **Selected Documents** area on the left containing the list of files to import, and a **Document Properties** area on the right that contains two tabs. One tab is called **Default Values** and the other is called **Selected Documents**.
</Paragraph>

5. To remove a document from the **Selected Documents** section, check it and click **Remove selected**. By default, the knowledge uploader imports all of the documents in the **Selected Documents**, even if you do not select them.
6. Optionally, use the **Document Properties** section to specify the properties that the knowledge uploader displays together with the files in \{\{Primary.Company}} \{\{FRS\_VARS.ProductName}}. You can specify properties for all files (using the **Default values** tab) or for individual files (using the **Selected Document** tab after highlighting the file in the **Selected Documents** section).

<Paragraph indent="1">
  You can specify these properties:
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **Article Type**: The article type. Select from Document, Error Message, Issue Resolution, Patch, Q\&A, or Reference.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **Article Status**: The status of the article. Select from Approved, Draft, Expired, Failed Review, Pending Review, Published, or Submitted.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **Target collection**: The target. Select from Company Policy & Procedure, Customer Knowledge, Internal Knowledge, IT Knowledge, or Service Desk.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **Category**: The category for this knowledge article.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **Subject**: The subject of the article.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **Resolution**: The resolution details if the knowledge article contains this information.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **Description**: A description for the knowledge article.
</Paragraph>

7. Click **Import**. The application starts importing the selected knowledge articles.

<Paragraph indent="1">
  If you select a file that has already been imported into the knowledge base, the application displays the **Collision warning** dialog box.
</Paragraph>

8. Depending on what you want to do with the file that has already been imported, click one of the following:
   - **Apply for all**: Applies your choice to all previously imported files that are found during the import process.
   - **Update**: Overwrites the file that already exists in the Knowledge Base with the file that is being imported.
   - **Create New**: Does not overwrite the file being imported with the file that already exists in the Knowledge Base. After importing, there are two knowledge articles with the same name in the Knowledge Base.
   - **Cancel**: Cancels the import process.

<Paragraph indent="1">
  The application displays a dialog box with the status of the import.
</Paragraph>

9. Click **OK**.
10. Optionally, do the following:
    - Review the import logs by clicking the **Log** tab. You can clear or save the log.
    - Review the import history by clicking the **Import History** tab. Click a number in the **Imported** column to display more information about an import session.

## Using the Knowledge Import Tool

<hint type="info">
  There are two knowledge tools. This topic is about the Knowledge Import Tool and **not** the NXT Knowledge Import Tool.
</hint>

Follow these steps to import knowledge into \{\{Primary.Company}} \{\{FRS\_VARS.ProductName}}.

1. Specify the article file type to search for, by doing the following:

<Paragraph indent="1">
  a. Click the **Settings** icon <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/xvyCJ-mxbsLkyjf9JOE6k_settings.png" size="10" isUploading="false" initialPath="../../Resources/Images/buttons/settings.png" githubPath="Content/Resources/Images/buttons/settings.png" width="16" height="16" showCaption="false"></Image> in the **Looking for** section. The application displays the **File search filters** dialog box.
</Paragraph>

<Paragraph indent="1">
  b. In the **Add New** drop-down list, select one or more file types:
</Paragraph>

<Paragraph listStyleType="disc" indent="3">
  **Microsoft office documents**: Files that have .doc, .docx, .xls, and .xlsx extensions.
</Paragraph>

<Paragraph listStyleType="disc" indent="3">
  **PDF files**: Files that have a .pdf extension.
</Paragraph>

<Paragraph listStyleType="disc" indent="3">
  **Images**: Files that have .bmp, .jpg, and .gif extensions.
</Paragraph>

<Paragraph indent="1">
  c. Optionally, you can add more file types by clicking **Custom**. The application adds a line item to the filter list called Custom. Click the line item to open it for editing. You can change the filter name and specify one or more extensions. Click **OK**. The application adds the file types that you specified to the **Looking for** section.
</Paragraph>

2. Specify which local or network folders to search for knowledge articles and then perform the search:

<Paragraph indent="1">
  a. Click the **Settings** icon <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/xvyCJ-mxbsLkyjf9JOE6k_settings.png" size="10" isUploading="false" initialPath="../../Resources/Images/buttons/settings.png" githubPath="Content/Resources/Images/buttons/settings.png" width="16" height="16" showCaption="false"></Image> in the **In Folders** section. The application displays the **Import folders** dialog box.
</Paragraph>

<Paragraph indent="1">
  b. Click **Add New**. The application displays the **Browse For Folder navigation** dialog box.
</Paragraph>

<Paragraph indent="1">
  c. Navigate to a folder and highlight it. (To optionally add a new subfolder to the folder, click **Make New Folder**.)
</Paragraph>

<Paragraph indent="1">
  d. Click **OK**. The application adds the folder to the list of folders to search. If you created a new folder, the application adds it to the file application as an empty folder.
</Paragraph>

<Paragraph indent="1">
  e. Optionally, check **Recursive** for a folder to specify that the application should search the subfolders within the folder.
</Paragraph>

<Paragraph indent="1">
  f. Optionally, check **Monitor for changes** to configure the knowledge uploader to monitor all of the folders on the list for changes to the file types that you specified in step 1b.
</Paragraph>

<Paragraph indent="1">
  g. Repeat steps b-f to continue to add folders to the list as necessary.
</Paragraph>

<Paragraph indent="1">
  h. Click **OK**.
</Paragraph>

<Paragraph indent="1">
  The knowledge uploader performs the search and displays all of the files that meet the search criteria.
</Paragraph>

3. Select the files to upload:

<Paragraph indent="1">
  a. If you think the contents of the scanned folders might have changed, click **Rescan Folders** to regenerate the file list.
</Paragraph>

<Paragraph indent="1">
  b. In the file list, check the files to upload, or click **Select All** to upload all of the files.
</Paragraph>

<Paragraph indent="1">
  c. Click **Upload** to launch the **Article Creation** wizard and start the upload process.
</Paragraph>

<Paragraph indent="1">
  The **Article Creation wizard** page contains two main areas: a **Selected Documents** area on the left containing the list of files to import, and a **Document Properties** area on the right that contains two tabs. One tab is called **Default Values** and the other is called **Selected Documents**.
</Paragraph>

4. To remove a document from the **Selected Documents** section, check it and click **Remove selected**. By default, the knowledge uploader imports all of the documents in the **Selected Documents**, even if you do not select them.
5. Optionally, use the **Document Properties** section to specify the properties that the knowledge uploader displays together with the files in \{\{FRS\_VARS.ProductName}}. You can specify properties for all files (using the **Default values** tab) or for individual files (using the **Selected Document** tab after highlighting the file in the **Selected Documents** section).

<Paragraph indent="1">
  You can specify these properties:
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **Article Type**: The article type. Select from Document, Error Message, Issue Resolution, Patch, Q\&A, or Reference.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **Article Status**: The status of the article. Select from Approved, Draft, Expired, Failed Review, Pending Review, Published, or Submitted.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **Target collection**: The target. Select from Company Policy & Procedure, Customer Knowledge, Internal Knowledge, IT Knowledge, or Service Desk.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **Category**: The category for this knowledge article.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **Use file name as Subject**: Check to use the article file name to fill in the **Subject** field.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **Subject**: Only use this field if you did not check **Use file name as Subject**. Enter a subject for the article.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **Resolution**: Enter the resolution if the knowledge article contains one.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  **Description**: Enter a description for the knowledge article.
</Paragraph>

6. Click **Import**. The import process begins.

<Paragraph indent="1">
  If you select a file that has already been imported into the knowledge base, the application displays the **Collision warning** dialog box.
</Paragraph>

7. Depending on what you want to do with the file that has already been imported, click one of the following:
   - **Apply for all**: Applies your choice to all previously imported files that are found during the import process.
   - **Update**: Overwrites the file that already exists in the knowledge base with the file that is being imported.
   - **Create New**: Does not overwrite the file being imported with the file that already exists in the knowledge base. After importing, there are two knowledge articles with the same name in the knowledge base.
   - **Cancel**: Cancels the import process.

<Paragraph indent="1">
  The application displays an Import dialog box with the status of the import.
</Paragraph>

8. Click **OK**.
9. Optionally, do the following:
   - Review the import logs by clicking the **Log** tab. You can clear or save the log.
   - Review the import history by clicking the **Import History** tab. Click a number in the **Imported** column to display more information about an import session.
