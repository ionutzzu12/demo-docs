\{\{FRS\_VARS.Company}} \{\{FRS\_VARS.Neurons\_Bots}} provide powerful hyper-automation capabilities. \{\{FRS\_VARS.Neurons\_Bots}} capability within \{\{FRS\_VARS.Company}} \{\{FRS\_VARS.ITSM}} provides an integration between ITSM and \{\{FRS\_VARS.Company}} \{\{FRS\_VARS.Neurons}}, and enables you to invoke a \{\{FRS\_VARS.Neurons\_Bot}} from ITSM.

<hint type="info">
  \{\{FRS\_VARS.Neurons\_Bots}} can be invoked from a business object that has a relationship (link) to, or can be invoked from, a **CI** business object record. Also note that, if the business object from which a \{\{FRS\_VARS.Neurons\_Bot}} is invoked has a relationship with more than one **CI** record, an error is displayed.
</hint>

Supported OOTB (Out Of The Box) business objects you can invoke a \{\{FRS\_VARS.Neurons\_Bot}} from if linked to a **CI** record are:

- **Incident**.
- **Change**.
- **Problem**.
- **Knowledge**.

## Invoke a Neurons Bot

1. Open the business object record that is linked to the **CI** record you want to invoke a \{\{FRS\_VARS.Neurons\_Bot}} upon.

<hint type="info" indent="2">
  The business object selected to invoke the \{\{FRS\_VARS.Neurons\_Bot}} must only be linked to a single **CI** record; if it is linked to more than one **CI** record, an error is displayed.
</hint>

<Paragraph indent="1">
  Click **More**, and select the required \{\{FRS\_VARS.Neurons\_Bot}} \{\{FRS\_VARS.Quick\_Action}}.
  If the \{\{FRS\_VARS.Neurons\_Bot}} \{\{FRS\_VARS.Quick\_Action}} requires user input, a dialog box opens.
</Paragraph>

3. Enter the required input into the dialog box, and click **OK**. Skip this step if no input is required.
   The \{\{FRS\_VARS.Neurons\_Bot}} is invoked and performs the action.

When the \{\{FRS\_VARS.Neurons\_Bot}} invokes successfully, a \{\{FRS\_VARS.Neurons\_Bots}}**&#x20;Invocation** confirmation is displayed.

## Check a Neurons Bot Invocation and Invocation Results

Once a \{\{FRS\_VARS.Neurons\_Bot}} is invoked, you can check the invocation status and results from the related **CI** business object record.

<ExpandableHeading>
  ## To check invocation status

  The **Neurons Bot Invocations** tab displays a list of bot invocations. Each entry contains status information for a single bot invocation. A single bot invocation may provide multiple results, which are displayed in the **Neurons Bot Results** tab.

  1. Open the related **CI** business object record.
  2. Select the \{\{FRS\_VARS.Neurons\_Bot}}**&#x20;Invocations** tab.
     If this tab is not visible, click the **Restore hidden tabs** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/8qgkd6bNoUEOedgWUlCcL_hiddentabsicon.png" alt caption size="10" width="19" height="20"}, and then select it from the list.
  3. Double-click the invocation record in the list.
     The **Edit&#x20;**\{\{FRS\_VARS.Neurons\_Bot}}**&#x20;Invocation** dialog opens.

  *The Edit Neurons Bot Invocation Dialog*

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/vAFTOTp4NjNSXmUdPC0ki_editneuronsbotinvocationdialog.png" alt="Screenshot of the edit neurons bot invocation dialog" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/Edit_Neurons_Bot_Invocation_Dialog.png" githubPath="Content/Resources/Images/help-images/Edit_Neurons_Bot_Invocation_Dialog.png" showCaption="false"></Image>

  <hint type="info">
    In this version of \{\{FRS\_VARS.ITSM}}, edits are not enabled for this dialog.
  </hint>

  - **Bot Name**: Name of the bot that was invoked.
  - **Completion State**: Current state of invocation, which can be:
    - Pending
    - In Progress
    - Success
    - Failed
    - Timed Out
    - Skipped
    - Unknown
  - **Invoked On**: Date/time the bot was invoked.
  - **Started On**: Date/time the bot started running and processing inputs.
  - **Finished On**: Date/time the bot finished running.
  - **CI Results**: Contains invocation results. There may be multiple results. Double-click results to access the result records directly.
  - **Inputs**: Contains a log of input parameter ID's (in JSON) against the values sent when invoking the bot. This field is empty if no parameter IDs are sent.
  - **Failure Code**: Contains a failure code if the bot fails to run or complete.
  - **Failure Reason**: Contains information on the reason the bot failed to run or complete.
</ExpandableHeading>

## Troubleshooting a Neurons Bot Invocation

If you experience an error and a \{\{FRS\_VARS.Neurons\_Bot}} fails to run when invoked, check the table below for possible reasons for the error, and possible solutions.

<table isTableHeaderOn="true">
  <tr>
    <td>
      Issue
    </td>
    <td>
      Possible Reason(s)
    </td>
    <td>
      Solution
    </td>
  </tr>
  <tr>
    <td>
      **When run, the following Quick Action error is displayed:** **CI** record with \{\{FRS\_VARS.Neurons}} device information must be linked prior to running this \{\{FRS\_VARS.Quick\_Action}}.
    </td>
    <td>
      No **CI** record is linked to the current record, or is not being run directly against a **CI** record.
    </td>
    <td>
      Link a **CI** record that was imported from \{\{FRS\_VARS.Neurons}} to the current record.
    </td>
  </tr>
  <tr>
    <td>
      **When run, the following Quick Action error is displayed:** **CI** record requires \{\{FRS\_VARS.Neurons}} device information prior to running this \{\{FRS\_VARS.Quick\_Action}}
    </td>
    <td>
      The \{\{FRS\_VARS.Quick\_Action}} is being run directly against a **CI** record, but does not have the **Agent ID** field populated.
    </td>
    <td>
      Ensure the \{\{FRS\_VARS.Neurons\_Bots}} device synchronization has been configured to import the **Agent ID**, and it has been run successfully.
      Run directly against a **CI** record with an **Agent ID**.
    </td>
  </tr>
  <tr>
    <td>
      **When run, the following Quick Action error is displayed:** Unable to resolve relationship from \<Business Object A> to \<Business Object B> via reference name.
    </td>
    <td>
      The \{\{FRS\_VARS.Quick\_Action}} is run against an unsupported business object. Business objects that are supported OOTB are:
      - **Incident**.
      - **Change**.
      - **Problem**.
      - **Knowledge**.
    </td>
    <td>
      Only run a \{\{FRS\_VARS.Neurons\_Bot}} against a supported business object.
    </td>
  </tr>
  <tr>
    <td>
      **When run, the following Quick Action error is displayed:** Multiple Agent IDs linked. Cannot determine desired agent to target.
    </td>
    <td>
      More than one **CI** record is linked to the current record.
    </td>
    <td>
      Ensure that there is only a single **CI** record with a \{\{FRS\_VARS.Neurons}} **Agent ID** linked to the current record.
    </td>
  </tr>
  <tr>
    <td>
      **When run, the following Quick Action error is displayed:** The method or operation is not implemented.
    </td>
    <td>
      The Action Service feature flag has not been enabled.
    </td>
    <td>
      Request \{\{FRS\_VARS.Company}} Cloud Operations to enable the Action Service feature flag. Refer to [More Information](../../More-Information.mdx).
    </td>
  </tr>
  <tr>
    <td>
      You created a \{\{FRS\_VARS.Neurons\_Bot}} \{\{FRS\_VARS.Quick\_Action}} in the \{\{FRS\_VARS.Graphical\_Action\_Designer}}, but it is not shown when attempting to run it.
    </td>
    <td>
      The **EnableActionService** global constant has not been enabled.
    </td>
    <td>
      Set the **EnabledActionService** global constant to **True**. Refer to [Working with Global Constants](../Constants/Defining_Global_Constant.mdx).
    </td>
  </tr>
</table>
