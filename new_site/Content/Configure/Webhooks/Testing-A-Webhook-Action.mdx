You can pass JSON or XML sample data to test data mapping between an external tool and the Action *before&#x20;*&#x79;ou save it. This enables you to verify that the Action is valid before you send events from an external tool.

1. Start [Creating a Webhook](Creating-A-Webhook.mdx).
2. After entering information into the fields, click **Test Action**.

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/0hw2PvY9eJb_N8TT7QQef_testingwebhookaction.png" position="flex-start" size="40" indent="1" initialPath="images/testing_webhook_action.png" githubPath="Content/Configure/Webhooks/images/testing_webhook_action.png" showCaption="false"></Image>

3. Paste valid JSON or XML that you expect to be sent from the external tool to the webhook endpoint into the **Webhook Body**.

<hint type="info">
  **Note:** The **Character Count** on the right shows you how close you are to the 100,000 character limit.
</hint>

4. Click **Format Code**.

<Paragraph indent="1">
  If the JSON or XML is not valid or will be too long after formatting, you get information about the error so you can fix it. If the supplied code is valid, it gets formatted for you.
</Paragraph>

5. Select **Save Business Object after action** if you wish to commit your test result to the business object.

<Paragraph indent="1">
  This option defaults to OFF to stop accidental committals.
</Paragraph>

6. Click **Test**.

<Paragraph indent="1">
  A response appears in the **Response&#x20;**&#x73;ection.
</Paragraph>
