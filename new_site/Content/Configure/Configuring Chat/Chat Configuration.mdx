**Role**: Administrators.

Configure the following settings for the chat feature to work:

- Global Constant - In the \{\{FRS\_VARS.Configuration\_console}}, under **Global Constants**, set the **ChatEnabled** global constant Value field to **True**.

## To enable the chat feature for a user role

1. Log in to \{\{FRS\_VARS.ProductName}} as an \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}.
2. From the Configuration Console, select **Configure** > **Users and Permissions** > **Roles and Permissions**.
3. Select the role for which you want to enable the chat feature.
4. Select **Overwrite default branding options with the options listed below**.
5. Select **Enable Chat as Analyst Button**.
6. [Enable chat support for Line of Business (LOB) departments](Enable-chat-support-LOB.mdx) to better direct chat users to an analyst with relevant expertise.

<hint type="info">
  - All check boxes under **Branding Options** are disabled until you select the **Overwrite default branding options with the options listed below** option.
  - Chat requests to the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Analyst-Service-Desk-Analyst}} team are routed using the Round Robin algorithm, based on the Service Desk Analyst's login state.
</hint>

<hint type="info">
  Chat with Support (Analyst) is not currently supported in the new Navigation UI. This capability is available only when using the classic navigation experience.
</hint>
