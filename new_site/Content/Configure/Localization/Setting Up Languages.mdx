### About the Available Languages and Cultures

\{\{FRS\_VARS.ProductName}} provides a full translation for some languages and supports several other languages. Support means that you can add the language to \{\{FRS\_VARS.ProductName}} but you must use the translation tools to provide the translations for your language.

A *culture* comprises the language, number formats, currency symbols, and country specific settings. For \{\{FRS\_VARS.ProductName}}, the words *culture* and *language* are used interchangeably.

The format for the culture name is the two-letter language code in lower case and the two-letter subculture code in upper case. Examples include ja-JP for Japanese (Japan) and en-US for English (United States). In cases where a two-letter language code is not available, a three-letter code derived from ISO 639-2 is used.

### Setting the Default Language for a New User

You can set the default language for a new user. This becomes the default language for the user and displays when the user initially logs in.

Follow these steps to set the default language for a new user:

1. From the Configuration Console, click **Configure > Users and Permissions > Users**. The application displays the **Users** workspace.
2. Open the new user record.
3. Click the **Details** tab.
4. In the **Language** field in the **Organizational Information** section, select a language from the drop-down list.
5. Click **Save**.

<hint type="info">
  If a user logs in to a new machine, the application displays their original default language even if their default language has recently changed.
</hint>

### Deleting a Language and Culture

You can delete a language and culture in \{\{FRS\_VARS.ProductName}}; however, although it appears that the language and culture is removed, there are still a large number of columns associated with that language and culture that remain in the database.

1. From the Configuration Console, click **Configure > Organizational Profile > Languages** to open the **Languages** workspace.
2. To disable a language and culture for all roles, click the **delete** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/USPl9Egu4Y2Blx7HKC3mA_deleteredx.png" alt caption size="10" width="14" height="13"} next to it.
3. In the confirmation message dialog box, click **Yes**.

### Adding a Language and Culture with Translations

Follow these steps to add a language and culture that has translations. See :Link[Supported Languages and Cultures with Translations]{label="Supported Languages and Cultures with Translations" overridedLabel="Supported Languages and Cultures with Translations" spaceId docId="ZOdB7cSrDXVspZWLePaWn" version="v2" docAnchorId="#Supported_Languages_and_Cultures" loadingMethod="dynamic" newTab="false" href="Setting Up Languages.mdx"}.

Only add the languages and cultures that your organization actively uses. Selecting additional languages and cultures expands the database with additional fields that cannot be completely removed even if you disable the language and culture at a later time.

1. From the Configuration console, click **Configure** > **Organizational Profile** > **Languages** to open the **Languages** workspace.
2. Click **Add new\...**. The application displays a list of additional languages and cultures.
3. Select the language and culture to add. The application automatically saves your option and includes it in the **Language** workspace.
4. Return to the **Languages** workspace and click **Refresh** to verify that your selection was added to the list.

<Paragraph indent="1">
  The application automatically checks **Enabled** for the new language and culture, which enables the new language and culture for all roles.
</Paragraph>

### Adding a Language and Culture Without Translations

Many more cultures are supported in \{\{FRS\_VARS.ProductName}} than have been translated. See :Link[Supported Languages and Cultures Without Translations]{label="Supported Languages and Cultures Without Translations" overridedLabel="Supported Languages and Cultures Without Translations" spaceId docId="ZOdB7cSrDXVspZWLePaWn" version="v2" docAnchorId="#Supported_Languages_andCultures_Without_Translations" loadingMethod="dynamic" newTab="false" href="Setting Up Languages.mdx"}. If you choose one of the supported languages and cultures without translations, you must supply the translation.

<hint type="info">
  This export and import process only provides translations for metadata. It does not provide translations for resource files. Also, this process does not include validation list data, roles, desktops, and dashboard charts.
</hint>

1. From the Configuration Console, click **Configure > Organizational Profile > Languages** to open the **Languages** workspace. The **Languages** workspace appears with a list of default languages (culture names).
2. Click **Add new\...**. The application displays the list of additional languages and cultures.
3. Select the language and culture to add. The application adds the new language and culture.

<Paragraph indent="1">
  *Adding a New Language and Culture*
</Paragraph>

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/3I2NOmUZZ5ZnQxj8ih1Le_addingnewculture.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/adding_new_culture.png" githubPath="Content/Resources/Images/help-images/adding_new_culture.png" showCaption="false"></Image>

<Paragraph indent="1">
  This process takes a few minutes. When the application finishes adding the language and culture, it displays a **Success** dialog box.
</Paragraph>

4. Click **OK**.
5. Return to the **Languages** workspace and under **Export All**, click **Export**.

<Paragraph indent="1">
  The application creates a spreadsheet. Depending on your browser configuration, the application saves the spreadsheet in the default download folder or in a folder that you specify.
</Paragraph>

<Paragraph indent="1">
  The spreadsheet file name includes the name of the target language. For example, the name might be Translation (All) of Greek (Greece),xlsx
</Paragraph>

<Paragraph indent="1">
  *Example of a Translation Spreadsheet.*
</Paragraph>

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/LqrWVuyHzyhTf2JLOH3lX_exporttranslationspreadsheet.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/export_translation_spreadsheet.png" githubPath="Content/Resources/Images/help-images/export_translation_spreadsheet.png" showCaption="false"></Image>

6. Deliver this spreadsheet to your translator.

<Paragraph indent="1">
  The translator translates the text strings into the target language and then returns the spreadsheet to you.
</Paragraph>

7. Go to the **Languages** workspace, and beside the language and culture that you added, click **Import** to import the spreadsheet with the translated values.

<Paragraph indent="1">
  This process takes a few minutes. When the application is done importing the spreadsheet, it displays a **Success** dialog box.
</Paragraph>
