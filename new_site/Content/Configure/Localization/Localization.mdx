## About Localization

You can localize many areas of \{\{FRS\_VARS.ProductName}} to serve international customers.

Localization provides the capability to display the user interface in a language-and culture-specific way. Support is provided for non-English languages as well as for different cultures within a language, such as French (France) and French (Canada). The localization tools allow you to set the language for your locale.

<hint type="info">
  The terms *culture* and *locale* are used interchangeably.
</hint>

Localization offers the following:

- Self Service and Service Desk users can select the locale of their choice, and administrators can configure the user interface for each locale. See [Setting Up Languages](<Setting Up Languages.mdx>).
- Service Desk Analysts and Service Desk Managers with permissions to edit quick actions can edit the Self Service templates in the supported languages, as described in [Translating the Self Service Action Templates](<../QuickActions/Classical Action Designer/Translate the Self Service QA Templates.mdx>).
- Administrators can translate forms, lists, layouts, saved searches, and quick actions, which users automatically access when choosing their locale. See [Setting Up Languages](<Setting Up Languages.mdx>).
- Administrators can set up the locales and translation strings. See :Link[Using the Ivanti Translation Tool]{label="Using the Ivanti Translation Tool" overridedLabel="Using the Ivanti Translation Tool" spaceId docId="_EuTeCZuBV-NNqINHqp4H" version="v2" docAnchorId="#To_Use_the_Translation_tool" loadingMethod="dynamic" newTab="false" githubPath="Using the Translation Tool.htm#To_Use_the_Translation_tool" href="Using the Translation Tool.mdx"}.
- Administrators can translate label expressions. See :Link[Example: Translating Label Expressions]{label="Example: Translating Label Expressions" overridedLabel="Example: Translating Label Expressions" spaceId docId="ltWDyd6Ziziwqpkx_xlua" version="v2" docAnchorId="#Example-Translating-Label-Expressions" loadingMethod="dynamic" newTab="false" githubPath="Translate_Label_Expressions.htm#Example-Translating-Label-Expressions" href="Translate_Label_Expressions.mdx"}.
- Administrators can determine who made translation changes and when they were made. See [Using the Ivanti Translation Tool](<Using the Translation Tool.mdx>).
- Prompts used in quick actions can be translated. See [Translating Quick Actions and Their Prompts](<Translate Prompts for Quick Actions.mdx>).
- Results from expressions can be translated. See [Expression Grammar](<../../Reference/Expressions/Expression Grammar.mdx>).

## Unsupported Localization Features

- The text strings used for English locales are read-only.
- You cannot configure the user interface to use more than one currency symbol at a time.
- Transactional data (such as that displayed in a list view for a new incident, problem, approval, and so on) is not translated.
