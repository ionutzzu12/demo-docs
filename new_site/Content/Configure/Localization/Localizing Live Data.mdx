Use this method to localize live content immediately.

<hint type="info">
  To use AI-powered **Auto-translation** feature that performs automatic translation of content, refer [Auto-Translation](../../AITSM/Auto-Translation.mdx) chapter.
</hint>

## Updating Live Data

1. Locate the database of language files.
   1. Log in as an \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}} and navigate to the **Configuration Console**.
   2. From the **Settings** menu, select **Organization Profile** > **Languages**.
   3. The **Languages** workspace displays a table of language files.
2. Click **Edit** in the **Edit Live Data** column.

<ExpandableHeading>
  ## Content Strings

  1. Select **Content Strings** to load the **Select criteria by which you would like to load the data** menu.
  2. Choose **Business Objects** or **Business Object Properties**.
     - **Business Objects** loads and presents data according to the name of the business object you want to change. For example, you can translate data for all Address business objects.
     - **Business Object Properties** loads and presents data according to the business object property you want to change. For example, you can translate data for all business objects that are **Forms**.
  3. For either selection:
     1. Select the item you want to translate, and click **Next**.
     2. The **Localization Workbench & Settings** workspace populates with live data for translation. Make your changes to translations from this interface.
        - Use the column headers to filter the data in order to find the label you want to translate in the **Source** column.
        - Make the desired changes to the **Target** property by clicking the field. The **Translation Status** property immediately changes to **Modified**.

<ExpandableHeading>
  ## Application Strings

  In \{\{FRS\_VARS.Self\_Service\_Portal-Self-Service}} UI v3, you can use the default system file to create a custom file and update the language entries for any existing language, or you want a language that exists in Neurons for ITSM application and is not translated. Use the system file from any existing language, add the required translations, and upload your custom file to enable that language without Ivanti Support.

  <hint type="info">
    You cannot edit the existing system file directly. Instead, download it, make the required changes, and upload it as a custom file in the application.
  </hint>

  **Prerequisite**

  - Custom files must be saved using the .langjson extension.
  - To add new extension, go to **Configuration console** > **Security Controls** > **Allowed File Extensions** and then click **Add New** and then add .langjson to the list.

  To translate into a new language or modify an existing language:

  1. Select **Application Strings** from the dropdown and click **Next**.

  <Paragraph indent="1">
    The **Localization Workbench & Settings** workspace opens, and displays **ID** and **translation** columns.
  </Paragraph>

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/YwfaaRYy3oNBRI4QcyLLJ_application-strings.png" alt="Click Edit under the Edit Live Data column to open the Localization Workbench workspace" position="flex-start" size="40" indent="2" initialPath="../../Resources/Images/loc-Images/application-strings.png" githubPath="Content/Resources/Images/loc-Images/application-strings.png" showCaption="false"></Image>

  2. In the workspace, select the required language from and click **Export JSON** dropdown to download system file.
  3. For a new language, add translations for all fields in file and to modify existing language, update the required entries.
  4. To upload the modified file, click **Custom files** and then clic&#x6B;**&#x20;Import JSON**.

  <Paragraph indent="1">
    The **Paste JSON** window opens.
  </Paragraph>

  5. Paste the modified details in the window and click Validate & Save.

  <Paragraph indent="1">
    The confirmation window appears and you can view the translation column with modified details.
  </Paragraph>

  <Paragraph indent="1">
    The custom file is now added as a supported language for the Self Service role.
  </Paragraph>
</ExpandableHeading>

## Updating Multiple Translation Records

You can simultaneously select and update multiple translation records with the same translation value by filtering those with identical source values.

To update mutiple translation records:

1. From the Configuration Console, click **Configure** > **Organizational Profile** >**&#x20;Languages**to display the **Languages** workspace.
2. Click **Edit** in the **Edit Live Data** column.
3. In the **Localization Workbench & Settings** workspace, choose between **Content Strings**and **Picklist Strings**; Select the item you want to translate, and click **Next**.
4. Click the filter icon in the **Source&#x20;**&#x63;olumn header to select the search string. The application displays only the translation records that match your search string.
5. Modify the translation value in th&#x65;**&#x20;Target&#x20;**&#x63;olumn for the object.
6. A confirmation dialog box appears. Click **Yes**.

<Paragraph indent="1">
  The application updates the corresponding Translation records with the modified string value.
</Paragraph>

## Grouping Multiple Translation Records

You can group and update multiple translation records by filtering the translation records that have the same source value.

To group multiple translation records:

1. From the Configuration Console, click **Configure** > **Organizational Profile** >**&#x20;Languages**to display the **Languages** workspace.
2. Click **Edit** in the **Edit Live Data** column.
3. In the **Localization Workbench & Settings** workspace, choose between **Content Strings** and **Picklist Strings**.
   - Select the item you want to translate, and click **Next**.
4. Click **Filter Options** drop-down list and select **Group by Duplicates Source** option. The application displays only the translation records that are grouped together based on the same source value.
5. Modify the translation value in th&#x65;**&#x20;Target&#x20;**&#x63;olumn for the object.
6. If there are multiple records that match the source string, a confirmation dialog box appears. Click **Yes**.

<Paragraph indent="1">
  The application updates the corresponding Translation records with the modified string value.
</Paragraph>
