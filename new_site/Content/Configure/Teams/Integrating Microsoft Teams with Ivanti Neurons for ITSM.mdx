**Role**: Administrator.

**Minimum Version**: \{\{FRS\_VARS.ProductName}} 2021.1

\{\{FRS\_VARS.ProductName}} seamlessly integrates with Microsoft Teams Bot to create a custom Teams app. This enables Self Service users to chat with Service Desk Analysts, create incidents, submit customized service requests, and check the status of their tickets from the Teams app. This enhanced integration improves user experience by streamlining ITSM interactions directly within Microsoft Teams.

<hint type="info">
  For environments upgraded to version 2026.2, Ivanti recommends administrators to remove the existing bot configuration and create a new bot using the updated UI. Create the new bot by following the steps described in the sections below.
</hint>

## Integration Process

<ExpandableHeading>
  ## Prerequisites

  - Enable the Global Constant from the Configuration console > **Global Constants** > **MSTeamsIntegrationEnabled** > set it to **True**. By default, this option is set to **False**.
  - Enable the \{\{Primary.Product}} Chat feature. Refer to [Configuring the Chat Feature](<../Configuring Chat/Chat Configuration.mdx>).
  - Ensure Microsoft Teams application is installed on your computer with Admin credentials.
  - For on‑premises deployments, ensure that Secure Sockets Layer (SSL) is enabled. SSL is mandatory for the integration to function correctly and does not apply to cloud environments.

  <hint type="info">
    It is recommended to set up the chat application first, followed by the Teams Bot application.
  </hint>
</ExpandableHeading>

  ## Configuring Custom Questionnaire in Teams Bot for Incident Creation

  You can configure custom questions in the Teams Bot before initiating an incident creation. The supported fields for customization include **Summary**, **Description**, **Impact**, and **Urgency**. Additionally, you can configure any text-based incident field based on your specific requirements. This flexibility allows you to customize the Teams Bot interaction to gather relevant information for a more detailed incident creation process.

  <hint type="info">
    - You cannot configure fields with duplicate display names.
    - In some cases, the Teams Bot may not respond as expected due to a missing central-config-api-key entry in the Teams Bot **appsettings.json** file. This setting is part of the internal configuration and is not included in the standard documentation. If you've followed all setup steps and are still experiencing issues, please reach out to the **Ivanti Support Team** for assistance. Configuration changes should only be made under the guidance of Support to ensure system integrity.
  </hint>

  To configure the custom questionnaire:

  1. Open Configuration console, and click **Microsoft Teams Integration** under **Extend**.

  ## Enabling feedback flow for Microsoft Teams

  You can manage the feedback flow in Microsoft Teams. By default, the feedback card is turned off. You can enable or disable it in the Admin UI.

  Go through the following steps to configure this setting.

  1. Log in to \{\{FRS\_VARS.ProductName}} as an Administrator.
  2. In the Configuration console, **Build** > click on **Microsoft Teams Integration**.
  3. Open **Session Configuration Settings**.
  4. Locate **Enable Feedback Card**. By default, this option is disabled.
  5. To enable feedback flow, check the box and clear the cache.

  <Paragraph indent="1">

  ## Enabling Service Request for Microsoft Teams

  Request offerings published in ITSM are configurable in the MS Teams bot. Navigate to the Request Offering workspace and check the **Enable for Teams Bot&#x20;**&#x6F;ption. No additional configuration is required, providing a seamless experience.

  1. Log in to \{\{FRS\_VARS.ProductName}} as an Administrator.
  2. Select **More** > **Request Offering**. Select an offering from the list.
  3. In the **Define Request Offering** tab, select the **Enable for Teams Bot&#x20;**&#x63;heckbox to enable this in the Teams bot.
  4. Only request offerings with the bell icon have the **Enable for Teams Bot** checkbox available.
  5. Click **Save**.

  ### Limitations and Tips for Using the Service Request Feature in MS Teams Bot

  To get the best out of the service request feature in the MS Teams bot, here are some key points to keep in mind:

  1. **Form Design Limitations**:
     Advanced features like aligning rows or adding custom masking to text or numeric fields, are not supported by the bot.
  2. **Pick List Restrictions**:
     Certain fields in pick lists, such as pricing or descriptions, are not supported in the bot. Also, **Select all** and **MultiSelect&#x20;**&#x6F;ptions are unavailable in the bot.
  3. **Expression Limits**:
     Complex formulas or expressions that directly reference other fields or objects are not supported in the bot. Functions, like “!” are not supported by the bot.
  4. **File Uploads**:
     You can upload one file per upload field in a Service Request template. Supported formats are .png, .jpg, .pdf, .jpeg, or .doc.
  5. **Dropdowns**:
     Default values in drop-downs are not supported by the bot. All options are listed any preselection.
  6. **Validation Rules**:
     Complicated rules, especially those that rely on matching field names and values, aren’t supported by the bot.

  <hint type="info">
    Ivanti recommends usage of supported features. Ensure to test the Service Request templates before deployment.
  </hint>
