A prompt is a display window that requires users to provide information before proceeding with a process or action (example: Specify the reason for putting an Incident into Pending status). A stored prompt can be used multiple times within \{\{Primary.Company}} \{\{FRS\_VARS.ProductName}} anywhere that a Quick Action or an expression is used that involves user interaction.

Use a prompt to provide information, such as:

- Name or ID of an Incident owner.
- Start and end dates to search for Change Requests within a date range.
- Reasons for escalating an Incident.

Stored prompt data types are:

- Text
- Number
- Date and Time
- Boolean
- Table
- Pick List

## Stored Prompt Use Examples

You can use stored prompts for any number of reasons, such as populating email addresses, formatting forms and pages, or standardizing a company's branding across the platform.

**Stored prompts can be used:**

- In searches in comparison clauses to prompt users to select a value to search.
- In Quick Actions to prompt users for more information before proceeding with the action.
- In expressions to prompt users to specify a value as part of an expression.
