This function uses the whole interval method to calculate the difference in minutes. If you need to calculate the number of minutes using the boundary crossing method, use the [DateDiffSQL](DateDiffSQL.mdx) function instead.

See [Date Functions](<_Date Functions.mdx>) for more information about using the whole interval method versus using the boundary crossing method.

## Syntax

DiffMinutes(startDateTime, endDateTime, timeZone)

## Parameters

| Parameter     | Description                                                                                                                                                                                                                                                                                                                       |
| ------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| startDateTime | The start date and time to be compared.                                                                                                                                                                                                                                                                                           |
| endDateTime   | The end date and time to be compared.                                                                                                                                                                                                                                                                                             |
| timeZone      | (Optional) The time zone, in IANA format.<br />**NOTE**: This parameter is optional.  If you specify a time zone, the application assigns that time zone to both the **startDateTime** and **endDateTime** parameters. If you do not specify a time zone, the application uses the time zone associated with the current session. |

## Returns Value

Number value.

If the value of th&#x65;**&#x20;startDateTime** parameter is earlier than the value of th&#x65;**&#x20;endDateTime** parameter, the returned value is positive. If the value of th&#x65;**&#x20;startDateTime** parameter is later than the value of the **endDateTime** parameter, the returned value is negative.
