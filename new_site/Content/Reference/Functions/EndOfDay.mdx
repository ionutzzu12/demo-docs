Returns the UTC date and time for the end of the local day based on the local date and time of the current session.

## Syntax

EndOfDay(numDays, baseDateTime, timeZone)

## Parameters

| Parameter    | Description                                                                                                                                                                                                          |
| ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| numDays      | (Optional) The number of intervals. If not specified, uses the default interval which is 0. Can be either a positive integer, to indicate days in the future, or a negative integer, to indicate days in the past.   |
| baseDateTime | (Optional) The base date for the function. If not specified, uses the default date which is the current date and time.                                                                                               |
| timeZone     | (Optional) The time zone, in IANA format.                                                                                                                                                                            |

## Return Value

<Paragraph indent="1">
  DateTime value in UTC time.
</Paragraph>

<hint type="info">
  - If the application has to display the return value, it implicitly converts it to text, thereby allowing the display format to change depending on your culture code. See :Link[Available Languages and Cultures]{label="Available Languages and Cultures" overridedLabel="Available Languages and Cultures" spaceId docId="ZOdB7cSrDXVspZWLePaWn" version="v2" docAnchorId="#culture" loadingMethod="dynamic" newTab="false" githubPath="../../Configure/Localization/Setting Up Languages.htm#culture" href="../../Configure/Localization/Setting Up Languages.mdx"} for a list of the culture codes used by \{\{FRS\_VARS.ProductName}}.
  - In this release of \{\{FRS\_VARS.ProductName}}, there is a known issue, where if the time zone offset between the current date and time and the DateTime value returned by the function is different, the return value is off by one hour. This happens when the current date is before daylight savings time and the DateTime value that is returned is after daylight savings time, or vice versa.
</hint>

## Example

$(EndOfDay())

<Paragraph indent="1">
  If the current date is January 15, 2014 and your time zone is Pacific time (which is UTC-7 during daylight savings time but UTC-8 during standard time), this function returns the last second of the day, which is January 15, 2014 at 11:59:59.9999 PM converted to UTC time, which is January 16, 2014 at 07:59:59.9999 AM.
</Paragraph>
