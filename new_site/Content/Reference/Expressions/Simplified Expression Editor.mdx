The **Simplified Expression Editor** has all the same basic features as the traditional **Expression Editor**. Refer to [About Expressions](Expressions.mdx) and [Using the Expression Editor](<Using Expression Editor.mdx>) to understand the use and importance of expressions, stored expressions, and the **Expression Editor**.

The **Simplified Expression Editor** has a codeless interface with auto-population of parameters and other features, allowing you to simplify expressions and troubleshoot without leaving the editor.

## Enable the Simplified Expression Editor

**To enable the&#x20;****Simplified Expression Editor****:**

1. Contact the \{\{Primary.Product}} [support team](https://www.ivanti.com/support) to turn on the **NewExpressionEditor** feature flag to enable the **Simplified Expression Editor** for your account.
2. Set the Value as True of the *EnableSimplifiedExpressionEditor* global constant to enable it.

### Contextual Help

The **Simplified Expression Editor** uses the **Lower Messaging Area** to display helpful information. In addition to messages about syntax, it provides function descriptions and parameter details for functions added from the Tools pane.

### Autopopulation of Mandatory Parameters

When you add a function to an expression, the mandatory parameter names are added to your expression automatically. For example, if you select the **ChildFold&#x20;**&#x66;unction, the parameter&#x73;**&#x20;parentObjectRef, recId, childRelationshipRef, childExpression,** and **aggregatorFunction&#x20;**&#x61;re all added for you as per the image below.

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/r0F6QnXZHMjOp2SEiHs54_simplified-ee-mandatory-params.png" position="flex-start" size="40" initialPath="../../Resources/Images/help-images/simplified-ee-mandatory-params.png" githubPath="Content/Resources/Images/help-images/simplified-ee-mandatory-params.png" showCaption="false"></Image>

The **Lower Messaging Area** is populated with the mandatory parameter names, a function description, parameters and their types. Mandatory parameters display a red asterisk (\*) to the right of the parameter name.

### Dropdown menu of Parameter Functions

Some functions have a dropdown menu where you can choose from available function parameters.

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/vft-p2JV_IXu9bT5V3sNF_simplified-ee-parameter-dropdown.png" alt="Dropdown menu from the parameter inside the expression editor." position="flex-start" size="40" initialPath="../../Resources/Images/help-images/simplified-ee-parameter-dropdown.png" githubPath="Content/Resources/Images/help-images/simplified-ee-parameter-dropdown.png" showCaption="false"></Image>

The functions with dropdown menus for parameters are as follows:

| **Function**          | **Parameter**   |
| --------------------- | --------------- |
| ChildFold             | parentObjectRef |
| HierarchicalChildFold | parentObjectRef |
| ExecuteSavedSearch    | BORef\|BOName   |
| ForEachChild          | parentObjectRef |
| EvalField             | objectRef       |
| IsTrueForAllChildren  | parentObjectRef |

The dropdown menus contain all the business object names in your system. Once selected, the parameter is replaced as ObjectName#, for example, Incident#.

### Using Code Snippets

The **Simplified Expression Editor** contains **Code Snippets** in the **Tools&#x20;**&#x70;ane. To add these to an expression, you can either double-click the **Code Snippet** or drag it in to the **Expression Editor** workspace.
