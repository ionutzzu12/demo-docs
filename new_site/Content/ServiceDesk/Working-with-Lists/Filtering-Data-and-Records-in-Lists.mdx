Filtering data allows you to display records that meet your filter criteria and hide the records that you do not want to view.

<hint type="info">
  When a column is filtered, a green filter icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/fk7xt7e1Zw8j9Xzdlp2vJ_greenfiltericon.png" alt caption size="10" width="10" height="7"} is displayed on the column header, indicating that the data displayed in the column is based on some filter criteria. Click the filter icon to see the applied filters or to clear the filter criteria.
</hint>

## Filtering List Records for a Non-Validated List

A non-validated list is a list that is not prepopulated. For example, a column that holds text such as the **Summary** field or the **Customer** field is a non-validated list because you can enter any value, and do not select from a list of predefined items.

1. Log in to \{\{Primary.Product}}.
2. Open a workspace. In this example, open the **Incident** workspace.
3. Hover over the **Summary** column header and click the filter icon:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/QG3yTJ6W4hdCTFZqENvvI_filter.png" alt caption size="10" width="21" height="15"}.
4. Hover over **Filters**. The application displays a text field.

<hint type="info">
  By default, the application does not filter any of the records until you enter a value.
</hint>

5. Enter the text in the filter box to filter the data by that text string. In this example, enter "email" in the filter text box.

<Paragraph indent="1">
  The application compares the text against the contents of the corresponding fields for each record in the list and displays matching records.
</Paragraph>

6. To clear the filter, delete the text in the filter text box.

## Filtering List Records for a Validated List

A validated list is a list that is prepopulated. For example, the **Priority** column holds priority values of 1, 2, 3, 4, 5, and 6. You can only select a priority from the list and you cannot enter a different value.

1. Log in to \{\{Primary.Product}}.
2. Open a workspace. In this example, open the **Incident** workspace.
3. Hover over the **Status** column header and click the filter icon:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/QG3yTJ6W4hdCTFZqENvvI_filter.png" alt caption size="10" width="21" height="15"}.
4. Hover over **Filters**. A list of options is displayed.

<hint type="info">
  If there are less than 500 values, the application displays a checkbox for each value. If there are more than 500 values, you must manually enter the values to select. By default, the application does not filter any of the records until you select an item or enter a value.
</hint>

5. Select the checkbox against the values to filter. In this example, select the **Resolved&#x20;**&#x63;heck box.

<Paragraph indent="1">
  The application displays all the records of status **Resolved**.
</Paragraph>

6. To clear the filter, clear the checkbox selected.

## Filtering List Records by Date

You can show only those items that were created or modified on a certain date or within a date range. This feature works only on date fields.

1. Log in to \{\{Primary.Product}}.
2. Open a workspace. In this example, open the **Incident** workspace.
3. Hover over the **Created On** column header and click the filter icon:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/QG3yTJ6W4hdCTFZqENvvI_filter.png" alt caption size="10" width="21" height="15"}.
4. Hover over **Filters** and the date options appear.

<Paragraph indent="1">
  *Date Options*
</Paragraph>

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/Txe16yEdGoyi9csZQiTcQ_dateoptionsfilter.png" alt="Filtering by date options" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/Common/DateOptionsFilter.png" githubPath="Content/Resources/Images/Common/DateOptionsFilter.png" showCaption="false"></Image>

5. Click the **calendar** icon next to the appropriate filter criteria.
   - To find records matching an exact date, use the equals (=) box.
   - To find records after a specific date, use the greater than (>) box.
   - To find records before a specific date, use the less than (\<) box.
   - To find records within a date range, enter a start date in the greater than (>) box and an end date in the less than (\<) box.

<Paragraph indent="1">
  The application displays the records based on the defined date filter.
</Paragraph>

6. To remove the date filter, click **Reset** at the bottom of the filter box.

## Resetting Filtered List

You can reset the filter values to view the unfiltered records using the **Clear Grid Column Filters** option. Follow these steps to reset the filter values.

1. Log in to \{\{Primary.Product}}.
2. Open a workspace.
3. Click **Clear Grid Column Filters** to list all applied filters.

<Paragraph indent="1">
  *Clear Grid Column Filters*
</Paragraph>

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/OI7hLfRpCaJ6_U5lq7-0R_clear-grid-column-filters.png" alt="Clearing Grid Column Filters" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/Common/Clear-Grid-Column-Filters.png" githubPath="Content/Resources/Images/Common/Clear-Grid-Column-Filters.png" showCaption="false"></Image>

<hint type="info">
  The **Clear Grid Column Filters** button is disabled if no filters are applied.
</hint>

4. To reset an individual filter, clear the selected checkbox from the list of applied filters.
5. Select **Clear All** to reset the filter.

<Paragraph indent="1">
  The application reset all saved filters and displays all records without any filter.
</Paragraph>
