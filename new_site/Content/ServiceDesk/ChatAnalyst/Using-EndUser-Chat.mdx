If your administrator has assigned you as the chat analyst, you can pick up support chat requests to answer the \{\{FRS\_VARS.Self\_Service\_Mobile\_portal}} users' queries or solve their issues without them having to raise an Incident.

In case the chat icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/akem_x-3xXsKQITqsKaNX_chatgrayicon.png" alt caption size="10" width="16" height="16"} is not available for your user role, then you are not assigned the chat analyst role. For further assistance, reach out to your organization's administrator.

The number of chat requests you are assigned at a time is setup by your administrator. A minimum of 1 and maximum of 20 chat requests can be assigned to you at a time. However, only when you accept the chat request the session starts.

<ExpandableHeading>
  ## To chat with the Self Service Mobile user

  1. Log in to \{\{FRS\_VARS.ProductName}}.
  2. Click the chat icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/akem_x-3xXsKQITqsKaNX_chatgrayicon.png" alt caption size="10" width="16" height="16"} > **Turn On**.
  3. Confirm by clicking **Turn On** in the prompt message.

  <Paragraph indent="1">
    The chat icon turns green :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/Vv9pvdgGmkjyYDZ25qkrP_chatgreenicon.png" alt caption size="10" width="16" height="16"} indicating you are **Online** and available to take up chat requests.
  </Paragraph>

  <hint type="info">
    Unless your chat status is **Online**, you will not be assigned to any chat request.
  </hint>

  4. The chat icon turns red indicating you have a chat request assigned to you. Click the chat icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/6DhEvfbIebxNn5uQXh4Xf_chatredicon.png" alt caption size="10" width="16" height="16"}.

  <Paragraph indent="1">
    The name and department of the user requesting the chat is displayed.
  </Paragraph>

  5. Click **Start Chat** to accept and start chatting.
  6. Respond by typing a message in the text box and click the **Send** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/YYt9x-4HyBUVqeSwtGbl3_chatsendicon.png" alt caption size="10" width="16" height="16"}.

  <hint type="info">
    If the chat icon turns red again, it indicates a new chat request. You can accept and chat with multiple users at a time. The number of chats you can be assigned on at a time is configured by your administrator.
  </hint>

  7. Click the **Create an Incident Now** link in the chat window to create an Incident if you are unable to resolve the issue.
  8. The username and other fields are auto-filled. Modify as need be and click **Ok**.

  <Paragraph indent="1">
    An Incident will be created, and the Incident ID will be displayed on the chat window.
  </Paragraph>

  9. Click the **Incident ID** link from your chat window to open the Incident.

  If using a \{\{FRS\_VARS.unified\_product\_experience}} via \{\{FRS\_VARS.Neurons\_platform}}, the chat icons look like this:

  <hint type="success">
    This feature is **Early Access** only and may not be available in your environment. For more information, please contact your Customer Success Manager.
  </hint>



  From left to right the icons mean:

  - **Error**
  - **Not connected**
  - **Connected**
  - **Connected with 2 messages waiting**
</ExpandableHeading>

<ExpandableHeading>
  ## To skip a chat that is patched to you

  - Do not click on the chat icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/6DhEvfbIebxNn5uQXh4Xf_chatredicon.png" alt caption size="10" width="16" height="16"} if do not want to accept a chat for some reason.

  <Paragraph indent="1">
    The chat will be assigned to someone else while your status will still be **Online**.
  </Paragraph>

  <hint type="info">
    However, if you do not accept chat requests subsequently for the number of times set as a limit by your administrator, your status will automatically turn **Offline**. Reach out to your administrator, to know the limit.
  </hint>
</ExpandableHeading>

**Additional actions you can perform from the chat window**

- :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/cXAErPKHnUYEh_qqkHNga_chat-dropdown-arrow-icon.png" alt caption size="10" width="14" height="9"} Drop-down arrow - displays the following options:
  - :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/6pqCez3jl4-rdJIanFd3u_chat-view-history-icon.png" alt caption size="10" width="16" height="13"} **Chat History** - displays the chat history from sessions that has Incident created. How many previous sessions' history should be stored is configured by your administrator.
  - :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/joPzA_N1ru-7ppJMh6fUw_chat-full-screen-icon.png" alt caption size="10" width="16" height="16"} **Full screen** - displays the chat window in full screen.
  - :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/8KD9TI68bfKzaft0wqQWb_chat-increase-font-icon.png" alt caption size="10" width="16" height="12"} **Increase text Size** - increases the font size.
  - :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/J3ZTxxFwXdY4HEGjMZUXU_chat-view-profile-icon.png" alt caption size="10" width="16" height="16"} **View Profile** - displays the profile of the current user in the chat.
- :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/-FOX87NlHRVU2wRK3B07g_chat-copy-icon.png" alt caption size="10" width="16" height="16"} **Copy** - copies the chat text.
- :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/3JOp2_vtBehHnS7Be8nmF_chat-minimize-icon.png" alt caption size="10" width="16" height="16"} **Minimize** - minimized the chat window.
- :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/i8oDYppZ4BxFLfP0ea6vG_chat-close-icon.png" alt caption size="10" width="16" height="16"} **Close** - closes the chat window.

<hint type="info">
  * If an Incident was created in the previous chat session, the Incident ID is displayed in the current chat window.
  * You cannot close an active chat window unless the \{\{FRS\_VARS.Self\_Service\_Mobile\_portal}} user closes it.
</hint>
