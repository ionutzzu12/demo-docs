### About Read and Unread Flags

<hint type="info">
  The read and unread flags are only available for incidents, service requests, problems, changes, releases, and projects.
</hint>

Read and unread flags are available beginning with \{\{FRS\_VARS.ProductName}} Release 2016.2. The application only displays the flags for notes (journal entries) or emails. You must be the owner of a record to see the flags and associated toolbar button. Self Service users do not see the read and unread flags.

When viewing a list of records, the application displays a bar next to a record that only the record owner can see. The bar alerts you that there is a new note or incoming email. The application displays a toolbar button that lets you set the flag as needed.

Changing the ownership of the record does not affect the flag status for existing notes or emails that are unread (or were marked as unread) by the previous owner. They also show as unread for the new owner of the record.

### Using the Read and Unread Flags

1. Log in to the application in any role except Self Service.
2. Open any business object that contains a record that you own.

<hint type="info" indent="1">
  The read and unread flags are only available for incidents, service requests, problems, changes, releases, and projects.
</hint>

3. To determine which records have unread notes or emails, view the record in a list. The application displays an indicator (typically a blue bar, but you can change this; see :Link[Example: Using Flex Settings to Implement a Visual Indicator]{label="Example: Using Flex Settings to Implement a Visual Indicator" overridedLabel="Example: Using Flex Settings to Implement a Visual Indicator" spaceId docId="L-udUBqA5thleQP_Q_9NU" version="v2" docAnchorId="#Example-Using-Flex-Settings-to-Implement-a-Visual-Indicator" loadingMethod="dynamic" newTab="false" githubPath="../../Configure/Forms/Add Related Object Lists to Forms.htm#Example-Using-Flex-Settings-to-Implement-a-Visual-Indicator" href="../../Configure/Forms/Add Related Object Lists to Forms.mdx"}) on the left of any record that has an unread note or email.

4. To determine which notes or emails for a record are unread, open the record and click the **Details** tab. 6. To only mark a single note or email as read, do the following:

<Paragraph indent="1">
  a. Click the <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/WTLKbMa5PI7qeazzbj0FD_edit-old.png" size="10" isUploading="false" initialPath="../../Resources/Images/buttons/edit-old.png" githubPath="Content/Resources/Images/buttons/edit-old.png" width="16" height="16" showCaption="false"></Image>**edit** icon on the note or email to open it. </Paragraph>

<Paragraph indent="1">
  b. Click **Mark As Read** to mark the note or email as read, then save it.
</Paragraph>

<Paragraph indent="1">
  c. Click **Refresh** to refresh the page.
</Paragraph>

7. To mark a single note or email as unread, for example if it needs special attention or follow-up, do the following:

<Paragraph indent="1">
  a. The application displays the **Mark as Unread** button.
</Paragraph>

<Paragraph indent="1">
  b. Click **Mark as Unread** to mark the note or email as unread, then save it.
</Paragraph>

<Paragraph indent="1">
  c. Click **Refresh** to refresh the page.
</Paragraph>
