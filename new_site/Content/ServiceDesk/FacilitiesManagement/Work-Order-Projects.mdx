A Work Order Project is a job that consists of several tasks/jobs and the time span for completion takes several days to months. The Work Order Projects workspace helps in tackling the several tasks that needs to be completed to progress the project. It helps you to understand if the project is progressing as scheduled, indicates any delays or diversions, know the completion status, and take necessary actions to complete the project.

## To create a Work Order Project

1. Open the **Work Order Projects** workspace.
2. Enter information into the fields:
   - **Project Name** and **Summary**.
   - **Owner**: Project owner who is responsible for completing the project.
   - **Start Date** an&#x64;**&#x20;End Date**.
   - **Status**: Set as **Active** once the project starts. Use **On-hold**, **Canceled**, and **Closed** when the project reaches that state.
   - **Schedule Status**, **Budget Status**, and **Project Health**: Set to **On Track** by default. Use **Off Track** and **At Risk** to indicate the status as and when necessary.
3. **Phase**: Add different phases to the project which helps in breaking up the project into smaller measurable chunks of work.
   - Click **New Project Phase** to add a phase to the project.
   - You can even link to an existing phase by clicking **Link** and selecting the phase.
4. **Schedule**: A calendar view of the tasks added to the project. It gives a clear picture in a calendar view as to what the status of each task is. This helps in tracking and measuring tasks' status by percentage completed.
5. **Status Updates**: Notes can be added for additional clarity on the project status in this tab.
6. **Tasks**: All tasks required for project completion are added in this tab.
7. **Risks**: Any risk that is foreseen for project, the probability of it occurring, and the contingency plan to overcome the risk is detailed in this tab.
8. **Issue**: Any issue that rises during the project is detailed in this tab.
9. **Project Change**: If the project requires some changes, add the change details in this tab.
10. **Incident**, **Problem**, **Change**, **Service Request**, **Knowledge**: You can link these to the project from the relevant tabs.
11. **Activity History**: All emails from the project are displayed in this tab.
12. **Audit History**: All changes impacting the project status and schedules are displayed in this tab.
