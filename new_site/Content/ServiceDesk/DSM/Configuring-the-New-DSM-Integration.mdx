The Desktop and Server Management (DSM) integration configuration uses web services. Be sure to configure the DSM integration web services from the **DSM Integration** workspace and not from the **Web Service Connections (BETA)** workspace.

To set up your \{\{FRS\_VARS.ProductName}} application for the DSM integration:

1. From the \{\{FRS\_VARS.Configuration\_console}}, click **Extend > DSM Integration** to open the **DSM Integration** workspace.
2. Enter information into the fields.

| Field                                                                                                     | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| --------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| DSM Web Services called from SM                                                                           |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Web Service Address                                                                                       | The web server address of DSM. This is the URL that \{\{FRS\_VARS.ProductName}} uses to call DSM.<br />Click **Test Connection** to test the connection and make sure that the application can reach the web service.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| User Name                                                                                                 | The user name of a user in DSM, such as "DSMAdmin". This user must have permission to run requests from \{\{FRS\_VARS.ProductName}}.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| User Unique ID                                                                                            | The unique ID for the DSM user. This value is from the DSM Console, from the field called **Properties > Object > Unique ID**.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| Web Services called from DSM                                                                              |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Status Update URL                                                                                         | The \{\{FRS\_VARS.ProductName}} inbound web service server name appended with the \{\{FRS\_VARS.ProductName}} tenant name. The application automatically populates this field with a value entered in the System Configuration Wizard when you initially installed the application. If you change this value, the URL must be in this format: http(s):// *SM\_server*:*Port*/InboundWebservice/*Tenant\_URL*.asmx<br /><br />Click **Test Connection** to test the connection and make sure that the application can reach the web service.<br /><br />This is the same value as the **Infrastructure > Configuration > ITSM Integration Settings > ITSM 7: URL of the DSM Integration Service (ITSM)** field in the DSM Console.                                          |
| Authentication Key                                                                                        | The authentication key. Select one from the drop-down list. The drop-down list is populated from the API keys in \{\{FRS\_VARS.ProductName}}, which are generated from the **API Keys** workspace. See :Link[Configuring the API Key]{label="Configuring the API Key" overridedLabel="Configuring the API Key" spaceId docId="KwnvB4RB_9eQ0euWXsSNc" version="v2" docAnchorId="#Configuring-the-API-Key" loadingMethod="dynamic" newTab="false" githubPath="../../Configure/ImportExportData/Accept-Messages.htm#Configuring-the-API-Key" href="../../Configure/ImportExportData/Accept-Messages.mdx"}.<br /><br />This is the same value as the **Infrastructure > Configuration > ITSM Integration Settings > Heat Cloud: Authentication key** field in the DSM Console. |
| Other DSM Integration Configuration                                                                       |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Web Service Script List                                                                                   | Lists the web service scripts associated with the new DSM integration. If your application has a new DSM integration, these web service scripts must be turned on.<br /><br />Only use web service scripts if you are implementing a new DSM integration. These are not used with the DSM legacy integration, and if they are used, the application gives an error.<br /><br />Click **Activate All** to activate all of the web service scripts or click **Deactivate All** to deactivate all of the web service scripts.                                                                                                                                                                                                                                                 |
| Legacy DSM Integration Service Information<br />This section is only used for the DSM legacy integration. |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| URL                                                                                                       | Automatically populated by the application. Specifies the DSM URL for the DSM legacy integration.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| Client Authentication Key                                                                                 | Automatically populated by the application. Specifies the authentication key for the DSM legacy integration.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |

3. Click **Save**.
