1. Log in to the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Console}} console.
2. Open the **Request Offering** workspace.
3. Click **New > Request Offering**.

<hint type="info" indent="1">
  If the **Service** name appears as GUID, the service has been deleted or is marked non-published.
</hint>

4. Enter information into the fields.

## **Design request form**

1. Click **Change Image** to edit the default image. See [Default Icons for Request Offerings](Default-Icons-for-Request-Offerings.mdx).
   - The image size cannot be larger than 50 KB.
   - We recommend using square images.
   - The image size can be as large as you like, but is scaled down to 48 by 48 pixels.
2. Click **Layout Configuration** to define into how many columns the fields are to be arranged on the form.

  *Layout Configuration*
</Paragraph>

| --------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Align rows in columns       | Aligns your rows into columns. Only useful to forms with multiple columns.<br />Items are placed on the page from left to right into columns, then down the page. Items of different sizes can cause the layout to look jagged. This option aligns the columns of each row. |
| Show column number in a row | Shows the column placement. When you place an item on the form, an icon appears showing which column it inhabits. This is most useful when using a two or three column display.                                                                                             |

4. Drag items from the **Form Designer Tools** pane on the right onto the **Request Form** area.

## **Plan Request Fulfillment**

<hint type="info">
  - You cannot edit the workflow if the production metadata is in read-only mode. When this happens, the system displays a message.
  - For more information about setting the production metadata to read-only, see the *Configuration Database Guide for&#x20;*\{\{FRS\_VARS.ProductName}}.
</hint>

1. Select a fulfillment option from the **Fullfill as** drop-down list.

    <td>
      ### Copying a Workflow Between Request Offerings### Copying a Workflow for an Existing Request Offering### Copying a Workflow for a New Request Offering### Verifying the Copied Workflow in a Service Request
      Creates a workflow in the Workflow Designer. See [Using Workflows](../../Configure/Workflows/Using_Workflows.mdx).
      You can define a multi-step fulfillment workflow with the Workflow Designer. The designer supports the inclusion of all required approvals and fulfillment tasks. If financial approval is required, include an approval block dedicated for financial approval.
      All Service Request submissions with associated workflows are queued as a high priority. Once queued, the workflow uses the priority defined in the Request Offering.
      Administrators can reuse existing workflows to create new workflows or update workflows for Request Offerings. The system clones the selected workflow in the backend, creating an independent copy for the target Request Offering.
      This approach allows administrators to use an existing workflow as a template and modify it to meet the requirements of the new or updated Request Offering, eliminating the need to build workflows from scratch.
      To copy a workflow to an existing Request Offering, perform the following:
      1. Log in to \{\{Primary.Product}} as an \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}.
      2. Click **More** > **Request Offering**.
      The list of available Request Offerings is displayed.
      3. Select the Request Offering for which you want to replace the workflow.
      4. Open the **Plan Request Fulfillment** tab.
      5. Locate and click **Use Existing Workflow**.
      The **Use Existing Workflow** dialog box opens.
      6. Select the **Service** and **Request Offering** that contains the workflow you want to reuse.
      7. Click **Replace Workflow**.
      8. Click **Save**.
      The selected workflow replaces the existing workflow and is saved to the database.
      To copy a workflow while creating a new Request Offering, perform the following:
      1. Log in to \{\{Primary.Product}} as an \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}.
      2. Click **More** > **Request Offering**.
      3. Click **New Offering**.
      4. In the **Define Request Offering** tab, complete the required details across all tabs.
      5. Open the **Plan Request Fulfillment** tab.
      6. Click **Use Existing Workflow**.
      The Use Existing Workflow dialog box opens.
      7. Select the **Service** and **Request Offering** that contains the workflow you want to reuse.
      8. Click **Replace Workflow**.
      9. Open the **Public Action Access** tab and select **Publish**.
      10. Click **Save**.
      After creating the Request Offering, verify that the workflow is available by performing the following:
      1. Navigate to **Service Request**.
      2. Click **New** and enter the required request details.
      3. Open the Parameters tab and locate the workflow copied from the Request Offering.
      4. Click **Save**.
      <Paragraph indent="1">
        Confirm that the workflow executes as expected.
      </Paragraph>
    </td>
  </tr>
</table>

## **Publish Action Access**

1. Select an option from the **Status** drop-down list.

| Field     | Description                                                                                                                                                                                                                                                                                                                                  |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| In Design | Saves the form without publishing it to users. You can modify this form later.                                                                                                                                                                                                                                                               |
| Published | Saves the form and makes the request offering available under request offerings, but not under the Service Catalog. When setting this status, by default the organizational unit of the creator/owner is added with all permissions checked. This offering is available in the Service Catalog for this organizational unit unless modified. |

2. Click **Add**.

| Field     | Description                                                                                                                                                                                                                                                                                                                    |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Org. Unit | Selects an organizational unit from the list. You can only publish request offerings to organizational units to which you have access. For example, if you do not have access to the entire organization, your published offering will only be available to users in the organizational unit that you have selected and below. |
| Location  | Only publishes to specific locations. You can designate multiple locations.                                                                                                                                                                                                                                                    |
| Submit    | To submit the request offering from the Service Catalog by the designated organizational unit.                                                                                                                                                                                                                                 |
| Edit      | To modify the request offering (not the resulting service request).                                                                                                                                                                                                                                                            |
| Cancel    | To cancel the request offering.                                                                                                                                                                                                                                                                                                |
| Delete    | To delete the request offering (not the resulting service request).                                                                                                                                                                                                                                                            |
| Copy      | To duplicate the request offering (not the resulting service request).                                                                                                                                                                                                                                                         |

3. Click the **remove** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/USPl9Egu4Y2Blx7HKC3mA_deleteredx.png" alt caption size="10" width="14" height="13"} to remove the organizational unit from the list.

<hint type="info" indent="1">
  - The request offering owner/creator has edit permissions by default.
  - You can only publish to an organizational unit to which you belong, and its parent and child units.
</hint>

4. Click **Save**, **Save & Exit**, or **Cancel**.
