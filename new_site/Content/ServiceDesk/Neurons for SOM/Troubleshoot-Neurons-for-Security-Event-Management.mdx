Use these solutions to troubleshoot the software.

- **An error appears when using the Quick Actions on a Security Event:**

<Paragraph indent="1">
  You may see an error that says the Category or Service cannot be set when the Create Incident Action is used. To resolve this, modify the Create Incident Quick Action. Refer to Modifying a Quick Action in [Using Quick Actions](../../Configure/QuickActions/Using_Quick_Actions.mdx)
</Paragraph>

<Paragraph indent="1">
  a. Open the Actions Manager workspace.
</Paragraph>

<Paragraph indent="1">
  b. Edit the **Create Incident** Action, and then change the **Service** field value to a different service (you can use any other value). You may also need to change the **Category** to a different value.
</Paragraph>

<Paragraph indent="1">
  c. Select **Save**.
</Paragraph>

- **Importing events from an external source may fail:**
  - If you're using Microsoft Windows Server 2012 R2 for your \{\{FRS\_VARS.ITSM}} instance, importing events may not work due to an SSL authentication issue with this version of Windows.
