Perform the following procedures to connect \{\{FRS\_VARS.Neurons\_for\_SOM}} to your Jira instance.

<ExpandableHeading>
  ## Credentials and Initial Setup

  1. Log into \{\{FRS\_VARS.ITSM}} as an Administrator and open the **Ticket Sharing and Synchronization** (TSS) workspace.
  2. Select **New Ticket Sharing and Synchronization**.
  3. Enter th&#x65;**&#x20;Integration Name** (example: Jira).
  4. Select **Incident&#x20;**&#x66;rom the **Business Object Name** menu.
  5. Select **Enabled&#x20;**&#x66;rom the Status menu.
  6. In the **Service Name** field, find and select Jira.
  7. Save.

  <Paragraph indent="1">
    Four new tabs now appear after the **Credentials&#x20;**&#x74;ab: **Filters**,**&#x20;Field Mapping**, **Data Mapping**, and **Data Rules**.
  </Paragraph>

  8. The **Credentials&#x20;**&#x74;ab now contains three additional fields:

  <Paragraph indent="1">
    a. **Service URL**: Relevant Jira instance URL (example: https\://customer version.atlassian.net/rest/api/2).
  </Paragraph>

  <Paragraph indent="1">
    b. **Username**: Jira account username (example: JiraUsername\@company.com).
  </Paragraph>

  <Paragraph indent="1">
    c. **Auth Token**: Token provided by Jira (example: Token1232323232323).
  </Paragraph>

  9. Save.
</ExpandableHeading>

<ExpandableHeading>
  ## Filters Tab

  1. In the **Filters&#x20;**&#x74;ab, click **New Connector Service Filters**.
  2. In the **New Connector Service Filters** dialog, select **Summary&#x20;**&#x66;rom the **Incident Field Name** menu.
  3. In th&#x65;**&#x20;Incident Field Value** field, enter Trojan.
  4. **Operation with next Filter** determines how filtering works with other filters. Select the appropriate operator (AND or OR) (example: add a “Ransomware Detected” filter).
  5. When all appropriate filters have been added, Save.
</ExpandableHeading>

<ExpandableHeading>
  ## Field Mapping Tab

  1. In the **Field Mapping** tab, click **New Connector Service Field Mapping**.

  <Paragraph indent="1">
    a. In the **New Connector Service Field Mapping&#x20;**&#x64;ialog, select **Urgency&#x20;**&#x66;or th&#x65;**&#x20;Incident Field Name**.
  </Paragraph>

  <Paragraph indent="1">
    b. Enter priority for the **Jira Field Name**. This field is in lowercase in Jira.
  </Paragraph>

  <Paragraph indent="1">
    c. Select **Poll this field for any changes**.
  </Paragraph>

  <Paragraph indent="1">
    d. Save.
  </Paragraph>

  2. Clic&#x6B;**&#x20;New Connector Service Field Mapping**.

  <Paragraph indent="1">
    a. In the **New Connector Service Field Mapping** dialog, select **Summary&#x20;**&#x66;or th&#x65;**&#x20;Incident Field Name**.
  </Paragraph>

  <Paragraph indent="1">
    b. Enter summary for the **Jira Field Name** . This field is in lowercase in Jira.
  </Paragraph>

  <Paragraph indent="1">
    c. Enter summary for the **Service Field Depth**.
  </Paragraph>

  <Paragraph indent="1">
    d. Select **Poll this field for any changes**.
  </Paragraph>

  <Paragraph indent="1">
    e. Save.
  </Paragraph>

  3. Clic&#x6B;**&#x20;New Connector Service Field Mapping**.

  <Paragraph indent="1">
    a. In the **New Connector Service Field Mapping** dialog, select **Provide default value** (to switch field mapping from Jira to ISM).
  </Paragraph>

  <Paragraph indent="1">
    b. Enter project for th&#x65;**&#x20;Jira Field Name**.
  </Paragraph>

  <Paragraph indent="1">
    c. Enter ESM for the **Default Value**.
  </Paragraph>

  <Paragraph indent="1">
    d. Enter project^key for th&#x65;**&#x20;Service Field Depth**.
  </Paragraph>

  <Paragraph indent="1">
    e. Save.
  </Paragraph>

  4. Clic&#x6B;**&#x20;New Connector Service Field Mapping**.

  <Paragraph indent="1">
    a. In th&#x65;**&#x20;New Connector Service Field Mapping** dialog, select **Description&#x20;**&#x66;or the **Incident Field Name**.
  </Paragraph>

  <Paragraph indent="1">
    b. Enter description for the **Jira Field Name**. Note: This field is in lowercase in Jira.
  </Paragraph>

  <Paragraph indent="1">
    c. Enter description for the **Service Field Depth**.
  </Paragraph>

  <Paragraph indent="1">
    d. Save.
  </Paragraph>

  5. Clic&#x6B;**&#x20;New Connector Service Field Mapping**.

  <Paragraph indent="1">
    a. In th&#x65;**&#x20;New Connector Service Field Mapping&#x20;**&#x64;ialog, select **Status&#x20;**&#x66;or the **Incident Field Name**.
  </Paragraph>

  <Paragraph indent="1">
    b. Enter status for the **Jira Field Name**. Note: This field is in lowercase in Jira.
  </Paragraph>

  <Paragraph indent="1">
    c. Select **Poll this field for any changes**.
  </Paragraph>

  <Paragraph indent="1">
    d. Save.
  </Paragraph>

  6. Clic&#x6B;**&#x20;New Connector Service Field Mapping**.

  <Paragraph indent="1">
    a. In th&#x65;**&#x20;New Connector Service Field Mapping** dialog, select **Provide default value** (to switch field mapping from Jira to ISM).
  </Paragraph>

  <Paragraph indent="1">
    b. Enter issuetype for the **Jira Field Name**.
  </Paragraph>

  <Paragraph indent="1">
    c. Enter Task for the **Default Value**.
  </Paragraph>

  <Paragraph indent="1">
    d. Enter issuetype^name for the **Service Field Depth** field.
  </Paragraph>

  <Paragraph indent="1">
    e. Save.
  </Paragraph>
</ExpandableHeading>

  ## Data Rules Tab

  1. In the Data Rules tab, select **New Connector Service Field Constraint**.

  <Paragraph indent="1">
    a. In the **Field Mapping Value** menu, select **Incident=Status :: Jira=Status\[Closed,Done]**.
  </Paragraph>

  <Paragraph indent="1">
    b. In the **Business Object Field Name** menu, select **CauseCode**.
  </Paragraph>

  <Paragraph indent="1">
    c. Enter Hardware for the **Business Object Field Value**.
  </Paragraph>

  <Paragraph indent="1">
    d. Save.
  </Paragraph>

  2. Clic&#x6B;**&#x20;New Connector Service Field Constraint**.

  <Paragraph indent="1">
    a. In the **Field Mapping Value** menu, select **Incident=Status :: Jira=Status\[Closed,Done]**.
  </Paragraph>

  <Paragraph indent="1">
    b. In the **Business Object Field Name** menu, select **Resolution**.
  </Paragraph>

  <Paragraph indent="1">

  </Paragraph>

  <Paragraph indent="1">

<ExpandableHeading>
  ## Additional Operations

  1. Click the **Additional Operations** tab. If the tab is not visible, you may have to refresh or go to List View and select your TSS again.
  2. Select the following options:
     - **Push Comments added in HEAT to Jira**
     - **Pull Comments added in Jira to HEAT**
     - **Push Attachments added in HEAT to Jira**
     - **Pull Attachments added in Jira to HEAT**
  3. In the **Comment Business Object** menu, select **Journal#Notes**.
  4. Save.
</ExpandableHeading>

<ExpandableHeading>
  ## Comment Field Mapping

  1. Click th&#x65;**&#x20;Comment Field Mapping** tab. If the tab is not visible, you may have to refresh or go to List View and select your TSS again.
  2. Click **New Comment Field Mapping**.

  <Paragraph indent="1">
    a. In the **Incident Field Name** menu, select **Notes**.
  </Paragraph>

  <Paragraph indent="1">
    b. Enter Description for th&#x65;**&#x20;Jira Field Name**.
  </Paragraph>

  <Paragraph indent="1">
    c. Save.
  </Paragraph>

  3. Click on New Comment Field Mapping.

  <Paragraph indent="1">
    a. In the **Incident Field Name** menu, select **Source**.
  </Paragraph>

  <Paragraph indent="1">
    b. Leave the **Jira Field Name** blank.
  </Paragraph>

  <Paragraph indent="1">
    c. Enter Other for the **Incident Default Value**.
  </Paragraph>

  <Paragraph indent="1">
    d. Save.
  </Paragraph>

  4. Click on New Comment Field Mapping.

  <Paragraph indent="1">
    a. In the **Incident Field Name** menu, select **Subject**.
  </Paragraph>

  <Paragraph indent="1">
    b. Enter Summary for the **Jira Field Name**.
  </Paragraph>

  <Paragraph indent="1">
    c. Save.
  </Paragraph>
</ExpandableHeading>
