Set up Constants prior to configuring a Jira or ADO integration. Without properly configured Constants, it is impossible to run the \{\{FRS\_VARS.Quick\_Action}} to import Security Incident fields to use for TSS (Ticket Sharing and Synchronization), and the TSS Workflows won’t run.

1. Sign in to \{\{FRS\_VARS.ProductName}} as an \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}. Open the Constants workspace. Refer to the following table for values:

| Name                | Value                                           | Comment                                                               |
| ------------------- | ----------------------------------------------- | --------------------------------------------------------------------- |
| HEATServiceName     | FRSHEATIntegration                              | Do not change                                                         |
| HEATServicePassword |                                                 | Password used to log in to ISM (internally: “manage” - Do not change) |
| HEATServiceRole     | Administrator                                   | Do not change                                                         |
| HEATServiceTenantID | tenant name+domain name                         |                                                                       |
| HEATServiceURL      | Full URL -including “http(s)”, followed by API  |                                                                       |
| HEATServiceUsername | name of authorized Administrator                |                                                                       |
| TenantID            | Full URL - including “http(s)”, followed by API |                                                                       |
| TenantPassword      | Password used to log in to ISM                  |                                                                       |
| TenantRole          | Administrator                                   |                                                                       |
| TenantURL           | tenant name+domain name                         |                                                                       |
| TenantUsername      | name of authorized Administrator user           |                                                                       |

2. Make any modifications necessary and save.
