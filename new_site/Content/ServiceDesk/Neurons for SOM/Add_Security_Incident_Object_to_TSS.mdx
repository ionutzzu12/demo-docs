Add the Security Incident business object to the Ticket Sharing and Synchronization (TSS) feature.

**To add the Security Incident business object to TSS:**

1. Sign in to \{\{FRS\_VARS.ProductName}} as an \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}. Open the Business Objects workspace.
2. Select **New Business Object**.

<Paragraph indent="1">
  a. For the **Business Object Name**, enter ivnt\_SecurityIncident.
</Paragraph>

<Paragraph indent="1">
  b. For the **Business Object Name Display Name**, enter Security Incident.
</Paragraph>

<hint type="info">
  Both names can be verified in the **Configuration Console > Build > Business Objects > Security Incident**.
</hint>

<Paragraph indent="1">
  a. Save.
</Paragraph>

3. The **Fields&#x20;**&#x74;ab should be updated and include a number (currently 149). If the tab is not updated, click **Refresh&#x20;**&#x6F;r click on the **Fields&#x20;**&#x74;ab.

<Paragraph indent="1">
  You can also run the **Create fields for BO** \{\{FRS\_VARS.Quick\_Action}} manually if needed by clicking **Action Menu → Connector** (for example, to update the fields list if new fields were added later to the Security Incident business object).
</Paragraph>

4. Click the **Comment Business Object** tab. Click **New Comment BO**.

<Paragraph indent="1">
  In the **Comment Business Object** field, select **Journal Notes**. Leave the **ParentBOLink&#x20;**&#x75;nchanged.
</Paragraph>

5. Save.
6. To check that the operation succeeded, open the **Ticket Sharing and Synchronization** workspace. Clic&#x6B;**&#x20;New Ticket Sharing and Synchronization** and pull down the options available in the **Business Object Name** field. Security Incident should now be available.
