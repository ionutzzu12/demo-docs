You can perform several functions from the \{\{FRS\_VARS.Mobile\_Analyst\_portal}}. For more information, check out the links below:

- Create, view, edit, track, and resolve incidents. Refer to [Working with Incidents](Mobile-Analyst-Incident.mdx).
- Create, view, edit, track, and fulfill Service Requests. Refer to [Working with Service Requests](Mobile-Analyst-Service-Requests.mdx).
- View and edit Configuration Item (CI). Assign user to CI and create incident for CI. Refer to [Working with Configuration Item](Working-with-Configuration-Items.mdx).
- Create, view, edit, track, assign, and complete tasks. Refer to [Working with Tasks](Mobile-Analyst-Task.mdx).
- Add emails, notes, and attachments to incidents and tasks. Refer to Working with [Working with Journal and Attachments](Mobile-Analyst-Journals-and-Attachments.mdx).

<hint type="info">
  * Ensure that you have [defined and enabled](../../Configure/Constants/Defining_Global_Constant.mdx) the EnableNewUIElements :Link[global constant]{label="global constant" overridedLabel="global constant" spaceId docId="6CPp-wWweHGMGP37lRlsY" version="v2" docAnchorId="#EnableNewUIElements" loadingMethod="dynamic" newTab="false" githubPath="../../Configure/Constants/Default_Global_Constants.htm#EnableNewUIElements" href="../../Configure/Constants/Default_Global_Constants.mdx"} to enhances additional capabilities on Mobile Analyst experience, for example: add email, notes, and attachments.
  * When the mobile app is idle for a longer time, you will be logged out. This is based on the SSO session timeout configuration.
</hint>
