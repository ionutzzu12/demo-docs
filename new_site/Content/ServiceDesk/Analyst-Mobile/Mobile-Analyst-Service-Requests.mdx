From your mobile device you can create Service Requests, view all the Service Requests assigned to you, edit them, complete the tasks for the \{\{FRS\_VARS.Service\_Requests}}, and add notes and tasks to the Service Requests.

On the Service Request workspace page, you can filter and sort the service requests by expanding the **Filter & Sort** drop-down. You can expand the **Filter By** option to select the suitable options that fit your search criteria, like Status, Priority, Resolve By, Owner, Scanned Period, and so on. Similarly, you can expand the **Sort By** option and select suitable options from the drop-down list. You can also contact your administrator to customize your filters and sorts.

On the Service Request workspace page, you can select the Tile View icon to view all the service requests in a tile view format, where you can get additional information about the user. When you click the image circle under any particular Service Request, a user information card pops up and displays the username, designation, department, email address, and phone number.

<hint type="info">
  You can use this feature only on Android and iOS mobile devices and not on iPad or desptop.
</hint>

Some of the fields may not be compatible in responsive user interface. <ExpandableHeading>
  ## Creating a Service Request

  1. From the **Home** page, tap the :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/KwVyQ2a8d-_Mm_KDj8Mbm_newiconra.png" alt caption size="10" width="21" height="20"} icon and select **Create Service Request**.
  2. Enter information into the fields:
     - **Customer**, **Team**, **Owner**, and **Urgency**.
     - **Status** - Initially it is **Active**. Change the status as the Service Request progresses through various stages until it is finally **Fulfilled**.
  3. Tap **Next**. This button is enabled only when you enter or modify the details of one of the field.

  <Paragraph indent="1">
    A list of Request Offerings is displayed.
  </Paragraph>

  4. You can scroll and select or filter by category and/or search by a keyword and select a Request Offering.
     - To filter - tap the :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/xXCM2B_4qMEv289yb68lE_filtericonmobileanalyst.jpg" alt caption size="10" width="16" height="16"} icon and select one or more category check boxes and click **Apply**.

  <Paragraph indent="2">
    When you select the **Most Popular** category, you cannot select any other category.
  </Paragraph>

  <Paragraph listStyleType="disc" indent="2">
    To search - enter a keyword in the **Search** bar.
  </Paragraph>

  <Paragraph indent="2">
    When you apply a filter and then search, the search results display matching results from the selected categories only.
  </Paragraph>

  5. Select a Request Offering and enter information into the fields.

  <Paragraph indent="1">
    The fields displayed vary based on the selected Request Offering.
  </Paragraph>

  6. Tap **Review**. This button is enabled only when all the mandatory fields are filled.
  7. Tap the :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/VJjW1XI5rfp7nChWXIXN7_backicon.png" alt caption size="10" width="16" height="15"} icon to make any changes.
  8. Select **Yes** in the prompt message. Modify as needed and tap **Review**.
  9. Tab **Submit**. The Service Request is created.

  <hint type="info">
    On the **Home** page, this :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/vcqt-ZcbeTDXjVcaIp8qK_mainbuttoniconmobileanalyst22x22.png" alt caption size="10" width="22" height="22"} icon is available in the header. It helps to create service requests and it is accessible from all the pages.
  </hint>

  <hint type="info">
    Request Offering Templates might not appear for users with the Mobile Analyst role if the required object permissions are not enabled. To resolve this, administrators must configure the appropriate permissions in the Configuration Console. The following steps outline how to enable access so Mobile Analysts can view and use Request Offering Templates seamlessly.

    1. Log in to Neurons for ITSM as an administrator.
    2. Click **Configuration Console**.
    3. Expand **Roles and Permissions**, select **Mobile Analyst**.
    4. Click **Object Permissions** and search for **SelfServiceFavorite**.
    5. Enable **View permissions** for **SelfServiceFavorite**.
    6. Click **Save** to apply the changes.
  </hint>

  1. From the **Home** page, tap the **My Service Reqs** tab.

  <Paragraph indent="1">

  </Paragraph>

  2. Select one to view its details.

<ExpandableHeading>
  ## Editing a Service Request

  1. From the **Home** page, tap the **My Service Reqs** tab.

  <Paragraph indent="1">
    A list of all active \{\{FRS\_VARS.Service\_Requests}} you are assigned on is displayed.
  </Paragraph>

  2. Select one to view and edit.
  3. Tap **Edit**. The Service Request is displayed in edit mode.
  4. Expand the **Customer Info** section. This section has all the information pertaining to the \{\{FRS\_VARS.Service\_Request-Service-Request}}.

  <hint type="info">
    You can edit the fields in the **Customer Info** tab only if the \{\{FRS\_VARS.Request\_Offering-Request-Offering}} used in the \{\{FRS\_VARS.Service\_Request-Service-Request}} is set as editable.
  </hint>

  5. Edit the details as required and tap **Save&#x20;**&#x74;o save the changes.
</ExpandableHeading>

  ## Working with Tasks in a Service Request

  You can view, accept, and complete tasks assigned to you in a \{\{FRS\_VARS.Service\_Request-Service-Request}}.

  1. From the **Home** page, tap the **My Service Reqs** tab.

  <Paragraph indent="1">
    A list of all active \{\{FRS\_VARS.Service\_Requests}} you are assigned on is displayed.
  </Paragraph>

  2. Select one to open and view its details.
  3. Expand the **Tasks&#x20;**&#x73;ection. A list of all tasks are displayed with the task owner name.
  4. Select a task you are assigned as the owner and tap **Edit**.

  <Paragraph indent="1">
    The task is displayed in edit mode.
  </Paragraph>

  5. You can perform the following actions for a Task:

  <Paragraph indent="1">
    a. To complete the task - tap the More <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/bhZC2RDjm9lZUowq_FLnW_moreiconmobileanalyst.jpg" size="10" isUploading="false" initialPath="../../Resources/Images/MobileAnalyst/buttons/MoreIconMobileAnalyst.jpg" githubPath="Content/Resources/Images/MobileAnalyst/buttons/MoreIconMobileAnalyst.jpg" width="7" height="16" showCaption="false"></Image> and select **Completed**. <ExpandableHeading>
  ## Adding a Task to a Service Request

  You can add tasks for a \{\{FRS\_VARS.Service\_Request-Service-Request}} and assign a relevant owner to it.

  1. From the **Home** page, tap the **My Service Reqs** tab.

  <Paragraph indent="1">
    A list of all active \{\{FRS\_VARS.Service\_Requests}} you are assigned on is displayed.
  </Paragraph>

  2. Select a \{\{FRS\_VARS.Service\_Request-Service-Request}} for which you wish to add a task to.
  3. Tap the **Add Task** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/Dttexsgzjm8NsSc5yRlLb_mobileanalystaddtaskicon.png" alt caption size="10" width="16" height="13"}. The Create New Task page is displayed.
  4. Enter the **Summary**, select a **Team**, and **Owner**, and tap **Save**.
</ExpandableHeading>
