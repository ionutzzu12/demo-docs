**Role** : \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}

You can track and visualize \{\{Primary.Product}} information, such as cost, value, and consumption. Analytic metrics display your data graphically and allow you to chart both current data and trends. The charts are flexible, customizable, and easy to modify.

Key advantages:

- View analytic dashboards on smart devices with a supported browser.
- Save and download chart images for email or other media.
- Access real-time transaction data from operational activities across service management workspaces.
- Use threshold-based and trend information in the Analytic Metrics workspace to retrieve selected data.

**Component of Analytics**

- **Visualization component**: Supports ten chart types. You can configure visual cues to indicate when key performance indicators (KPIs) are exceeded. For more information, see :Link[Chart Types]{label="Chart Types" overridedLabel="Chart Types" spaceId docId="Odz29FwTIQccM-QaEZEo0" version="v2" docAnchorId="#Chart_Types" loadingMethod="dynamic" newTab="false" githubPath="Metric Dashboards and Charts.htm#Chart_Types" href="Metric Dashboards and Charts.mdx"}.
- **Data source management**: Provides access to all service management objects and saved searches.
- **Analytic metric definition**: Allows you select, group, and update specific data. For more information, see :Link[Defining the Metric Settings]{label="Defining the Metric Settings" overridedLabel="Defining the Metric Settings" spaceId docId="iuN7d8f-bAz9LSqHtfX3m" version="v2" docAnchorId="#Metric_Settings" loadingMethod="dynamic" newTab="false" href="Analytic Metrics.mdx"}.
- **Role based Access** : To allow non administrator roles to create and publish dashboards and dashboard parts, see [Adding Layouts to a Role](<../../Configure/Layouts/Assigning Layouts to Roles.mdx>).

  **Multiple Data Sources**: You can compare data from multiple sources, or from a single source using different saved searches, by using the split series function. See :Link[Adding or Editing a Data Source]{label="Adding or Editing a Data Source" overridedLabel="Adding or Editing a Data Source" spaceId docId="iuN7d8f-bAz9LSqHtfX3m" version="v2" docAnchorId="#Data_Source" loadingMethod="dynamic" newTab="false" href="Analytic Metrics.mdx"}.

  ## Creating a Metric

  <hint type="info">
    The application updates Analytic Metrics dashboards every 60 minutes. You cannot change this interval.
  </hint>

  1. Log into the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Console}}.
  2. Open the **Analytic Metrics** workspace.
  3. In the top right corner, click **Edit Mode**.
  4. From the **Settings** menu, click **Manage Metrics**.

  <Paragraph indent="1">
    The **Manage Metrics** workspace appears.
  </Paragraph>

  5. Click **New**.
  6. choose one of the following options:
     - **Create Metric** - Create a new measurement.
     - **Historical Metric** - Track historical data, such as total down time for a selected period.
  7. Enter information into the fields:

  | Field       | Description                                                                    |
  | ----------- | ------------------------------------------------------------------------------ |
  | Category    | Choose a category from the dropdown list, or enter a name to create a new one. |
  | Name        | Enter a definition. This name appears in the definition list.                  |
  | Description | (Optional) Enter a description of the metric.                                  |

  8. Add additional information as described on the following tabs:
     - **Data Source**: See :Link[Adding or Editing a Data Source]{label="Adding or Editing a Data Source" overridedLabel="Adding or Editing a Data Source" spaceId docId="iuN7d8f-bAz9LSqHtfX3m" version="v2" docAnchorId="#Data_Source" loadingMethod="dynamic" newTab="false" href="Analytic Metrics.mdx"}.
     - **Analytic Snapshot**: See :Link[Configuring Analytic Snapshot Information]{label="Configuring Analytic Snapshot Information" overridedLabel="Configuring Analytic Snapshot Information" spaceId docId="iuN7d8f-bAz9LSqHtfX3m" version="v2" docAnchorId="#Analytic_Snapshot" loadingMethod="dynamic" newTab="false" href="Analytic Metrics.mdx"}.
     - **Action Triggers**: See :Link[Creating an Action Trigger]{label="Creating an Action Trigger" overridedLabel="Creating an Action Trigger" spaceId docId="iuN7d8f-bAz9LSqHtfX3m" version="v2" docAnchorId="#Creating2" loadingMethod="dynamic" newTab="false" href="Analytic Metrics.mdx"}.
     - **Access Rights**: See :Link[About the Access Rights Tab]{label="About the Access Rights Tab" overridedLabel="About the Access Rights Tab" spaceId docId="iuN7d8f-bAz9LSqHtfX3m" version="v2" docAnchorId="#About_the_Access_Rights_Tab" loadingMethod="dynamic" newTab="false" href="Analytic Metrics.mdx"}.
     - **Object Mapping**: See :Link[About Object Mapping]{label="About Object Mapping" overridedLabel="About Object Mapping" spaceId docId="iuN7d8f-bAz9LSqHtfX3m" version="v2" docAnchorId="#About_Object_Mapping" loadingMethod="dynamic" newTab="false" href="Analytic Metrics.mdx"}.
  9. Test and save the metric:

  <Paragraph indent="1">
    a. At the bottom of the window, choose the chart type.
  </Paragraph>

  <Paragraph indent="1">
    b. Click **Run**. The application runs the metric and opens the **Chart Preview** window.
  </Paragraph>

  <Paragraph indent="2">
    The execution time appears in the window title bar.
  </Paragraph>

  <Paragraph indent="1">
    c. Close the window to continue.
  </Paragraph>

  <Paragraph indent="1">
    d. Click **Save**. The definition appears in the list in alphabetical order.
  </Paragraph>
</ExpandableHeading>

The **Access Rights** tab allows you to manage which roles have access to this metric.

  ## About Historical Metrics

  ## Creating a Historical Metric

  1. Activate the historical data from within the metric chart you want from the **Analytic Snapshot** tab. See :Link[Configuring Analytic Snapshot Information]{label="Configuring Analytic Snapshot Information" overridedLabel="Configuring Analytic Snapshot Information" spaceId docId="iuN7d8f-bAz9LSqHtfX3m" version="v2" docAnchorId="#Analytic_Snapshot" loadingMethod="dynamic" newTab="false" href="Analytic Metrics.mdx"}.
  2. Click **Edit Mode** in the top right corner to go into edit mode.
  3. From the **Settings** menu, click **Manage Metrics**. The **Manage Metrics** workspace appears.
  4. Click **New** and choose **Historical Metric**.
  5. On the **General** tab, enter the required information.

  | Field       | Description                                                                                        |
  | ----------- | -------------------------------------------------------------------------------------------------- |
  | Category    | The category. Choose a category from the drop-down list, or enter a name to create a new category. |
  | Name        | A definition. This name appears in the definition list.                                            |
  | Description | Optional. A description of the metric.                                                             |

  6. Click the **Data Source** tab and enter the required information.

  | Field          | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
  | -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
  | History from   | Choose a history from the drop-down list. This list is populated when metrics are collecting historical data. See :Link[Configuring Analytic Snapshot Information]{label="Configuring Analytic Snapshot Information" overridedLabel="Configuring Analytic Snapshot Information" spaceId docId="iuN7d8f-bAz9LSqHtfX3m" version="v2" docAnchorId="#Analytic_Snapshot" loadingMethod="dynamic" newTab="false" href="Analytic Metrics.mdx"}. The following fields might appear depending on the value chosen. |
  | Using series   | Choose from the drop-down list.                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
  | Show           | Choose from the drop-down list.                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
  | Detail Level   | The detail level.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
  | Intervals      | The number of times the application calculates the data. The default value is 1.                                                                                                                                                                                                                                                                                                                                                                                                                          |
  | Data Point     | Shows data points as a series.                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
  | Hide Zero Rows | Hides rows that have no data.                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |

  7. Click the **Access Rights** tab and move individual roles you want to have access to this metric from the list on the left to the right. You can also check **Allow access to All** to automatically give all roles access to the metric data.
  8. Click the **Object Mapping** tab and enter the required information. See :Link[About Object Mapping]{label="About Object Mapping" overridedLabel="About Object Mapping" spaceId docId="iuN7d8f-bAz9LSqHtfX3m" version="v2" docAnchorId="#About_Object_Mapping" loadingMethod="dynamic" newTab="false" href="Analytic Metrics.mdx"}.
  9. To test, at the bottom of the window, choose the chart type then click **Run**. The metric is run and the **Chart Preview** window opens. The execution time appears in the window title bar. Close the window to continue.
  10. Click **Save**. The definition appears in the list in alphabetical order.
</ExpandableHeading>

  ## Publishing Dashboards

  When you create a new dashboard, only you can view it. To make it available to others, publish it to specific roles:

  1. Log into the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Console}}.
  2. Open the **Analytic Metrics** workspace.
  3. In the top right corner, click **Edit Mode**.
  4. From the **Settings** menu, click **Publish Dashboards**.

  <Paragraph indent="1">
    The **Publish Dashboards** window appears.
  </Paragraph>

  5. From the **Role** menu, choose a role to which to publish your dashboards.
  6. To publish a dashboard, move it from the **Available Dashboards** list to the **Published Dashboards** list. You can:
     - Drag and drop the dashboard, or
     - Select a dashboard and use the arrow keys.
  7. To change the order of appearance, move the dashboards up or down in **Published Dashboards** list. You can:
     - Drag and drop the dashboard, or
     - Select a dashboard and use the arrow keys.
  8. To change the order of appearance, move the dashboards up or down in **Published Dashboards** list. You can:
     - Drag and drop the dashboard, or
     - Select a dashboard and use the arrow keys.
  9. If required, return to the **Role** menu, choose another role to publish the dashboards.
  10. Repeat steps 1-8 to publish and arrange dashboards for each role.
  11. Click **Save**.
</ExpandableHeading>
