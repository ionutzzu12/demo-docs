To cancel a change request, you must do so before you update the status to requested.

<hint type="info">
  Unless you are a \{\{FRS\_VARS.Neurons\_ITSM\_-\_Change\_Manager}}, you can only cancel a change request when it is in the logged state. \{\{FRS\_VARS.Neurons\_ITSM\_-\_Change\_Manager}}s can change the status of a change request at any time.
</hint>

## Canceling a Change Request that is in the Logged State

Follow this procedure to cancel a change request while it is still in the logged state.

1. Log in to \{\{FRS\_VARS.ProductName}} as a \{\{FRS\_VARS.Neurons\_ITSM\_-\_Change\_Manager}}.
2. Open the **Change** workspace.
3. Open the change request to modify.
4. Select **Canceled** from the **Status** drop-down list.
5. Click **Save**. The **Cancellation Reason** window open.
6. Enter the reason for canceling the request and click **OK**. The status of the change request is set to *"*&#x43;anceled".

<Paragraph indent="1">
  An email is sent to the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Change\_Manager}} and the change requester of the cancellation.
</Paragraph>

Canceling a Change Request that is in Other States



Follow the steps above and then continue here:

1. Click the **Workflow Instance** tab.
2. Select the workflow instance record, if any.
3. Click **Abort****&#x20;from the&#x20;****Workflow Instance** toolbar. The workflow instance status is set to aborted.
