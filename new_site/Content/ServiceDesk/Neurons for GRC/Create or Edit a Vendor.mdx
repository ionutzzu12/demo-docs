You can create a Vendor record from within \{\{FRS\_VARS.Company}} \{\{FRS\_VARS.Neurons\_for\_GRC}}, or create a Vendor Onboarding request from the \{\{FRS\_VARS.Company}} \{\{FRS\_VARS.Neurons}} \{\{FRS\_VARS.Self\_Service\_Portal-Self-Service}} portal.

When a Vendor Onboarding request is submitted from the \{\{FRS\_VARS.Self\_Service\_Portal-Self-Service}} portal, it is automatically added into the **Vendor** workspace list as a new Vendor business object record, with **Status** set to On Hold. The new Vendor record can be opened from the **Vendor** workspace and edited.

<hint type="info">
  To create or edit a Vendor, you must log in to \{\{Primary.Product}} as a **GRC Manager**.
</hint>

**To create a Vendor from within&#x20;**\{\{FRS\_VARS.Neurons\_for\_GRC}}**:**

1. Open the **Vendor** workspace.
2. Click **New Vendor** to open a blank **Vendor** form.
3. Enter the information into the **Overview** section fields as required.
4. Select the **Contact Details** tab, and enter the Vendor contact information.
5. Use the **Additional Contacts**, **Tasks**, **Journals**, **Attachments**, **Risk Assessments**, **Risks** and **Migration Plans** tabs to add any new associated records for the Vendor.

<Paragraph indent="1">
  Use the **Assets** tab to link the Vendor with any relevant assets.
</Paragraph>

6. Click **Save**.

**Create a Vendor Onboarding request:**

A Vendor Onboarding request is created in the \{\{FRS\_VARS.Company}} \{\{FRS\_VARS.Neurons}} \{\{FRS\_VARS.Self\_Service\_Portal-Self-Service}} portal. A new Vendor record is automatically created, and appears in the **Vendor** workspace list. The status On Hold is applied to all new Vendor records created from a Vendor Onboarding request .

1. Log in to the \{\{Primary.Product}} \{\{FRS\_VARS.Self\_Service\_Portal-Self-Service}} portal, or change role to **Self Service**.
2. Select **IT Services**, and then select **Service Catalog**.
3. Select **General Services**.
4. Select **Vendor Onboarding**, and then enter the Vendor details into the form.
   Refer to :Link[Vendor Onboarding Request]{label="Vendor Onboarding Request" overridedLabel="Vendor Onboarding Request" spaceId docId="lJ78xN-EK7cUwvxzMhnJN" version="v2" docAnchorId="#h3_Vendor_Onboarding_Request" loadingMethod="dynamic" newTab="false" githubPath="../ServiceCatalog/Default Catalog Offerings.htm#h3_Vendor_Onboarding_Request" href="../ServiceCatalog/Default Catalog Offerings.mdx"} for details on the form fields.
5. Click **Review & Submit**:
   - If the Vendor details are correct, click **Submit**.
   - If further edits are required, click **Edit**, make required changes and re-submit.

When submitted a new Vendor record is created. To edit the record, log in to \{\{Primary.Product}} as **GRC Manager**, and open the **Vendor** workspace to edit the record.

## Edit a Vendor

Edit existing Vendor records, for example: Edit Vendor Request status from **On Hold** to **Good Standing**.

**To edit a Vendor:**

1. Open the **Vendor** workspace.
2. Select the Vendor record you want to edit, and then double-click the entry.
3. Edit the information as required.
4. Click **Save**.
