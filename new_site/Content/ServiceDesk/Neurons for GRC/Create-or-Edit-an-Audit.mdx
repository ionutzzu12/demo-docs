Conduct an Audit for review of compliance related to an industry standard or key Configuration Items.

<hint type="info">
  To create or edit an Audit, you must log in to \{\{Primary.Product}} as a **GRC Manager**.
</hint>

**To create an Audit:**

1. Open the **Audit** workspace.
2. Click **New GRC Audit** to open a blank **Audit** form.
3. Enter the information into the fields as required.
   - **Lead Auditor Email** auto-populates based on the **Lead Auditor** field.
   - **Status** auto-populates as **New**.
   - **Priority** auto-calculates based on **Probability** and **Impact**.

<hint type="info" indent="1">
  Mandatory fields are marked with an asterisk.
</hint>

4. Click **Save**.
5. In the **Details** tab, enter the information into the fields as required.
6. Use the **Participants**, **Assets**, **Linked Audits**, **Mitigation Plans**, **Authority Documents**, and **Risk Assessments** tabs to link associated records to the Audit.
   - Use the **Exceptions** tab to create new Exceptions, or click (**Go to**) to fully open an existing Exception record associated with an Audit.

<hint type="info" indent="1">
  When creating a new Exception from within an Audit record, the **Exception Type** and **Audit** fields are automatically populated in the Exception record form. You should not edit these fields.
</hint>

<Paragraph listStyleType="disc" indent="2">
  If the **Recurring Audit** checkbox is selected in the **Details** tab, the **Scheduled Audits** tab displays the recurring Audits.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  Use the **Tasks** tab to create new Tasks.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  Use the **Controls** tab to create new Controls or link to existing Controls.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  Use evidence requests to support the Audit. To create a new Evidence record, click **New Evidence** in the **Evidence** tab, or to create a batch of Evidence records, click **New Evidence Record(s)** at the top of the **Audit** form.
  You can also create a new Task from within an Evidence request record.
</Paragraph>

<Paragraph listStyleType="disc" indent="2">
  Use the **Audit Findings** tab to document evidence, audit conclusion, and response.
</Paragraph>

<hint type="info" indent="1">
  You can restore hidden tabs using the **plus** sign (to the right of the tabs).
</hint>

7. Click **Save** and then **Refresh**.

<hint type="info" indent="1">
  When the status is set to **Approving**, an email is automatically sent to the Approver, and a record of the approval request appears in the **Approvals** tab.
</hint>

8. When the Audit is approved, you can change the **Status** to **Active**.
9. Click **Save**.

<hint type="info" indent="1">
  Click **Refresh** If changes you made or relevant tabs are not shown in the record after you have saved it.
</hint>

**Notes on Status:**

- When an Audit record's status is **Closed**, **Completed** or **Canceled**, all associated Evidence records status is set to **Closed**.
- The **New Evidence Record(s)** button is enabled only when the Audit record status is set to **Evidence Requested**, **Approving** or **Active**, for all other statuses, the **New Evidence Record(s)** button is disabled.

## Edit an Audit

**To edit an Audit:**

1. Double-click the Audit to open the details.
2. Edit the information as required.
3. Click **Save**.
