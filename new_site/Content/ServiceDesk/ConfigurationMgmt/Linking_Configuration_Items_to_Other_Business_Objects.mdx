## Creating an Incident, Problem, Change, or Release Record

You can create a new business object record within the **CI** workspace, automatically linking it to the configuration item.

1. Log in to the application as a Configuration Manager.
2. Open the **CI** workspace.
3. Open the configuration item you for which to create the record. The details page appears.
4. Select the corresponding tab (Incident, Problem, Change, or Release), then click **New&#x20;*****object*** or click the drop-down list to select one of the templates. The **New&#x20;*****object*** form appears. When using a template, some fields contain default values. See [Using Default Business Object Templates](../../Configure/BusinessObjects/Default_Templates.mdx) for more information on each template.

5. Enter information into the fields. See [Managing Incidents](../Incidents/Managing-Incidents.mdx), [Creating a Change Request](../Change/Creating-a-Change-Request.mdx), [Working with Problems](<../Problem/Creating a Problem.mdx>), or [Creating a Release Request](../ReleaseManagement/Creating-a-Release-Request.mdx) for more information on the fields.
6. Click **Save** to save your changes to the business object record. Click **Save**.
   - The new record appears under the corresponding tab.
   - The new record appears in the corresponding workspace.
   - Change Managers can see and edit configuration item records linked to change records.
   - Problem Managers can see and edit configuration item records linked to problem records.
   - Release Managers can see and edit configuration item records linked to release records.

## Linking to an Incident, Problem, Change, or Release Record

You can link an existing business object record to a configuration item within the **CI** workspace.

1. Log in to the application as a Configuration Manager.
2. Open the **CI** workspace.
3. Open a configuration item. The details page appears.
4. Select the corresponding tab (incident, problem, change, or release).
5. Click **Link&#x20;**:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/3aNmOvuWqAurGzGYP6YMW_link.png" alt caption size="10" width="18" height="11"}. A list of object records appears.
6. Select the record to link, then click **Select**. The linked record appears in the list.
7. Click **Save**.

  The number of linked records appears in the tab name.
</Paragraph>

## Unlinking from a Business Object Record

1. Log in to the application as a Configuration Manager.
2. Open the **CI** workspace.
3. Open a configuration item. The details page appears.
4. Select the record from the corresponding tab.
5. Click **Unlink** :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/QAMJj7VdVctxsHoLLCgEG_unlink.png" alt caption size="10" width="18" height="15"}.

The record no longer appears on the list.

## Viewing Linked Record Details

You can view the linked record within the corresponding workspace and modify the data. For example, you can update the status of a change record.

1. Log in to the application as a Configuration Manager.
2. Open the **CI** workspace.
3. Open a configuration item. The details page appears.
4. Select the corresponding tab (incident, problem, change, or release).
5. Select the record to view from the list, then click **Go To**.

The corresponding workspace opens showing the record details.
