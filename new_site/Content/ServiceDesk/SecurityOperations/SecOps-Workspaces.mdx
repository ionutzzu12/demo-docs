The workspaces for Security Operations in \{\{Primary.Product}} cater to the different Security Operations roles and functions, each workspace has a different function. The workspaces visible to you and accessible by you are configured by your organization's \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}} or Security Administrator based on your user role and responsibilities.

To know more about each workspace, click on the respective link below:

- [Security Incidents](SecOps-Incidents.mdx) - Security breaches and vulnerabilities are reported as Security Incidents. Based on the category, automated workflows are triggered.
- [Security Requests](SecOps-Requests.mdx) - Employees can raise security-specific requests which will be fulfilled by the Security Operations team.
- [Security Announcements](SecOps-Announcement.mdx) - The Security Operations team can send out announcements to the organization which will be displayed in the Self Service portal.
- [External Task](SecOps-External-Task.mdx) - Tasks created and assigned to external teams.
- [Security Knowledge Base](SecOps-Knowledge-Base.mdx) - The Security Operations team can create knowledge articles and publish to the Self Service portal.
