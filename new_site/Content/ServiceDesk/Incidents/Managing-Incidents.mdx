Managing incidents include the following functionalities:

- Creating incidents
- Viewing incidents
- Modifying incidents

<ExpandableHeading>
  ## Creating an Incident

  You can create an incident in several ways:

  - Using a blank form in the **Incident** workspace. Refer to :Link[Creating an Incident Using a Blank Form]{label="Creating an Incident Using a Blank Form" overridedLabel="Creating an Incident Using a Blank Form" spaceId docId="crTb5NPQ-I8esrvjKIssw" version="v2" docAnchorId="#Creating_an_Incident_Using_a_Blank_Form" loadingMethod="dynamic" newTab="false" href="Managing-Incidents.mdx"}.
  - Cloning an existing incident in the **Incident** workspace. Refer to :Link[Creating an Incident by Cloning an Existing Incident]{label="Creating an Incident by Cloning an Existing Incident" overridedLabel="Creating an Incident by Cloning an Existing Incident" spaceId docId="crTb5NPQ-I8esrvjKIssw" version="v2" docAnchorId="#Creating_an_Incident_by_Cloning_an_Existing_Incident" loadingMethod="dynamic" newTab="false" href="Managing-Incidents.mdx"}.
  - Using a template in the **Incident** workspace.  When you use a template, the application automatically fills in certain information, depending on the template that you use. You can do either of the following:
    - Create an incident and then apply the template to it. Refer to :Link[Creating an Incident By Using a Template]{label="Creating an Incident By Using a Template" overridedLabel="Creating an Incident By Using a Template" spaceId docId="crTb5NPQ-I8esrvjKIssw" version="v2" docAnchorId="#Creating_an_Incident_by_Using_a_Template" loadingMethod="dynamic" newTab="false" href="Managing-Incidents.mdx"}.
    - Apply a template to an existing incident. Refer to :Link[Applying a Template to an Existing Incident]{label="Applying a Template to an Existing Incident" overridedLabel="Applying a Template to an Existing Incident" spaceId docId="crTb5NPQ-I8esrvjKIssw" version="v2" docAnchorId="#Applying_a_Template_to_an_Existing_Incident" loadingMethod="dynamic" newTab="false" href="Managing-Incidents.mdx"}.
  - For an external contact. Refer to :Link[Creating an Incident for an External Contact]{label="Creating an Incident for an External Contact" overridedLabel="Creating an Incident for an External Contact" spaceId docId="crTb5NPQ-I8esrvjKIssw" version="v2" docAnchorId="#Creating_an_Incident_for_an_External_Contact" loadingMethod="dynamic" newTab="false" href="Managing-Incidents.mdx"}.

  <ExpandableHeading>
    ## Creating an Incident Using a Blank Form

    1. Open the **Incident** workspace.
    2. From the toolbar, click **New Incident**. 

    <Paragraph indent="1">
      The blank form opens.
    </Paragraph>

    *Creating a New Incident Using a Blank Form*

    <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/7RklHvlt67EYwrZ9V0uYE_incidentblank.png" alt="Creating a new incident using a blank form" position="flex-start" size="40" initialPath="../../Resources/Images/help-images/incident_blank.png" githubPath="Content/Resources/Images/help-images/incident_blank.png" showCaption="false"></Image>

    3. Enter information into the fields. Refer to :Link[Viewing an Incident]{label="Viewing an Incident" overridedLabel="Viewing an Incident" spaceId docId="crTb5NPQ-I8esrvjKIssw" version="v2" docAnchorId="#Viewing_an_Incident" loadingMethod="dynamic" newTab="false" href="Managing-Incidents.mdx"}.

    <hint type="info">
      If the customer is not in the database, create a new customer record.
    </hint>

    <hint type="info">
      If you are creating the incident for an end user, set the status to active. When the incident is submitted by the user or through email, the status is logged.
    </hint>

    4. Set the status to Active for incidents submitted by end users
    5. Click **Save**.
       - The application notifies the end user that the incident has been logged.
       - The application notifies both the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Analyst-Service-Desk-Analyst}} and the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Manager}} about the incident assignment.
    6. Click **Refresh** to view the resolution target values. Refer to [About Resolution Details](<../../Configure/Escalations/Resolution Details.mdx>).
  </ExpandableHeading>

  <ExpandableHeading>
    ## Creating an Incident for an External Contact

    1. Log into the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Console}} console.
    2. Open the **External Contact** workspace.

    <Paragraph indent="1">
      A list of external contacts opens.
    </Paragraph>

    3. Open the external contact record.
    4. Click the **Incident** tab.
    5. Click **New Incident** then select a template.

    <Paragraph indent="1">
      The **Edit Incident** window opens.
    </Paragraph>

    6. Enter the required information. Each template has various fields auto-filled with data.
    7. Click **Save**.
  </ExpandableHeading>

  <ExpandableHeading>
    ## Creating an Incident Using a Template

    For a list and description of the incident templates, refer to [Default Incident Templates](Default-Incident-Templates.mdx).

    1. Log into the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Console}} console.
    2. Open the **Incident** workspace.
    3. Click **New Incident** > **Templates**.
    4. Select a template that matches the incident.

    A incident form opens.

    5. Enter any additional information.
    6. Click **Save**.

    - The system notifies the end user, \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Analyst-Service-Desk-Analyst}} and \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Manager}} about the incident assignment.
  </ExpandableHeading>

  <ExpandableHeading>
    ## Creating an Incident by Cloning an Existing Incident

    Log into the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Console}} console.

    1. Open the **Incident** workspace and open an existing incident.
    2. From the Action Menu, select **Form Actions** > **Clone Incident**.
    3. Select the incident status from the dropdown and click **Ok**.
       - The system creates a copy of the incident.
       - Child business objects (tasks, journal emails) are not copied.
       - Fields override only as defined by the quick action.
    4. Update other fields as required.
    5. Click **Save**.

    - The application notifies the end user that the incident has been logged.
    - The application notifies both the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Analyst-Service-Desk-Analyst}} and the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Manager}} about the incident assignment.
  </ExpandableHeading>

  <ExpandableHeading>
    ## Applying a Template to an Existing Incident

    When you apply a template to an existing incident, the system overwrites any existing data.

    For a list and description of the incident templates, refer to [Default Incident Templates](Default-Incident-Templates.mdx).

    1. Log into the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Console}} console.
    2. Open an incident.
    3. Perform one of the following:
       - From the Action Menu, select **Templates**.
       - From the toolbar, click **Apply Template**.
    4. Select a template from the list. The system auto fills information for **Category**, **Service**, **Impact**, **Urgency**, and **Summary** fields.
    5. Enter remaining data.
    6. Click **Save**.

    - The system notifies the end user, \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Analyst-Service-Desk-Analyst}} and \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Manager}} about the incident assignment.
  </ExpandableHeading>
</ExpandableHeading>

<ExpandableHeading>
  ## Modifying an Incident

  You can modify incident values or update the incident to reflect status changes. The fields you can edit depend on your role and permissions.

  1. Log into the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Console}} console.
  2. Open the **Incident** workspace and open an incident to modify.
  3. Change the information as required.
  4. Click **Save**.
