Only \{\{FRS\_VARS.Neurons\_ITSM\_-\_Release\_Manager}}s and \{\{FRS\_VARS.Neurons\_ITSM\_-\_Change\_Manager}}s can create releases by using the release form, template on the release form, or within a change form.

- If \{\{FRS\_VARS.Neurons\_ITSM\_-\_Change\_Manager}}s create a release, they can create the release record and set it to draft status while gathering requirements or formalizing plans and details for the release. When they are ready to submit the release record, they update its status to requested. They often create a release while working on a change.
- If \{\{FRS\_VARS.Neurons\_ITSM\_-\_Release\_Manager}}s create a release, they can set it to any state and work on the release. They also work on requested releases and update their status to active.

<hint type="info">
  Depending on your role, you might have access to certain fields and values in the release module. The \{\{FRS\_VARS.Neurons\_ITSM\_-\_Release\_Manager}} has full permissions to work with the **Release** business object. The default application is configured to allow the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Change\_Manager}} and \{\{FRS\_VARS.Neurons\_ITSM\_-\_Release\_Manager}} to create releases. The \{\{FRS\_VARS.Neurons\_ITSM\_-\_Change\_Manager}} has access to a subset of the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Release\_Manager}}'s permissions to work on the **Release** business object.
</hint>

1. Sign in to \{\{FRS\_VARS.ProductName}} as a \{\{FRS\_VARS.Neurons\_ITSM\_-\_Release\_Manager}} or a \{\{FRS\_VARS.Neurons\_ITSM\_-\_Change\_Manager}}.
2. Open the **Release** workspace.
3. Select **New Release** in the toolbar to open a blank form, or select a template from the drop-down list. ## Applying a Template to an Existing Release

<hint type="info">
  If you have already filled in some of the fields, they are overwritten when you apply the template. If the release type is major, the different combinations of scope and phase, as defined in the templates, automatically create milestones.
</hint>

1. In the **Release** workspace, open a release record.
2. Select **Action Menu** **>** **Templates**.

<hint type="info" indent="1">
  If you selected a template for a major release type, depending on the type of template you selected, \{\{FRS\_VARS.ProductName}} creates milestones for you. See [Working with Milestones](Working-with-Milestones/Working-with-Milestones.mdx). For all other types of release, you have the option of creating milestones.
</hint>

3. From the list of available templates, select a template from the list to apply it to the current release.

<hint type="info" indent="1">
  If you entered information in any of the fields, they are overwritten by the default values from the template that you selected. See :Link[About Release Milestones]{label="About Release Milestones" overridedLabel="About Release Milestones" spaceId docId="MX8V4eaqEViwUACgknNmT" version="v2" docAnchorId="#Best" loadingMethod="dynamic" newTab="false" githubPath="Working-with-Milestones/Working-with-Milestones.htm#Best" href="Working-with-Milestones/Working-with-Milestones.mdx"} for information on the milestones that are created when you select some of the available templates.
</hint>

4. Enter information in the relevant fields.
5. Select **Save**.

<hint type="info" indent="1">
  The release is in draft state until you request it. If you are the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Release\_Manager}}, you can update the state to active if you are ready to start working on it and create milestones for the release.
</hint>

## Requesting a Release

A new release request is set to draft. You can leave it in this state while gathering information. When you are ready to submit the release request, set the status to requested.

As a \{\{FRS\_VARS.Neurons\_ITSM\_-\_Change\_Manager}}, you might have already linked to a release or created a release from a change.

If you created a new release, you must request it.

<hint type="info">
  Follow this procedure to request a release that you created. After you have saved the release in requested status, you cannot make any other changes to it unless you have Release Manager permissions. The Release Manager can update the release from draft to active, or any other available state.
</hint>

1. In the **Release** workspace, open the release record.
2. Select **Quick Action** **>** **Request Release**.
   The status of the release is set to requested.
3. Select **Save**.

<Paragraph indent="1">
  A notification of the request is sent to the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Release\_Manager}}.
</Paragraph>
