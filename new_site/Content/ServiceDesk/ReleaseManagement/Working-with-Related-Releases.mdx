You can link or create releases that are related to the current release.

- If the release scope is master, link to or create the child releases.
- If the release scope is phase, link to the master release.

Related releases may not be necessary if you are creating a release that has a scope of standalone.

### Linking to a Related Release

1. Sign in to \{\{FRS\_VARS.ProductName}} as a \{\{FRS\_VARS.Neurons\_ITSM\_-\_Release\_Manager}}.
2. Open the **Release** workspace.
3. Open the release record for which to link a related release.
4. From the **Related Releases** tab, select **Link** from the toolbar.
5. Select the record to link and click **Select**.
6. Select **Save**.

<hint type="info" indent="1">
  The record is linked to the current release record and appears under the **Related Release** tab.
</hint>

### Creating a Related Release

1. Sign in to the application as a Release Manager.
2. Open the **Release** workspace.
3. Open the release record for which to link a related release.
4. From the **Related Releases** tab, select **New**.
   The **New Release** window opens.
5. Enter information into the fields. See [Creating a Release Request](Creating-a-Release-Request.mdx).
6. Select **Save**.
