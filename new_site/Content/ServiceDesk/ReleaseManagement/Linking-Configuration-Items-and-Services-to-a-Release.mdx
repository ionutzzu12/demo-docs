title: Linking Configuration Items and Services to a Release
docTags: 
createdAt: Tue Jun 30 2026 17:16:48 GMT+0300 (Eastern European Summer Time)
---

Affected configuration items and the impacted services affected by a change are directly linked to the change from the change record. When the change is linked to a release, the **Search** and **Link** quick action business rules automatically link all associated configuration items and services from the change to the release.

However, if additional configuration items and services are linked to the change after the change has been linked to the release, use the **Search** and **Link** quick actions to move the additional configuration items and services from the change to the release record.

For some releases, for example emergency releases, for which the records are created after-the-fact, you might be linking configuration items and services that are affected by the release directly to the release record.

1. Sign in to \{\{FRS\_VARS.ProductName}} as a \{\{FRS\_VARS.Neurons\_ITSM\_-\_Report\_Manager}}.
2. Open the release record to which you want to link affected configuration items and services.
3. From the Action Menu, choose **Search and Link > Link CIs from related Changes**.

<Paragraph indent="1">
  The configuration items from the change linked to the release appear in the **CI** tab of the release record.
</Paragraph>

4. From the Action Menu, choose **Search and Link > Link Services from related Changes**.

<Paragraph indent="1">
  The services from the change linked to the release appear in the **Service** tab of the release record.
</Paragraph>

5. Select **Save**.
6. (Optional) To disassociate a configuration item or a service that is no longer part of a change or impacted by a change, select the record in the **CI** or **Service** tab of the release record and select **Unlink**.
