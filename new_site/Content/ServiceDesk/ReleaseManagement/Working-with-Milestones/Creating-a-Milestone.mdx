If you use a template to create a major release, milestones are automatically created for you in the **Milestones** tab and in the **Schedule** tab of the release record.

You can create other milestones in addition to the defined ones or create new milestones for a release type other than major.

To create a milestone using the **Schedule** tab, see [Managing a Release Project using the Gantt Chart](Creating-a-Gantt-Chart.mdx). When you create a new milestone, it does not have any defined workflows or approvals associated with it. Work with your administrator to create a workflow for your milestone if you require an approval process for the new milestone.

1. Sign in to \{\{FRS\_VARS.ProductName}} as a \{\{FRS\_VARS.Neurons\_ITSM\_-\_Report\_Manager}}.
2. Open the release for which you want to create milestones.
3. Select **New Milestone** on the milestone toolbar.
4. Enter information into the fields.

5. Repeat steps 2 to 4 above to add milestones.
6. Set the milestone status to active.

<hint type="info" indent="1">
  When you update the status to active, tasks and checklists associated with the milestone appear and task notifications are sent to the appropriate task owner. To go to the parent release form from a milestone record, select **Open in Parent** on the toolbar.
</hint>

7. Select **Save**.
   - A milestone record is created and remains in pending state until you are ready to have it worked on by the owner.
   - The milestone workflow triggers only when the status is updated to active, if it is defined for the major release.

## Modifying a Milestone

You can add additional information to a milestone before sending it to the assigned owner. When you are ready to have it worked on, update the status to active.

1. Sign in to \{\{FRS\_VARS.ProductName}} as a \{\{FRS\_VARS.Neurons\_ITSM\_-\_Release\_Manager}}.
2. Open the **Milestones** workspace.
3. Open the milestone record to work on.
4. Edit the required information.
5. Select **Save**.
