---
title: Creating a New Milestone Using a Gantt Chart
docTags: 
createdAt: Tue Jun 30 2026 17:16:48 GMT+0300 (Eastern European Summer Time)
---

You can create ad-hoc milestones using either a Gantt chart or from the **Milestones** tab of a release record. The following procedure is for creating milestones without using a release template. If you use a release template to create a release, the system displays the defined milestones that are associated with the release template under the **Milestones** tab.

1. Sign in to \{\{FRS\_VARS.ProductName}} as a \{\{FRS\_VARS.Neurons\_ITSM\_-\_Release\_Manager}}.
2. Open the **Release** workspace.
3. Open a release record.
4. Select the **Schedule** tab. The system displays the default schedule view, showing a 10 week period.
5. Select **Add** **Milestone**.
   The **New Milestone** dialog box opens.
6. Enter information into the fields.

| Field Name                 | Description                                                                                                                                                                                                                                                                                                         |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Subject                    | Name of the milestone.                                                                                                                                                                                                                                                                                              |
| Description                | Details about the milestone.                                                                                                                                                                                                                                                                                        |
| Sequence Order             | The sequence number for the milestone. No workflows are associated with the sequence order if you are creating an ad-hoc milestone; the sequence number is only for your reference. The sequence number in the template-based milestones indicates the sequence that the milestones activate in.                    |
| Release                    | The release associated with the milestone.                                                                                                                                                                                                                                                                          |
| Release Number             | The release number that is auto-generated, based on the release record.                                                                                                                                                                                                                                             |
| Status                     | Status of the milestone. The status of a new milestone is set to **Pending**. No workflows are associated with the status value when creating an ad-hoc milestone; the status is only for your reference. The Status field in the template-based milestones indicates the sequence that the milestones activate in. |
| Owner                      | Name of the milestone owner. The default owner is the logged-in user.                                                                                                                                                                                                                                               |
| Environment                | The environment of the milestone. For example, Dev, Production, QA.                                                                                                                                                                                                                                                 |
| Started On                 | The start date and time. The default value is the current date and time.                                                                                                                                                                                                                                            |
| Target End Date            | The targeted end date of the milestone. If this field is blank, the milestone duration shifts to encompass the end date of the last task. If you open the milestone form, the system populates the target end date of the last task.                                                                                |
| Actual End Date            | The actual end date of the milestone.                                                                                                                                                                                                                                                                               |
| Percent Complete           | The percentage completed.                                                                                                                                                                                                                                                                                           |
| Estimates Cost             | The estimated cost of the milestone.                                                                                                                                                                                                                                                                                |
| Cost ($)<br />Time (Mins.) | The cost and time of the milestone that is auto-generated by the system.                                                                                                                                                                                                                                            |

7. Select **Save**. The new milestone is displayed on the Gantt chart.

<hint type="info" indent="1">
  (Optional) You can manually extend a milestone duration by selecting the duration column and updating the number of days. To align the milestone to the duration of the tasks associated with it, select **Align Milestone**.
</hint>
