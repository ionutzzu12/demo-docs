<hint type="info">
  The release workflows described here reference the out-of-the-box application. You can work with your administrator to create or modify the workflows to meet your business and organizational needs.
</hint>

Only major releases require approval; therefore, \{\{FRS\_VARS.ProductName}} has defined five different workflows to accommodate the various combinations of major releases.

- Master feature-based release workflow
- Master location-based release workflow
- Phase feature-based release workflow
- Phase location-based release workflow
- Standalone release workflow

Each release workflow contains defined milestones that are created when you create a major type of release using a release template. See [About Release Milestones](Working-with-Milestones/Creating-a-Milestone.mdx) for the list of milestones that are created based on the template you select, and see [Working with Milestones](Working-with-Milestones/Working-with-Milestones.mdx) for information about the milestone workflows.

The first milestone defined for a specific release starts when the status of the release is updated to active. When the Release Review Board approves the milestone, the status of the milestone is set to completed and the next milestone in the sequence starts. After each milestone defined for the release workflow is approved and its status is updated to completed, the release status is updated to closed.

When the trigger conditions are met, the workflow defined for those conditions is started. For this release, there are four milestones defined. Each milestone has its own workflow that runs. When the first milestone is approved and updated to completed status, the next one is invoked. After the final milestone is approved and updated to completed status, the release is closed.

## Viewing the Workflow Instance for a Release

To view the progress of a release record in the workflow when it is sent for approval, select the **Workflow Instance** tab of a release record.

Viewing the workflow instance for a release that is pending approval lets you know if the approval for the release has been voted on or is in waiting state.

1. Sign in to \{\{FRS\_VARS.ProductName}} as a \{\{FRS\_VARS.Neurons\_ITSM\_-\_Release\_Manager}}.
2. Open the **Release** workspace.
   A list of releases is displayed.
3. Double-click the release record for which to view the workflow.
4. Select **View Workflow** in the toolbar.
   The related workflows open.
