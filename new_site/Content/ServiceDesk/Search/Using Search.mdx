The search feature is available in most workspaces to help you easily find records. Advanced searches can include expressions to fine-tune your search criteria. You can create a saved search, keeping frequently used criteria handy, and even designate a saved search as a favorite.

<hint type="info">
  You cannot use HTML or other scripting characters such as "\<" in the search feature. Using these characters results in error messages.
</hint>

The search area has three main parts:

- The **All** link, displays the list of saved searches. You can choose to apply any of the saved searches that are available for your user role. Saved search allows you to a create custom search and save it for repeated use. Refer to [Using a Saved Search](<Saved Searches.mdx>). You can also create a new saved search by clicking the create new saved search icon on the right.
- **Show Filters** displays the available filter options. This option enhances the search functionality. Refer to [Working with the Show Filter Search](<Using Show Filters.mdx>).
- The **Simple Text Search** text box performs a simple text-based search based on the keywords entered in the search box. You can also search text based on the localized value. Refer to [About the Different Search Types](<Text Searching.mdx>).

Other search features include the following:

- **Related Items**: Available in the Self-Service Portal, this search automatically shows records related to the current issue. Refer to [Using Related Items and Object Matching](<Related Items.mdx>).
- **Record Search**: Users can search across related business objects. Refer to [Using Record Search](<Using Record Search.mdx>).
- Administrators can enable type ahead, full-text indexed, and partial word options for specific layouts.
