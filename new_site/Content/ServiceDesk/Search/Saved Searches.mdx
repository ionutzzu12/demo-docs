Use saved searches to save the defined search criteria. This allows you to quickly perform a search without needing to manually enter all of the search conditions each time. You can create, modify, and delete a saved search.

You can view all of the saved searches available to your role and those applicable to the current workspace.

<hint type="info">
  For information on how to optimize the saved search query for faster performance, refer to the **Improving Saved Search Query Performance** chapter in the \{\{FRS\_VARS.ITSM}} Performance Tuning Guide.
</hint>

## Viewing the List of Saved Searches Available for Your Role

1. Log in to \{\{FRS\_VARS.ProductName}}.
2. Open a workspace.
3. Click the drop-down arrow of the **All** button. The application displays the list of saved search available for that workspace and for the role.

## Creating a Saved Search

\{\{FRS\_VARS.ProductName}} provides an assortment of default saved searches based on your user role. You can also create your own search and add it to the saved searches list.

<hint type="info">
  You can only create saved searches for personal use if your role has create permissions. If your role has edit permissions, you can edit saved searches created by other users and publish saved searches to other roles. Permissions for saved searches are defined by the administrator in the application settings for each role.
</hint>

1. Log in to \{\{FRS\_VARS.ProductName}}.
2. Open a workspace and perform any of the following action.
   - Click the **Favorite Searches** drop-down arrow and click the **New Saved Search** option.
   - Alternatively, you can also create a saved search by clicking the **New Saved Search** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/mkLzCNnMihNrR2UjgHBtP_newsavedsearchicon.png" alt caption size="10" width="18" height="15"} in the workspace area.

<hint type="info">
  The **New Saved Search** icon only appears if your role has create or edit permissions.
</hint>

3. Configure the search as per your needs. Refer to [Using Additional Search Features](<Search Features.mdx>).
4. Click the **Settings** :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/LWj_gWbmGeL6BvIXiETNe_savedsearchsettingsicon.png" alt caption size="10" width="60" height="16"} icon.
5. In the **Saved Search Settings** dialog box, enter the following information:

|                   | Field             | Description                                                                                                                                                                                                                                                                                                        |
| ----------------- | ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Sharing           | Publish action to | The roles to which to publish this saved search. Select from the drop-down list. This saved search is available for all users in those roles.                                                                                                                                                                      |
|                   | Category          | Specifies if the search is on the saved search list or on a dashboard. Category is a way to group searches into meaningful collections.                                                                                                                                                                            |
| Defaults          | Personal Default  | Check **Personal Default** to make this saved search the default when you (the logged-in user) open this specific workspace. This search automatically executes and the results appear in the list when you open this workspace. Uncheck **Personal Default** if you do not want to make this your default search. |
|                   | Default for roles | The roles for which this saved search is the default. Choose the roles from the drop-down list. You can only designate roles to which you have write permissions.                                                                                                                                                  |
| Permissions<br /> | Edit              | Check box indicates the permission set for the user to edit Saved Search                                                                                                                                                                                                                                           |
|                   | Delete            | Check box indicates the permission set for the user to delete Saved Search.                                                                                                                                                                                                                                        |

<hint type="info" indent="1">
  If you only have create permissions, you can create and edit the saved search that you created. You cannot publish to your role or other roles.

  If you have edit permissions, you can publish to your role or other roles.
</hint>

6. Click **OK** to save the settings.
7. Click **Save as...** give your search a name and then click **Save**.

<Paragraph indent="1">
  The application saves the search and displays it on your list of saved searches.
</Paragraph>

<hint type="info">
  To apply a saved search for the Mobile Analyst role, click the :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/TwI40C6egRVISqRRFFyMZ_setasfavorite.png" alt caption size="10" width="55" height="14"} icon to set the saved search as favorite. Saved searches that are not set as favorite do not appear in the Mobile Analyst portal.
</hint>

## Viewing the Dependencies of a Saved Search

Saved searches that are deleted or modified might affect the display of data in a dashboard. You can view the dependencies associated with each saved search to determine if a dashboard is affected.

1. Log in to \{\{FRS\_VARS.ProductName}}.
2. Open a workspace.
3. Click the **Favorite Searches&#x20;**&#x64;rop-down arrow. The application displays the list of saved searches available for that workspace and for the role.
4. To see the entire list, click **All Saved Searches**.
5. Hover the cursor over any Search Name, If you have the correct permissions, four options appear: **Edit**, **Clone**, **Dependency**, and **Delete**. Click **Dependency**.

<Paragraph indent="1">
  The application displays a **Dependency** dialog box with a list of dashboard items that depend on this search.
</Paragraph>

6. Click **Close**.

## Modifying a Saved Search

Do not modify any default saved searches, as this could cause unexpected results due to dependent processes.

Before modifying a saved search, check for any affected dependencies.

You can modify or publish a saved search to any role if you have edit permissions. In this case, the **edit** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/pUcDCwFqa9aQFPQhBx47U_edit.png" alt caption size="10" width="15" height="16"} appears next to the saved search. Permissions are set by your administrator in the system settings for your role.

1. Log in to \{\{FRS\_VARS.ProductName}}.
2. Open a workspace.
3. Click the **Favorite Searches** drop-down arrow. The application displays the list of saved search available for that workspace and for the role.
4. To see the entire list, click **All Saved Searches**.
5. Hover the cursor over any Search Name, if you have the correct permissions, four options appear: **Edit**, **Clone**, **Dependency**, and **Delete**.

6. Click the **edit** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/pUcDCwFqa9aQFPQhBx47U_edit.png" alt caption size="10" width="15" height="16"}, then change any of the fields and search criteria.

## About Saved Searches and the Favorites List

Saved searches that are marked as favorites appear on the saved searches list with an orange star next to their name. Saved searches that are not favorites have a gray star.

The administrator establishes the favorite saved searches for all users, but individual users can manage their own list of favorites. You can add a saved search to the favorites list and you can remove a search from the favorites list. Removing a saved search from the favorites list does not delete the saved search.

The application shows all saved searches that are marked as a favorite in the saved search drop-down list.

## Removing a Saved Search from the Favorites List

1. Log in to \{\{FRS\_VARS.ProductName}}.
2. Open a workspace.
3. Click the **Favorite Searches** drop-down arrow. The application displays the list of saved search available for that workspace and for the role.
4. To see the entire list, click **All Saved Searches**.
5. Hover the cursor over the orange star of Search Name you want to remove from the Saved Search from the favorite list.
6. Click **Remove Favorite**.

## Adding a Saved Search to the Favorites List

1. Log in to \{\{FRS\_VARS.ProductName}}.
2. Open a workspace.
3. Click the **Favorite Searches** drop-down arrow. The application displays the list of saved search available for that workspace and for the role.
4. To see the entire list, click **All Saved Searches**.
5. Hover the cursor over the gray star on the Search Name you want to add to favorites list.
6. Click **Add as Favorite**.

## Deleting a Saved Search

You can delete a saved search if you have delete permissions. In this case, a **Delete** icon appears next to the saved search. Permissions are set by your administrator in the application settings for your role.

Before deleting a saved search, check for any affected dependencies. 1. Log in to \{\{FRS\_VARS.ProductName}}.
2. Open a workspace.
3. Click the **Favorite Searches** drop-down arrow. The application displays the list of saved search available for that workspace and for the role.
4. To see the entire list, click **All Saved Searches**.
5. Hover the cursor over the Search Name you want to delete. If you have the correct permissions, the application displays the :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/USPl9Egu4Y2Blx7HKC3mA_deleteredx.png" alt caption size="10" width="14" height="13"} **Delete** icon.
6. Click the **delete** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/USPl9Egu4Y2Blx7HKC3mA_deleteredx.png" alt caption size="10" width="14" height="13"}.
7. Click **Yes** in the confirmation window.

## Exporting a Saved Search

You can export the records found in a saved search to a Microsoft Excel (.xls) file. You must have permissions granted by the administrator to do this.

1. Log in to \{\{FRS\_VARS.ProductName}}.
2. Open a workspace.
3. Click the drop-down arrow of the **All** button. The application displays the list of saved search available for that workspace and for the role.
4. To see the entire list, click **All Saved Searches**.
5. Click the **Export to Microsoft Excel** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/SqZbfW9R3bm23gwVoKDOQ_exportexcel.png" alt caption size="10" width="16" height="16"} next to the saved search to export. You do not see the icon if your role does not have permissions.

<Paragraph indent="1">
  The application downloads the file to your local computer.
</Paragraph>

<hint type="info" indent="1">
  The Export to Excel feature on the Saved Search definition page does not support saved searches that include prompts. To export results to excel, run the saved search from the UI and use the **Quick Action** option.
</hint>

## Emailing a Saved Search

You can email the results of a saved search to yourself. You must have permissions granted by the administrator to do this.

1. Log in to \{\{FRS\_VARS.ProductName}}.
2. Open a workspace.
3. Click the drop-down arrow of the **All** button. The application displays the list of saved search available for that workspace and for the role.
4. To see the entire list, click **All Saved Searches**.
5. Click the **email** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/0ET1HS7bDcq3sxWQeSW8G_email.png" alt caption size="10" width="17" height="15"} for the saved search you want to send as an email. You will not see the icon if your role does not have permissions.

<Paragraph indent="1">
  The file is sent to the logged-in user's email address.
</Paragraph>

## Cloning a Saved Search

You can clone a saved search.

1. Log in to \{\{FRS\_VARS.ProductName}}.
2. Open a workspace.
3. Click the drop-down arrow of the **All** button. The application displays the list of saved search available for that workspace and for the role.
4. To see the entire list, click **All Saved Searches**.
5. Hover the cursor over the Search Name you want to clone. If you have the correct permissions, a **Clone** button appears.
6. Click **Clone**.
7. Click **Yes** in the confirmation window.

<Paragraph indent="1">
  The application adds a new saved search to the list. It has the same name as the saved search that you cloned, except that the new saved search has " - Copy" appended to the name. To change the name, edit the saved search.  See :Link[Modifying a Saved Search]{label="Modifying a Saved Search" overridedLabel="Modifying a Saved Search" spaceId docId="i-hArcXiCYKMAm89aYfDR" version="v2" docAnchorId="#ModifyingASearch" loadingMethod="dynamic" newTab="false" href="Saved Searches.mdx"}.
</Paragraph>
