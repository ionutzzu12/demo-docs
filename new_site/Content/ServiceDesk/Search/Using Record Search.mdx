Often with a form, you can link your record to another record. For example, you can link a configuration item to a change or problem.

You can also search records using a text search. Refer to :Link[About Simple Text Search]{label="About Simple Text Search" overridedLabel="About Simple Text Search" spaceId docId="kZBriSkNjCrRKE6tGEwm_" version="v2" docAnchorId="#About_Simple_Search" loadingMethod="dynamic" newTab="false" githubPath="Text Searching.htm#About_Simple_Search" href="Text Searching.mdx"} and :Link[About Type-Ahead Search]{label="About Type-Ahead Search" overridedLabel="About Type-Ahead Search" spaceId docId="kZBriSkNjCrRKE6tGEwm_" version="v2" docAnchorId="#Type_Ahead_Search" loadingMethod="dynamic" newTab="false" githubPath="Text Searching.htm#Type_Ahead_Search" href="Text Searching.mdx"}.

This relationship between the two records creates a status association as well as a cross-reference. For example:

- When an incident is linked to a problem, the incident status changes to waiting for resolution. If unlinked, the status reverts to the previous state. When the problem is closed, all linked incidents are resolved and the information is copied to the linked incidents and the affected customers are notified. If there is a workaround, notification is sent to the incident owner (such as the Service Desk Analyst), not the customers.
- When the problem to which an incident is linked is closed, all incidents linked to the problem are changed to resolved status.

To link one record to another:

1. Log in to \{\{FRS\_VARS.ProductName}}.
2. Open a workspace.
3. Open the record you want to link. For example, open a configuration item record from the **CI** workspace. The form appears with data in the fields.
4. Select a tab containing a link icon, then click the **link** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/3aNmOvuWqAurGzGYP6YMW_link.png" alt caption size="10" width="18" height="11"}. The record search window appears.

<Paragraph indent="1">
  If the values for the **Field** and the **Criteria** fields do not have the same data type (for example, if the value for **Field** is a Boolean and the value for **Criteria** is text), the application returns an error.
</Paragraph>

<hint type="info" indent="1">
  If you created a saved search prior to \{\{FRS\_VARS.ProductName}} Release 2014.3, and that saved search included values with different data types, you get an error if you run the saved search in \{\{FRS\_VARS.ProductName}} Release 2014.3. If this happens, contact your administrator. The administrator must open the saved search and click **Reset Default Search** to reset the search. This button is only available to administrators.
</hint>

6. Click the **add** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/JtIxGMfjslfCSitXWBu4o_addgreen.png" alt caption size="10" width="17" height="15"} to add another search line and enter more criteria. Repeat as needed.
7. To remove a search line, click the **delete** icon :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/USPl9Egu4Y2Blx7HKC3mA_deleteredx.png" alt caption size="10" width="14" height="13"}.
8. Click one of the following:
   - **Select**: Add the item to your record.
   - **Save Search as Default**: Automatically load your criteria into the record search window each time. This criteria appears throughout all objects.
   - **Reset Default Search**: Remove your default criteria.
   - **Add**: Create a new linked object record. For example, if you are linking an incident object, then a blank incident form appears. From here you can create a new incident record that is automatically linked to your parent record.
   - **Edit**: Modify a selected record. For example, if you have selected incident 10424, you can modify the information in this record from here (assuming you have privileges).
   - **Cancel**: Dismiss the window without linking the record.
9. Click **Save**.
