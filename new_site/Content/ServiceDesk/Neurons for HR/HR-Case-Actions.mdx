<ExpandableHeading>
  ## Duplicating an HR Case

  Sometimes you may need to split an HR Cases into two cases, in order to separate out different issues.

  1. Open the HR Case you want to split.
  2. Select **More** > **Form Actions** > **Duplicate HR Case**.

  After splitting, text is added automatically to the **Description&#x20;**&#x66;ield saying **This HR Case was split into 2 cases by \[user name]. The original Case ID is n**.

  <hint type="info">
    **Note:** All Case values are duplicated for the new HR Case and the Case number is automatically incremented.
  </hint>
</ExpandableHeading>

<ExpandableHeading>
  ## Escalating an HR Case to a Team

  1. Open the HR Case you want to escalate to another team.
  2. Select **More** > **Form Actions** > **Escalate to Team**.
  3. Select your chosen team from the drop-down list.
  4. Select **OK**.

  Th&#x65;**&#x20;Status** field is set to **Assigned&#x20;**&#x61;nd the HR Case is reassigned/escalated to your chosen team.
</ExpandableHeading>

<ExpandableHeading>
  ## Resolving an HR Case as Redundant

  Sometimes you may need to mark a case as redundant because it is a duplicate of another case that already exists.

  1. Open the HR Case you want to resolve as a duplicate.
  2. Select **More** > **Form Actions** > **Resolve Redundant**.
  3. Select the HR Case that your case is a duplicate of from the drop-down list.
  4. Select **OK**.

  The **Status&#x20;**&#x66;ield is set to **Closed&#x20;**&#x61;nd the **Resolution&#x20;**&#x68;as an automatic entry stating **Resolved as Redundant Case on \[date] by \[user name]. Primary Case Number: n**. The **Cause Code** is set to **Duplicate**.
</ExpandableHeading>

<ExpandableHeading>
  ## Setting an HR Case to Pending

  Sometimes an HR Case needs to be set to pending if, for example, you are waiting on information from the customer.

  1. Open the HR Case.
  2. Select **More** > **Form Actions** > **Set HR Case to Pending**.
  3. Select a reason why this case is pending from the drop-down list. Reasons include:
     - Customer is unavailable
     - Need information from customer
     - Needed for a future date
     - Waiting on request
  4. Select **OK**.

  The **Status** field is set to **Pending&#x20;**&#x61;nd the **Pending Reason** is set to your chosen reason.
</ExpandableHeading>
