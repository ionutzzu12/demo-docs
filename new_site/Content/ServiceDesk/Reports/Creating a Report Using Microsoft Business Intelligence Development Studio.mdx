### About Using Microsoft Business Intelligence Development Studio

This information explains how to create reports with \{\{FRS\_VARS.ProductName}} Cloud platform and how to use Microsoft Business Intelligence Development Studio (BIDS) to create reports directly from your desktop. You can access the BIDS documentation at [https://msdn.microsoft.com/en-us/library/ms173767(v=sql.105).aspx.](https://msdn.microsoft.com/en-us/library/ms173767\(v=sql.105\).aspx)

BIDS is the former integrated development environment (IDE) from Microsoft, which was used to develop data analysis and business intelligence solutions utilizing the Microsoft SQL Server Analysis Services, Reporting Services, and Integration Services.

BIDS is based on the Microsoft Visual Studio development environment, but customized with Microsoft SQL Server services-specific extensions and project types, including tools, controls and projects for reports, data flows, and data mining structure.

Report Managers and administrators can create their own customized report templates and then create reports from these custom report templates.

\{\{FRS\_VARS.ProductName}} provides default reports as a reporting solution that you can customize. If you have already customized some of your reports, you can download those reports from your own instance and update them in the solution.

### Creating a New Report with BIDS

Follow these steps to create a new report using the BIDS **New Report.rdl \[Design]** template:

1. Double click **New Report.rdl** in the **Solution Explorer**. The system displays the **NewReport.rdl \[Design]** template.

<Paragraph indent="1">
  *NewReport.rdl \[Design] Template*
</Paragraph>

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/If2sfVIZqrgyqTIbBSg5t_newnewreportdesign.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/New_New_Report_Design.png" githubPath="Content/Resources/Images/help-images/New_New_Report_Design.png" showCaption="false"></Image>

2. Expand **Data Source** on the left side. (If you do not see the **Report Data** panel on the left side of the screen, select **View > Report Data** to display the panel.)
3. Right-click **SaaSAppModel**.
4. Select **Add Dataset**.  The system displays the **Dataset Properties** dialog box.
5. Click **Query Designer...**. The system displays the **Query Designer** dialog box.

<Paragraph indent="1">
  *Query Designer*
</Paragraph>

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/cIrKv-3o_xGzWT8TJn0HC_querydesignersmall.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/Query_Designer_Small.png" githubPath="Content/Resources/Images/help-images/Query_Designer_Small.png" showCaption="false"></Image>

6. Use the **Query Designer** dialog box to create the Microsoft SQL query for this dataset.

<Paragraph indent="1">
  a. Click **Edit as Text** to create the **Microsoft SQL query**.
</Paragraph>

<Paragraph indent="1">
  b. Enter a query.
</Paragraph>

<Paragraph indent="1">
  c. Click **!** to verify the data.
</Paragraph>

<Paragraph indent="1">
  d. Click **OK** to run the query.
</Paragraph>

<Paragraph indent="1">
  e. Enter a meaningful name for the dataset and click **OK** to save the dataset.

### Editing an Existing Report with BIDS

If you make any changes to the report template parameters, you must recreate the corresponding reports.

To edit an existing report follow these steps:

1. Double click the report name from the **Solution Explorer**.
2. Edit the report by following the steps in :Link[Creating a New Report with BIDS]{label="Creating a New Report with BIDS" overridedLabel="Creating a New Report with BIDS" spaceId docId="WedIhvizA1vugfhN5biqM" version="v2" docAnchorId="#Creating" loadingMethod="dynamic" newTab="false" href="Creating a Report Using Microsoft Business Intelligence Development Studio.mdx"}.
3. If you upload a report template with the same name, the system asks if you want to update the existing report. If you select **Yes**, the system updates the existing report template in \{\{FRS\_VARS.ProductName}}.
4. To edit a customized or new report, access \{\{FRS\_VARS.ProductName}} and click **Download**. The system displays the **Report Templates** workspace.

<Paragraph indent="1">
  *Report Templates Workspace*
</Paragraph>

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/q2bSwUhS_4cnhtAZL3HrH_editanexistingreport.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/Edit_an_Existing_Report.png" githubPath="Content/Resources/Images/help-images/Edit_an_Existing_Report.png" showCaption="false"></Image>

5. From the Solution Explorer, right-click **Reports > Add > Existing Item**.
6. Edit the report according to your business requirements.

<Paragraph indent="1">
  *Existing Report*
</Paragraph>

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/cs1yHFEnFt8pLdmBzLg98_existingreport.png" position="flex-start" size="40" indent="1" initialPath="../../Resources/Images/help-images/Existing_Report.png" githubPath="Content/Resources/Images/help-images/Existing_Report.png" showCaption="false"></Image>

### About the SSRSReportExtension.dll File

As part of the reports module, \{\{FRS\_VARS.ProductName}} provides a custom Microsoft .NET DLL file called **SSRSReportExtension.dll**. This file calculates the duration for the \{\{FRS\_VARS.ProductName}} reports. In reports with DateTime values, you can filter the data based on a specified number of days, weeks, months, quarters, or years.

*Example Report with DateTime Value*

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/6Xc66O2bVecWSO_3RS0Yw_report-date-example.png" position="flex-start" size="40" initialPath="../../Resources/Images/help-images/report-date-example.png" githubPath="Content/Resources/Images/help-images/report-date-example.png" showCaption="false"></Image>

The **SSRSReportExtension.dll** file uses three methods called **getStartDateTime**, **GetEndDateTime**, and **GetValidationListSQL**. These methods run inside the Microsoft SSRS server and calculate the DateTime logic for the reports. The system passes the report parameters to those methods and the methods calculate the dates required for the reports.

In the **Dataset Properties** dialog box, under the **Parameters** heading, you can define parameters and their values.  You can use expressions for the parameter values.  For example, you could create a parameter called **@en\_datein** and set its value to **=Code.getStartDateTime(Parameters!DateRangeOptions.Value, Parameters!TimeZoneOffset.Value, Parameters!Last.Value)&#x20;**.  Use the query designer to declare and then call the parameter inside the query.
