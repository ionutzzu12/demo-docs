### About Scheduling a Report

You can schedule a report to run at specific times.

In previous releases, the system did not take into account daylight savings time when running scheduled reports. This is because the \{\{FRS\_VARS.ProductName}} reports use the Microsoft SSRS report builder which does not support the daylight savings time offset. Starting in \{\{FRS\_VARS.ProductName}} Release 2016.1, we added two **Time Zone** fields and the **Update Report Schedule DST** quick action to account for daylight savings time.

### Scheduling a Report

Use this procedure in conjunction with the procedure described in [Creating Reports and Report Templates](Creating_Custom_Reports.mdx).

1. Log in to \{\{FRS\_VARS.ProductName}}.
2. Open the **Reports** workspace. The system displays a list of reports.
3. Click the **edit** :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/WTLKbMa5PI7qeazzbj0FD_edit-old.png" alt caption size="10" width="16" height="16"} icon next to the report to set the scheduling parameters for.
4. In the **Schedule Information** section, enter the following (you can only select one):

5. Click **Save**.

After you run the scheduled report, you can download it from the report center (if you selected **Download** in the **Distribution Information** section) or email it (if you selected **Email** in the **Distribution Information** section). The schedule status indicates if the report ran successfully and the time when it ran.

### About Daylight Savings Time

Under certain conditions, reports created while Daylight Savings Time (DST) is in effect might not show events on the correct days. To prevent incorrect reporting, ensure that the event time does not occur exactly at midnight (12:00 AM) on the day that Daylight Savings Time ends.

For example, if you specify a date of 4 Nov 2012 12:00 AM, the record appears on the report as having occurred on November 3. Set the event time to something other than midnight (such as 1:00 AM) to yield a correct report. The following illustrates this concept:

### Working with the Update Report Schedule DST Quick Action

To ensure that scheduled reports run at the correct time during daylight savings time, run the **Update Report Schedule DST** quick action. You can either schedule it to run at certain intervals, such as daily or monthly, or you can manually run it any time.  When daylight savings time happens, the schedule shifts and the system runs the report at the correct time.

For this quick action to run correctly, you must enter a value other than **Your local time** in either of the **Time Zone** fields; that is, you must select a specific time zone for the report schedule.

Follow the steps below to manually run the **Update Report Schedule DST** quick action:

1. Log in to \{\{FRS\_VARS.ProductName}}.
2. Open the **Notification** workspace.
3. From the Action Menu, select **No Category > Update Report Schedule DST**. The system runs the **Update Report Schedule DST** quick action.
