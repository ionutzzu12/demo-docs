Once a Demand is approved and goes to the **Assessing** phase, it stays in the **Assessing** phase until it is accepted into a Portfolio. When the Portfolio Manager accepts the Demand into a Portfolio, the Demand becomes a Project. The Portfolio record now has a **Demands** tab, and the **Details** tab now has an **Estimated Effort** field, which is calculated based on the Projects in the Portfolio.

There is also a new set of Budget fields on the Overview form. In addition to OOTB Allocated and Consumed fields, there is now a **Remaining Budget**, **Estimated Consumed Budget**, and **Estimated Remaining Budget** calculation. Estimated fields are calculated using numbers from Demands in the Portfolio that are in **Assessing** status. Project budget information is also included. In the **Demands** tab, you can select Demands and select **Remove from Budgets** or **Include in Budgets** to see how the overall Portfolio budget is affected.

**To include a Demand in a Portfolio:**

1. Select the Demand in the **Demands&#x20;**&#x74;ab and click **Accept into Portfolio - Create Project**.
2. Select an **Owner&#x20;**&#x66;or the Project.
3. The Demand is converted to a Project and the Demand is set to **Closed**.
