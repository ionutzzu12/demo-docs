With the addition of resource capacity, you can select and updates resources from the Project Phase itself.

## Selecting a Resource

1. Log in as a Portfolio Manager, Project Manager, or Demand Manager.
2. Open the **Project Phase** workspace.
3. Select a Phase from the list.
4. Select the **Resources** tab.
5. Click **Link** and select the required resources from the list.
6. Click **Select** to add the resources.

7. Once the resources are added, click **Save**.

To calculate remaining capacity of a resource under the **Resources** tab, refer to :Link[Calculate Resource Capacity]{label="Calculate Resource Capacity" overridedLabel="Calculate Resource Capacity" spaceId docId="GFDw3UWZa6cQmfDyGu1Kj" version="v2" docAnchorId="#Calculat" loadingMethod="dynamic" newTab="false" githubPath="Using_Resource_Capacity.htm#Calculat" href="Using_Resource_Capacity.mdx"}.

**Create a task for the resource**:

To create a task for a resource, do the following:

1. Click the **Resources** tab in the **Project Phase** workspace.
2. Select an employee from the given list who has the capacity to complete a task.
3. Click **Create Task** and fill in the following fields:

<Paragraph indent="1">
  a. **Subject\***: Enter a name for the task.
</Paragraph>

<Paragraph indent="1">
  b. **Details**: Enter any details associated with the task.
</Paragraph>

<Paragraph indent="1">
  c. **Phase\***: From the dropdown, select the project phase you want to create the task for.
</Paragraph>

<hint type="info" indent="2">
  The dropdown lists only the Phases that are in the **Active** or **Initial** state and have a Project associated with them.
</hint>

<Paragraph indent="1">
  d. **Priority**: From the dropdown, assign the task priority. The value ranges from 1 to 5 (in the order of highest to lowest).
</Paragraph>

<Paragraph indent="1">
  e. **Planned Effort (Hrs.)**: Enter the expected effort in hours for completing the task.
</Paragraph>

4. Click **OK**.

The task is now created and it appears under the **Task** tab in the Project Phase workspace.

<hint type="info">
  Mandatory fields are marked with an asterisk.
</hint>

<hint type="danger">
  Some Project Phases do not have a Project linked when created by users with the Administrator role. As a result, these Project Phases do not appear as options when selecting a phase while creating a new task. Therefore, we recommend that users with the Demand Manager, Project Manager, or Portfolio Manager roles create new Project Phases from within the Project workspace.
</hint>
