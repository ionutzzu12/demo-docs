The following are some examples of how to use search.

<ExpandableHeading>
  ## Example: Creating a Saved Search with Expressions

  To search for all incidents that are breaching within the next 24 hours, set up the search query fields as follows:

  1. Log into the \{\{FRS\_VARS.Neurons\_ITSM\_-\_Service\_Desk\_Console}}.
  2. Open the **Incident** workspace.
  3. Open the **Saved Searches** menu. See :Link[Viewing the List of Saved Searches Available to Your Role]{label="Viewing the List of Saved Searches Available to Your Role" overridedLabel="Viewing the List of Saved Searches Available to Your Role" spaceId docId="RYWYC0IvsLLi8QWlusgWZ" version="v2" docAnchorId="#ViewingTheListOfSavedSearch" loadingMethod="dynamic" newTab="false" githubPath="Saved Searches.htm#ViewingTheListOfSavedSearch" href="Saved Searches.mdx"} for information about opening the **Saved Searches** menu.
  4. Click **New Saved Search**. A blank search form appears.
  5. Enter a search name, such as "Expression Example", and click **Save**.
  6. Set up your query as shown in the example below:

  | Business Object  | Field          | Operator         | Criteria                                                                            |
  | ---------------- | -------------- | ---------------- | ----------------------------------------------------------------------------------- |
  | Incident         | Status         | In List          | Active, Logged, Waiting for 3rd Party, Waiting for Customer, Waiting for Resolution |
  | AND              |                |                  |                                                                                     |
  | Is Related To    | At least       | 1                | Escalation Watch via IncidentAssociatedEscalationWatch (0...1 : 0...N)              |
  | WHERE            |                |                  |                                                                                     |
  | Escalation Watch | BreachDateTime | Greater or Equal | $(Today())                                                                          |
  | AND              |                |                  |                                                                                     |
  | Escalation Watch | BreachDateTime | Less than        | $(Tomorrow())                                                                       |

  7. Click **Run Search** to see the results.
  8. Click **Save**.

  ## Example: Using Find and Replace

  You can create a quick action that enables you to find and replace specific values in specific fields in an entire saved search list, a selection of items, or a single item.

  The **Find and Replace** quick action looks through the records for any instances of the specified field that contain the specified current value. When it finds that field and current value, it replaces the current value with the new value.

  *Find and Replace Quick Action -- Before*

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/3qlviqh6tXM6BdQH5POmE_findandreplacenew.png" position="flex-start" size="40" initialPath="../../Resources/Images/help-images/find_and_replace_new.png" githubPath="Content/Resources/Images/help-images/find_and_replace_new.png" showCaption="false"></Image>

  In this example, the name is Owner Change. No description is added. The category is Form Actions.  (If you do not choose a category, the quick action is listed in **No Category**.)

  *Find and Replace Quick Action -- After*

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/vHOW4aUbZS-489Z-YKCrR_findandreplacenewqa.png" position="flex-start" size="40" initialPath="../../Resources/Images/help-images/find_and_replace_new_qa.png" githubPath="Content/Resources/Images/help-images/find_and_replace_new_qa.png" showCaption="false"></Image>

  ## Example: Finding Records that will Breach

  This topic describes how to create a saved search that searches for records that will breech within a specific time, in this case, records that will breach in the next seven days. This search uses expressions.

  1. Log into the Service Desk Console.
  2. Open a workspace.
  3. Open the **Saved Searches** menu. See :Link[Viewing the List of Saved Searches Available to Your Role]{label="Viewing the List of Saved Searches Available to Your Role" overridedLabel="Viewing the List of Saved Searches Available to Your Role" spaceId docId="RYWYC0IvsLLi8QWlusgWZ" version="v2" docAnchorId="#ViewingTheListOfSavedSearch" loadingMethod="dynamic" newTab="false" githubPath="Saved Searches.htm#ViewingTheListOfSavedSearch" href="Saved Searches.mdx"} for information about opening the **Saved Searches** menu.
  4. Click **New Saved Search** to create a new search. See [Using Saved Search](<Saved Searches.mdx>) for more information on how to create a saved search.
  5. In the search criteria area, choose the following:

  | Business Object                                                      | Field          | Operator     | Criteria             |
  | -------------------------------------------------------------------- | -------------- | ------------ | -------------------- |
  | Escalation Watch via IncidentAssocResolutionEscWatch (0...1 : 0...1) | BreachDateTime | Equal to     | $(BeginningOfDay(7)) |
  | AND                                                                  |                |              |                      |
  | Escalation Watch via IncidentAssocResolutionEscWatch (0...1 : 0...1) | BreachDateTime | Greater than | $(CurrentDate())     |
  | AND                                                                  |                |              |                      |
  | Incident                                                             | Status         | Not In List  | Closed, Resolved     |

  <Paragraph indent="1">
    For more information, see [About Using Expressions in Searches and Dashboards](../../Reference/Expressions/Expressions_in_Searches_.mdx)
  </Paragraph>

  <Paragraph indent="1">
    *Example: Search the Next 7 Days*

  6. Click **Preview** to check your search results.
  7. Click **Save**.
